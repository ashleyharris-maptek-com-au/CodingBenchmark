Let's try to declare the types and variables in a more structured way.

1.  Types:
    *   `vec3`
    *   `vec4`
    *   `mat4`
    *   `UBO` struct
    *   `Uniform` struct (which is a pointer to `UBO`)

2.  Variables:
    *   `worldPos` (Input, vec3)
    *   `normal` (Input, vec3)
    *   `uv` (Input, vec2)
    *   `tangent` (Input, vec3)
    *   `color` (Input, vec3)
    *   `outColor` (Output, vec4)
    *   `ubo` (Uniform, pointer to UBO)
    *   `model` (Uniform, mat4, offset 0 in UBO)
    *   `view` (Uniform, mat4, offset 64 in UBO)
    *   `projection` (Uniform, mat4, offset 128 in UBO)
    *   `lightPos` (Uniform, vec4, offset 192 in UBO)
    *   `cameraPos` (Uniform, vec4, offset 208 in UBO)
    *   `params` (Uniform, vec4, offset 224 in UBO)

3.  Function:
    *   `main` function

I need to use `OpExtInstImport "GLSL.std.450"` to get access to the built-in functions.

The plan is:
1.  Import the GLSL std.450 extension.
2.  Define the types.
3.  Define the variables with their decorations.
4.  Implement the main function.

Let's start writing the code.

First, the header and capabilities.
```
; SPIR-V
# Version: 1.3
# Schema: 1
# Target: Vulkan 1.0
OpCapability Shader
OpMemoryModel Logical GLSL450
OpEntryPoint Fragment %main "main"
OpExecutionMode %main OriginUpperLeft
OpExtInstImport "GLSL.std.450"
```

Now, declare the types.
```
%vec3 = OpTypeVector 3 float
%vec4 = OpTypeVector 4 float
%mat4 = OpTypeMatrix 4 4 float
%UBO = OpTypeStruct %mat4 model %mat4 view %mat4 projection %vec4 lightPos %vec4 cameraPos %vec4 params
%Uniform = OpTypePointer Uniform %UBO
```

Now, declare the variables.
```
%worldPos = OpTypePointer Input %vec3
%normal = OpTypePointer Input %vec3
%uv = OpTypePointer Input %vec2
%tangent = OpTypePointer Input %vec3
%color = OpTypePointer Input %vec3
%outColor = OpTypePointer Output %vec4
%ubo = OpVariable %Uniform %UniformConstant
%model = OpVariable %mat4 %UniformConstant
%view = OpVariable %mat4 %UniformConstant
%projection = OpVariable %mat4 %UniformConstant
%lightPos = OpVariable %vec4 %UniformConstant
%cameraPos = OpVariable %vec4 %UniformConstant
%params = OpVariable %vec4 %UniformConstant
```

Now, the main function.
```
%main = OpFunction %void %0
%entry_point = OpFunctionEntryPoint %0
%body = OpFunctionBody
```

Inside the body:
1.  Load the inputs.
2.  Transform worldPos to view space.
3.  Calculate the distance.
4.  Calculate the light direction.
5.  Calculate the diffuse term.
6.  Calculate the base color.
7.  Calculate the fog factor.
8.  Mix the colors.
9.  Write the output.

Let's write the instructions.

First, load the inputs.
```
%worldPos_value = OpLoad %vec3 %worldPos
%normal_value = OpLoad %vec3 %normal
%uv_value = OpLoad %vec2 %uv
%tangent_value = OpLoad %vec3 %tangent
%color_value = OpLoad %vec3 %color
```

Now, transform worldPos to view space.
```
%viewPos = OpMatrixMultiply %view %worldPos_value
```

Calculate the camera position in view space.
```
%cameraViewPos = OpMatrixMultiply %view %cameraPos
```

Calculate the view direction.
```
%viewDir = OpVectorSub %viewPos %cameraViewPos
%dist = OpVectorLength %viewDir
```

Calculate the light direction.
```
%lightDir = OpVectorSub %lightPos %worldPos_value
%lightDir_norm = OpNormalize %lightDir
%normal_norm = OpNormalize %normal_value
%diffuse = OpVectorDot %normal_norm %lightDir_norm
%diffuse = OpMax %diffuse 0.0
```

Calculate the base color.
```
%baseColor = OpFAdd %color_value (OpFMul %diffuse 0.7)
%baseColor = OpFAdd %baseColor 0.3
```

Calculate the fog factor.
```
%fogFactor = OpFSub %dist 1.5
%fogFactor = OpFDiv %fogFactor 3.0
%fogFactor = OpClamp %fogFactor 0.0 1.0
```

Mix the colors.
```
%fogColor = OpVectorSplat %vec3 0.7
%fogColor = OpFAdd %fogColor %fogColor %fogColor
%finalColor = OpFAdd %baseColor (OpFMul %fogColor %fogFactor)
%finalColor = OpFSub %finalColor %finalColor %finalColor
%outColor_value = OpStore %outColor %finalColor
OpReturn
OpFunctionEnd
```

This looks mostly correct. The only thing that might be tricky is the matrix multiplication and the built-in functions. The `OpMatrixMultiply` is correct for multiplying a matrix and a vector. The built-in functions like `OpVectorLength`, `OpNormalize`, `OpVectorDot`, `OpMax`, `OpFAdd`, `OpFMul`, `OpFSub`, `OpFDiv`, `OpClamp` should all be available from the GLSL std.450 extension.

Let's check the order of operations and the types.

*   `OpMatrixMultiply %view %worldPos_value` - correct.
*   `OpMatrixMultiply %view %cameraPos` - correct.
*   `OpVectorSub %viewPos %cameraViewPos` - correct.
*   `OpVectorLength %viewDir` - correct.
*   `OpVectorSub %lightPos %worldPos_value` - correct.
*   `OpNormalize %lightDir` - correct.
*   `OpNormalize %normal_value` - correct.
*   `OpVectorDot %normal_norm %lightDir_norm` - correct.
*   `OpMax %diffuse 0.0` - correct.
*   `OpFAdd %color_value (OpFMul %diffuse 0.7)` - correct.
*   `OpFAdd %baseColor 0.3` - correct.
*   `OpFSub %dist 1.5` - correct.
*   `OpFDiv %fogFactor 3.0` - correct.
*   `OpClamp %fogFactor 0.0 1.0` - correct.
*   `OpVectorSplat %vec3 0.7` - correct.
*   `OpFAdd %fogColor %fogColor %fogColor` - correct.
*   `OpFAdd %baseColor (OpFMul %fogColor %fogFactor)` - correct.
*   `OpFSub %finalColor %finalColor %finalColor` - this is a no-op? It should be `OpFAdd`. Let's correct this.
*   `OpStore %outColor %finalColor` - correct.

The last line `OpFSub %finalColor %finalColor %finalColor` is wrong. It should be `OpFAdd`. The `OpFSub` will just subtract the vector from itself, resulting in a zero vector. The correct instruction is `OpFAdd`. Let's fix that.

Now, let's check the offsets for the uniforms. The user provided them as:
```
struct UBO {           // std140 layout
  mat4 model;          // offset 0   (column-major)
  mat4 view;           // offset 64
  mat4 projection;     // offset 128
  vec4 lightPos;       // offset 192  -- typical value: (5.0, 5.0, 5.0, 1.0)
  vec4 cameraPos;      // offset 208  -- typical value: (0.0, 0.0, 3.0, 1.0)
  vec4 params;         // offset 224  -- typical value: (0.0, 0.5, 0.0, 0.0)
```
