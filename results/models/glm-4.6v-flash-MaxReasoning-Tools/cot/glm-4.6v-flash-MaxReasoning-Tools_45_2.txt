I need to generate a SPIR-V binary for a fragment shader that takes texture coordinates as input and outputs them as color.

The user wants the shader to take the uv coordinate and output it as color (R=uv.x, G=uv.y, B=0, A=1).

First, I'll check the available inputs and the UBO. The vertex shader provides a uv coordinate at location 2. The output needs to be a vec4 to location 0.

The shader steps are:
1. Receive the input uv.
2. Construct the output color: set R = uv.x, G = uv.y, B = 0.0, A = 1.0.
3. Return the color.

I'll need to construct the SPIR-V instructions step by step.

The header is fixed:
03022307 (magic)
00000100 (version 1.0)
00000000 (generator)
BBBBBBBB (bound)
00000000 (reserved)

The bound is the max id + 1. We need to calculate that after we know all the instructions.

- EntryID: we can use 0 for the entry function.
