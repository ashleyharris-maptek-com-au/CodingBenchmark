The GLSL standard is "GLSL.std.450". The extension name is likely the same.

So, the first instruction would be OpExtInstImport with result ID and name "GLSL.std.450".

- OpTypeFunction
- OpTypeVector for vec3, vec4, etc.

The function has no parameters as it's a fragment shader.

We need to declare the input variables as OpVariable with decoration OpDecorationLocation and OpDecorationDescriptorSet and OpDecorationBinding.

The decorations:

Now, the actual shader code:

1. Read the UV coordinates from the input.
2. Compute the checkerboard pattern:
   - Multiply UV by 8 (or scale to [0,8) range).
   - Take floor to get integer part.
   - Compute parity: (floor.x + floor.y) % 2.
   - This parity determines the color (0.9 or 0.2).
3. Compute the light direction:
   - From lightPos to fragment position.
   - Normalize.
4. Compute the view direction:
   - From cameraPos to fragment position.
   - Normalize.
5. Compute the normal (from input).
6. Compute the diffuse color: max(0, dot(normalize(normal), normalize(lightDir))) * checkerboardColor.
7. Multiply by the material color (from input).
8. Output the color as vec4 with alpha 1.0.

Matrix multiplication: (M * v) = (dot(M[0], v), dot(M[1], v), dot(M[2], v), dot(M[3], v))

So, for each row of the matrix, take dot with the vector.

Similarly for view and projection.

Let's outline the steps:

1. Compute world position:
   - Get model matrix from UBO.
   - Get worldPos from input.
   - Compute row0: dot(model[0], worldPos)
   - row1: dot(model[1], worldPos)
   - row2: dot(model[2], worldPos)
   - row3: dot(model[3], worldPos)
   - Combine as vec4.

2. Compute view direction:
   - Get view matrix from UBO.
   - Get cameraPos from UBO.
   - Compute viewPos = view * (cameraPos - worldPos) or directly compute direction.
   - Actually, view matrix transforms from world to view space. So viewPos = view * (cameraPos - worldPos) or we can compute direction as (cameraPos - worldPos) transformed by view inverse? Wait, view matrix is from world to view space. So to get view space position, we can compute: viewPos = view * worldPos. Then view direction = viewPos - cameraPos_view = viewPos - (view * cameraPos). This is complex.

   Let's compute view space position: viewPos = view * worldPos.
   Then cameraPos_view = view * cameraPos.
   Then view direction = viewPos - cameraPos_view.

   So we need to compute view * cameraPos.

3. Compute light direction:
   - Get lightPos from UBO.
   - Compute lightDir = lightPos - worldPos.
   - Normalize.

4. Compute checkerboard pattern:
   - Get uv from input.
   - Scale by 8: (uv * 8.0)
   - Floor: floor(uv * 8.0)
   - Compute parity: (floor.x + floor.y) % 2
   - This is integer. Then mod with 2: parity % 2.
   - Then map to color: if parity == 0, color = 0.9; else 0.2.

   In SPIR-V, we can use OpFLOOR for float and OpCompositeExtract for integer components. Then OpAdd, OpSelect? Wait, OpSelect is for boolean selection. We can use OpSelect with a comparison.

   Let's see: we have floor.x and floor.y. Add them: OpFAdd. Then OpFloor? No, it's already integer. Then OpIAdd (since it's integer). Then OpSelect with OpICmp EQ to check if sum is even.

   Or we can compute parity as (floor.x + floor.y) & 1.

   Then use OpSelect with a constant to get the color.

   So:
   - OpFAdd (floor.x, floor.y) to get sum.
   - OpIAdd (sum, -1) to get (sum - 1) which is even if sum is odd? Wait, (sum & 1) is better.

   Let's do:
   - OpIAdd (sum, -1) -> parity' = sum - 1
   - OpSelect (parity', 0, 1) -> if sum is odd, parity' is even, so select 0 else 1.

   This is getting complicated.

   Let's use:
   - OpIAdd (sum, -1) -> parity' = sum - 1
   - OpSelect (parity', 0, 1) -> color = 0 if sum is odd, 1 if even.

   Then use OpSelect with this to choose between 0.9 and 0.2.

   So:
   - OpSelect (parity', 0.2, 0.9) -> checkerboardColor = 0.2 if sum odd else 0.9.

   This seems to work.

5. Compute diffuse color:
   - Normalize normal.
   - Normalize lightDir.
   - Compute dot product: OpDot.
   - OpMax (0.0, dot) to ensure non-negative.
   - Multiply by checkerboardColor.

6. Multiply by material color from input.

7. Output the final color as vec4 with alpha 1.0.
