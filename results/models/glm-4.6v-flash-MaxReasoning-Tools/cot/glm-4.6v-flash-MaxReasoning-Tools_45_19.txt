I need to generate a complete SPIR-V binary for a Phong lighting fragment shader.

The user wants a Phong lighting calculation with colored materials. The inputs are vertex positions, normals, UVs, tangents, and vertex color. The UBO contains model, view, projection matrices and light position, camera position and other parameters.

Key steps to implement:
1. Retrieve the uniform values from the UBO.
2. Transform worldPos to view space and then to clip space? Actually we need light position and camera position in world space for lighting calculations. We have lightPos and cameraPos in world space in the UBO.
3. Compute light direction (L) = (lightPos - worldPos).
4. Compute view direction (V) = (cameraPos - worldPos).
5. Compute normal (N) and maybe reflect direction (R) = reflect(-L, N).
6. Compute diffuse term: max(dot(N, L), 0) * color.
7. Compute specular term: pow(max(dot(R, V), 0), shininess) * (1,1,1).
8. Sum ambient, diffuse, specular.
9. Clamp the final color to [0,1].
10. Output as vec4.

The shader will be a fragment shader, so we need OpCapability Shader, and the execution model is Fragment.

The user also wants to clamp the sum to [0,1] and output alpha 1.0.

I need to construct the SPIR-V instructions step by step.

First, I will build the header.
- Magic number: 0x07230203 -> 03022307 (5 words)
- Version: 1.0 -> 0x00010000 -> 00000100
- Generator: 0 -> 00000000
- Bound: will be determined after we know the number of result IDs. For now we can put 0.
- Reserved: 0

- tangent: vec3, location 3
- color: vec3, location 4

We need to declare them as OpVariable with decoration Location and maybe DescriptorSet/Binding if we use them later.

The output: outColor: vec4, location 0.

UBO:
- mat4 model, view, projection
- vec4 lightPos
- vec4 cameraPos
- vec4 params
