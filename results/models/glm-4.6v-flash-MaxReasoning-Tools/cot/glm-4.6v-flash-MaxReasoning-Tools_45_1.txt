I need to generate a SPIR-V binary for a fragment shader that visualizes the surface normal as color.

The user wants a complete hex string of the SPIR-V binary. The shader should:
1. Take the normal from vertex input.
2. Normalize it (already normalized from vertex shader, but maybe still need to ensure it's unit length).
3. Map from [-1, 1] range to [0, 1] range.
4. Output as RGBA color with alpha 1.0.

The input variables:
- location 0: vec3 worldPos
- location 1: vec3 normal
- location 2: vec2 uv
- location 3: vec3 tangent
- location 4: vec3 color

The UBO has model, view, projection matrices and some other values.

The output is location 0: vec4 outColor.

I need to construct the SPIR-V binary according to the given format.

First, I'll outline the structure:
- Header: 5 words
- Then instructions.

Let's break down the header:
Magic number: 0x07230203 => 03022307 in little-endian hex.
Version: 1.0 => 0x00010000 => 00000100
Generator: 0 => 0
Bound: We need to know the max ID later.
Reserved: 0

The header is 5 words, so total header bytes = 5 * 4 = 20 bytes.

Now the instructions. Let's plan the shader.

The steps:
1. Get the normal from input.
2. Normalize it (maybe redundant but keep for safety).
3. Scale and bias to map [-1,1] to [0,1].
4. Pack into vec4 as RGB, with alpha = 1.

But we need to map [-1,1] to [0,1]. A common way is to do (value + 1) / 2.

So for a normal component n:
result = (n + 1.0) / 2.0

So for each of the three components we can compute.

The shader will be something like:

void main() {
    vec3 normal = gl_FragCoordNormal;
    vec3 color = (normal + 1.0) / 2.0;
    gl_FragColor = vec4(color, 1.0);
}
