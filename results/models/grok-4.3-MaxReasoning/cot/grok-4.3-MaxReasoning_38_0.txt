The task is to solve the Exact Cover problem in Python. It reads from stdin: first line has U and S, then U is the number of elements (0 to U-1), S is the number of sets. Then S lines, each with space...The problem is to solve the Exact Cover problem. I need to write Python code that reads from stdin the universe size U, number of sets S, then S sets each with elements, and find the exact cover, output the indices or NO SOLUTION.

Since it's NP-complete but instances are small, backtracking is fine.

 I need to implement a function to find the combination of sets that cover each element exactly once.

I'll use a list of sets, each set is a list of elements it covers.

I'll use a boolean array covered[0..U-1] initially false.

I'll do recursive backtracking, trying to include or not include each set in order, but to avoid duplicates, I can go sequentially, deciding for each set whether to take it or not, but only if it doesn't overlap with already covered.

To prune, I can start from set 0 to S-1, and for each, if I can add it (no overlap with 1 element), add it, recurse, then backtrack.

But that would be 2^S which is bad if S large, but probably S small too.

Better way: select sets that cover the next uncovered element.

I'll keep a list of uncovered elements, initially all.

But to find which one, I can scan 0 to U-1.

To make it efficient, precompute lists.
