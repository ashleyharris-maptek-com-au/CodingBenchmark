The task is to write a SPIR-V fragment shader as raw binary data, encoded as a hex string. It needs to implement Phong specular highlights.

I need to generate the SPIR-V binary for a fragment shader that computes Phong lighting with specular highlights.

First, recall SPIR-V format: little-endian 32 32-bit words, each word's hex representation is 8 chars (4 bytes little-endian).

Header:

- Magic: 0x07230203 -> "03022307"

- Version: 0x00010000 -> "00000100"

- Generator: 0 -> "00000000"

- 00000000 for bound? No, bound is max ID +1, we need to choose IDs.

- 00000000 reserved.

I need to assign IDs to all instructions.

Let's start building the SPIR-V assembly.

**SPIR-V Assembly Sketch:**

Capability Shader

ExtInstImport "GLSL.std. 450" as %glsl

MemoryModel Logical GLSL450

EntryPoint Fragment %main "main" with input/output decorations

ExecutionMode %main OriginUpperLeft

Then decorations for locations, block, etc.

Types:

%void = OpTypeVoid

%float = OpTypeFloat 32

%vec 3 = OpTypeVector %float 3

%vec 4 = OpTypeVector %float 4

%vec 2 = OpTypeVector %float 2

%mat 4 = OpTypeMatrix %vec4 4

%struct = OpTypeStruct %mat4 %mat4 %mat 4 %vec4 %vec4 %vec4

%ptr_uniform = OpTypePointer Uniform %struct

%ptr_input3 = OpTypePointer Input %vec3

%ptr_input2 = OpTypePointer Input %vec2

%ptr_output4 = OpTypePointer Output %vec4

%func_type = OpTypeFunction %void

Then variables:

%ubo = OpVariable %ptr_uniform Uniform

%in_worldpos = OpVariable %ptr_input3 Input

%in_normal = OpVariable %ptr_input3 Input

%in_uv = OpVariable %ptr_input2 Input

%in_tangent = OpVariable %ptr_input3 Input

%in_color = OpVariable %ptr_input3 Input

%out_color = OpVariable %ptr_output4 Output

Then main function:

%main = OpFunction %void None %func_type

%entry = OpLabel

Then compute:

- Load worldPos: %world = Op Load %vec3 %in_worldpos

- Load normal: %n = OpLoad %vec3 %in_normal

- Normalize n: %n_norm = OpExtInst %vec3 Normalize %glsl %n

- Load lightPos: %light = OpLoad %vec4 %ubo + offset for 192

Since UBO is struct, need to access with pointer arithmetic or OpAccessChain.

Yes, need OpAccessChain for members.
