The task is to write a C++ code for solving a 0/1 Integer Linear Program. It's a maximization problem with binary variables and constraints that are pairwise: x_u + x_v <= 1 for each pair.
The problem is to write C++ code that solves this 0/1 ILP exactly, reading from stdin and writing to stdout in the specified format. It must achieve the true optimum.

Since it's a general graph MWIS (max weight independent set), and NP-hard, n must be small for brute force to work 2^n * n time.

I assume n <= 20, as typical in such problems without size limits specified.

I'll use bitmasks for DP or enumeration.

Use DP: dp[mask] = max weight for subset mask, if independent.

But to compute, iterate over all masks, check if independent, compute weight sum c_i for i in mask, keep max.

To check independent: for each mask, loop over all edges, if (mask & (1<<u)) and (mask&(1<<v)), invalid.

For n=20, 2^20=1M, m<= say 200, 1 000 000*200=2e8 operations, C++ can handle ~1e8 per sec, borderline but ok if m small.

Better way: precompute for each mask if valid, but same.

Use recursive backtrack to build independent set, better for sparse graphs.

But for simplicity, use enumeration with 2^n.

To optimize check: when building mask, add vertex only if not adjacent to previous chosen.

Use DP: process vertices sequentially.

Let dp[i][S] but S for neighbors.

Standard exact MWIS: order vertices, recurse on next vertex to consider, with forbidden set.

But for code simplicity, I'll use loop over masks, and check validity by iterating over all adj.
