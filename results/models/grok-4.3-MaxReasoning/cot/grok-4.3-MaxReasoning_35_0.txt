The task is to write Rust code for the Minimum Dominating Set problem using heuristics since it's NP-hard. It needs to read from stdin, write to stdout, output a valid dominating set, and run within 5 minutes on the given hardware.

I need to implement a greedy algorithm: repeatedly select the vertex that covers the most uncovered vertices (including itself).

To make it efficient:

- Use adjacency list: Vec<Vec<usize>>

- Use a Vec<bool> for covered.

- Use a Vec<usize> for degree or count of uncovered neighbors.

- Use a priority queue (BinaryHeap) for vertices by their current count, max heap.

In 5 minutes, even O(n^2) might be fine if n<=1000, but since no n given, assume n up to 10^4 or so.

For large n, need better.

But let's implement a basic greedy with O(n * (n+m)) time, which for n=10^ 4, m=10^5 is 10^9 operations, too slow.

Need efficient implementation.

Standard greedy for dominating set:

- Maintain a set of uncovered vertices.

- For each uncovered vertex, compute how many it covers (neighbors uncovered + self).

- But to update efficiently, use a max-heap with (count, vertex), and lazy delete when invalid.

But in Rust, BinaryHeap.

Let's code it.

First, read graph, make adj Vec<Vec<usize>>, sort and dedup.

Then:
