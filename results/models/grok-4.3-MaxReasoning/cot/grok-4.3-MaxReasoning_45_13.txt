The task is to write a SPIR-V fragment shader as raw binary data, encoded as a hex string. It will be decoded from hex, validated with spirv-val, and rendered on a unit sphere.
The task is to write a SPIR-V fragment shader that colors based on world-space Y coordinate. It takes worldPos from input location 0, computes t = clamp(worldPos.y * 0.5 + 0.5), mixes colors from bottom (0.1,0.4,0.1 at t=0) to top (1.0,1.0,1.0 at t=1), outputs vec4(color,1.0) at location 0.

I need to generate the SPIR-V binary in hex string format.

I need to recall how to encode SPIR-V instructions.

 Let's assign IDs:

- 1: OpTypeVoid

- 2: OpTypeFloat 32

- 3: OpTypeVector float 3

- 4: OpTypeVector float 4

- 5: OpTypePointer Input 3 (for worldPos vec3)

- 6 : OpVariable Input %5

- 7: OpTypePointer Output 4

- 8: OpVariable Output %7

- 9: OpTypeFloat for constants

- 10: OpConstant for 0.5

- 11: OpConstant for 0.5

-  etc.

Now, to assign IDs, we assign sequentially as we go.

Let's list all Op instructions with assigned IDs.

1. OpCapability Shader

2. OpExtInstImport %1 "GLSL.std.450"

3. OpMemoryModel Logical GLSL450

4. OpEntryPoint Fragment %2 "main" %6 %8 wait, variables not yet defined.

The order is important: types before variables, variables before function, etc.

The order in SPIR-V is:

Header

Capability

ExtInstImport

MemoryModel

EntryPoint

ExecutionMode

Decorate

Then type declarations

Then variable declarations

Then function definitions

So let's follow that.

Let's assign IDs starting from 1 for result IDs.

- OpCapability Shader

- OpExtInstImport %1 "GLSL.std.450"

- OpMemoryModel

- OpEntryPoint %2 "main"   // entry ID 2

- OpExecutionMode %2

- OpDec %3 for worldPos location

- OpDec %4 for outColor location

Then type:

- OpTypeVoid %5

- OpTypeFloat %6 32

- OpTypeVector %7 %6 3

- OpTypeVector %8 %6 4

- OpTypeFunction %9 (void)

- OpTypePointer %10 Input %7

- OpTypePointer %11 Output %8

Then variables:

- OpVariable %12 %10 Input   // worldPos ID 12

- OpVariable %13 %11 Output   // outColor ID 13

Then function:

- OpFunction %2 %5 None %9   // function ID 2

- OpLabel %14

- OpLoad %15 %7 %12   // load worldPos to %15 vec3

- OpCompositeExtract %16 %15 1   // y to %16 float

- OpConstant %17 % 6 0.5   // %17 =0.5

- OpFMul %18 %16 %17   // %18 = y*0.5

- OpFAdd %19 %18 %17   // %19 = t

- OpConstant %20 %6 0.0

 OpConstant %21 %6 1.0

- OpExtInst %22 %1 43 %19 %20 %21   // %22 = clamp(t,0,1)

- OpConstant %23 %6 0.1

- OpConstant %24 %6 0.4

- // for bottom

- OpCompositeConstruct %25 %7 %23 %24 %23   // bottom

- OpConstant %26 %6 1.0   // reuse %21

- OpCompositeConstruct %27 %7 %21 %21 %21   // top

- OpExtInst %28 % 1 26 %25 %27 %22   // %28 = mix

- OpCompositeExtract %29 %28 0

- OpCompositeExtract %30 %28 1

- OpCompositeExtract %31 %28 2

- OpConstant %32 %6 1.0   // reuse

 OpCompositeConstruct %33 %8 %29 %30 %31 %32   // vec4

- OpStore %13 %33

- OpReturn

- OpFunctionEnd
