Let's clarify the byte order.

The SPIR file is a sequence of bytes, little-endian for the 32-bit words.

That is, for a word with value W, the bytes in file are low byte first: W & 0xFF, (W>>8)&0xFF, etc.

But for the string, the string bytes are placed in order in the instruction stream, so the first byte of string is the first byte in the file for that part.

For the string word, the first 4 bytes of string are placed in the 4 bytes of the word, and then the word is stored little-endian in the file.

So for string first 4 bytes B0 B1 B2 B3, the word value is B0 | B1<<8 | B2<<16 | B3<<24

Then in file, the 4 bytes are low to high: B0, B1, B2, B3

So the hex is B0 B1 B2 B3 as hex digits.

For "G L S L" B0=0x47, B1=0x4C, B2=0x53, B3=0x4C
