The task is to write a complete Node.js JavaScript program for planning grid paths for search drones.
The task is to write a Node.js program that reads a JSON input from stdin, plans paths for drones on a grid to cover probabilities, and outputs the paths in JSON to stdout.

Input has:

- probability: 2D array, -1 obstacle, >0 search value

- drones: array of {id 0 to n-1, x, y}

- maxStepsPerDrone: number

Output: {"paths": [path0, path1, ..., pathn]} where pathi is array of [x,y] positions for that drone.

Each path starts with start, then moves or stays, length <= maxSteps 1 + maxStepsPerDrone I guess.

I need to implement a planner that maximizes unique covered probability.

Since no external deps, pure JS.

For scaling, need efficient algo.

Let's assume grid size up to say 100x100, drones few, steps up to 100 or so.

But grader on larger, so need O(N log N) or better.

Idea: use greedy assignment.

First, identify all valuable cells: list of {val, x, y} where val >0 and not -1.

Sort by val descending.

Then, for each drone, simulate moving to the highest val cell that is reachable within remaining 0 steps, but need to plan the entire path.

A simple way: each drone will follow a path that goes to the closest uncovered high val cell, then next, etc, but choosing order.

But to coordinate, use a priority queue for cells.

Perhaps use a multi-source approach or something.

Since it's to beat greedy reference, probably the reference is each drone independently goes to its closest high, then next, etc.

To do better, I can have all drones collectively visit the highest ones first.

So, sort the valuable cells by val desc.

Then, for each cell in that order, assign it to the drone that can reach it with least cost, i.e. min dist, and has remaining steps enough, and not yet assigned to others.

But then, for each assignment, I need to route the drone to it, but drones have paths, so I need to plan the order for each drone.

This is like vehicle routing problem, hard but for small, can approximate with greedy.

Let's implement a sequential assignment with greedy routing.

Code structure:

- Read data

- Get rows = prob.length, cols = prob[0].length

- Create a list of targets: array of {val: prob[y][x], x, y} for all x 0..cols-1, y0..rows-1 if prob[y][x] >0

- Sort targets by val descending: targets.sort((a,b) => b.val - a.val)

- Create covered: Set of strings 'x,y' or 2D bool array bool covered[rows][cols ] = false

- For each drone, have its current position, its path array, its remaining steps

- Initialize for each drone: path = [[x,y]], current = start, remaining = maxSteps, covered[start] = true if val>0

- Then, for each target in sorted order (highest 1st):

  - Find which drone can reach it best: for each drone, compute shortest dist to target, if dist <= remaining for that drone, and not covered[target]

  - Choose the one with smallest dist, or if tie smallest id or something.

  - If found, for that drone, find the 1D shortest path from current to target, add to its path, update current, subtract dist from remaining, mark all cells on path as covered (but since shortest, mark target and intermediates)

  - But mark only if not already.

- After all assignments, for drones that have remaining steps, they can stay or move to some uncovered low val, but for now, just stay, so append stays.

This way, high val are assigned to closest drone that can reach in time.
