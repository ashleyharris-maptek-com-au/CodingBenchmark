The task is to write a complete Node.js JavaScript program for planning grid paths for search drones.

Input: JSON with probability (2D array, -1 obstacle, >0 search value), drones (array of {id, x, y}), maxStepsPerDrone.

Output: JSON {"paths": [[[x,y ],[x,y],...], ...]} for each drone's path.

Paths: sequences of [x,y] positions, starting at drone pos, length <= maxStepsPerDrone+1, moves UDLR or stay, no out of bound, no -1.

Score based on unique prob covered vs greedy ref.

 I need to implement a program that reads from stdin, outputs to stdout the JSON paths.

  Use good heuristic to maximize unique covered prob.

  Since scalable, need efficient algo.

  Grid size unknown, but with 64GB mem, perhaps large but single thread.

  Let's plan algo.

  Simple greedy: simulate time steps from 0 to maxSteps.

  At each time, for each drone, choose the move (incl stay) that leads to the highest prob uncovered cell in neighborhood, or something.

  But need to track visited (covered) cells globally.

  Yes, use a set of covered cells.

  But since 2D grid, use a Set of strings "x,y" or a 2D bool array.

  For each step:

    Collect for each drone its possible next positions: current and neighbors valid.

    Then, for each drone, select the next pos with max prob if not covered, prefer high prob.

    But to avoid all drones choosing same cell, perhaps use a priority list.

  To make it better: use a global priority queue of uncovered high prob cells.

  Then assign drones to closest or something.

  But for simplicity, let's do per step greedy assignment.
