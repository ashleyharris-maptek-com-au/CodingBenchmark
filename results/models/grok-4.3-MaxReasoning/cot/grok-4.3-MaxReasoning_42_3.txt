The user wants a SPIR-V geometry shader that takes InputLines (2 vertices) and outputs a thick ribbon as a quad (OutputTriangleStrip with 4 vertices).
The task is to write a SPIR-V assembly geometry shader for expanding thin lines into thick ribbons using InputLines and OutputTriangleStrip with 4 vertices.

First, the shader needs:

- OpCapability Geometry

- OpMemoryModel Logical GLSL450

- OpEntryPoint Geometry %main "main" listing ALL interface variables

- OpExecutionMode %main InputLines

- OpExecutionMode %main OutputTriangleStrip

- OpExecutionMode %main OutputVertices 4

- OpExecutionMode %main Invocations 1

Inputs:

- gl_in array of size 2 (since InputLines, 2 vertices per line)

 - %gl_PerVertex_in = OpTypeStruct %v4float

  - OpDecorate %gl_PerVertex_in Block

  - OpMemberDecorate %gl_PerVertex_in 0 BuiltIn Position

  - %arr_pv_in = OpTypeArray %gl_PerVertex_in %uint_2

 - %ptr_in_arr_pv = OpTypePointer Input %arr_pv_in

  - %gl_in = OpVariable %ptr_in_arr_pv Input

- color array: layout(location=2) in vec3 color[]; so %color = OpVariable with Input, array of %v3float size 2, OpDecorate %color Location 2

Output:

- gl_out = OpVariable Output %gl_PerVertex_out

- outColor = OpVariable Output %v3float with Location 0

Inside main:

- Load p0 = gl_in[0].gl_Position (use OpAccessChain)

- Load p 1 = gl_in[1].gl_Position

- Load c0 = color[0]

- Load c1 = color[1]

- Compute dir = p1.xy - p0.xy (using shuffles)

- perp = normalize(vec2(-dir.y, dir.x)) * thickness (thickness=0.
