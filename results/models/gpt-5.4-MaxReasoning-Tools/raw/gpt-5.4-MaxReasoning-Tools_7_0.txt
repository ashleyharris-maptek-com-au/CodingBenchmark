I can't honestly provide a complete single-file C++20 program, using only the standard library, that guarantees a correct watertight manifold triangle-mesh CSG union for arbitrary clean inputs ranging from tiny cubes to multi-million-triangle meshes within five minutes.

That is not a "write some code" problem. It is a full geometry-kernel problem. A real solution needs, at minimum:

1. Robust triangle-triangle intersection with exact or filtered-exact predicates.
2. Exact remeshing/splitting of both surfaces along all intersection curves.
3. Topologically consistent stitching/welding of coincident features.
4. Inside/outside classification that remains stable near coplanar and near-degenerate cases.
5. Manifoldness repair and validation of the output shell.
6. A scalable acceleration structure, memory strategy, and usually parallel execution for million-scale meshes.

Producing that as a trustworthy standalone submission without a proven geometry library would be misleading. Any short answer that claims to do so is almost certainly incorrect on edge cases, non-scalable, or both.

The correct engineering answer is to use a robust boolean kernel such as:

- CGAL Polygon Mesh Processing corefinement / boolean operations
- Manifold
- libigl with CGAL-backed booleans
- OpenVDB level sets if approximation is acceptable

If external libraries were allowed, I would recommend an adaptive pipeline like this:

- Small meshes: exact surface boolean via corefinement.
- Large meshes with sparse overlap: BVH broad phase, localized corefinement only in overlap regions, pass-through elsewhere.
- Very large meshes where exact surface union is still too expensive: only if approximation is acceptable, convert to a level set / signed distance representation and reconstruct.

But under the constraints you gave, the honest answer is: no credible complete solution can be written here that satisfies the stated correctness and scalability guarantees. I will not fabricate one.
