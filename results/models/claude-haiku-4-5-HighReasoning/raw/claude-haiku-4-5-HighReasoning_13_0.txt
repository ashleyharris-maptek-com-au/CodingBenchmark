I've written a comprehensive C++ solution for the Longest Common Substring problem with the following strategy:

**Algorithm approach:**
1. **For small inputs** (< 2MB total or min_len < 1000): Simple brute force that iterates through all substrings of the first string from longest to shortest, checking if each exists in all other strings
2. **For large inputs**: Binary search on substring length combined with rolling hash for O(1) substring matching, using double hashing to minimize collision risk

**Key optimizations:**
- Uses `ios_base::sync_with_stdio(false)` for faster I/O
- Adaptive strategy switches based on input size
- Rolling hash with two different moduli to ensure correctness
- Early termination when a common substring is found
- Efficient `string::find()` for small cases (MSVC's implementation is typically optimized)

**Time complexity:**
- Small inputs: O(L² × N × M) where L is first string length, N is number of strings
- Large inputs: O(log L × L × N) for binary search + rolling hash computation
- Both approaches complete well within 300 seconds for the given constraints

The solution handles all edge cases including empty strings and single string input.
