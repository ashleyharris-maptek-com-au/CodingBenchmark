I have completed the task of writing a scalable, memory-efficient maze solver in Rust.

The implementation utilizes Breadth-First Search (BFS) to ensure the shortest path is found. To handle the scale of up to 50,000x50,000 cells (2.5 billion total), the grid is stored as a 1D `Vec<u8>` and the path predecessors are tracked using a `Vec<u32>` array, optimizing memory usage to fit within the 32GB RAM constraint (utilizing approximately 12.5GB for the grid and predecessors).

The code is available in the project files as `Cargo.toml` and `src/main.rs`.
