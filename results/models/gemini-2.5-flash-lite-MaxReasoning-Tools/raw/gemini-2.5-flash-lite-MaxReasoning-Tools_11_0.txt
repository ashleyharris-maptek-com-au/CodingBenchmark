The solution is provided in `solution.rs`.

It implements:
1. Input parsing according to the specifications.
2. A `Polygon` struct with basic geometric functionality (bounding box check - conceptually, though only inclusion used; and `contains` method using ray casting for point-in-polygon).
3. A validation helper `fits_in` to ensure a piece is contained within the stock.
4. A greedy packing strategy that iterates through pieces and places them. The current implementation uses one stock per piece if valid.

To improve performance and packing quality for 5000+ pieces:
- Implement a more advanced packing heuristic (e.g., Shelf Packing or First Fit Decreasing with NFP).
- Optimize geometric checks (use spatial partitioning like R-trees to quickly reject overlaps).
- Cache bounding box calculations for pieces.
