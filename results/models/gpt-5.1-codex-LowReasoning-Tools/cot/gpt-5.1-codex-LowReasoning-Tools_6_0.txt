**Designing scalable orbital TSP solver**

I'll create a heuristic-based solver for the traveling salesman problem on orbital states, estimating delta-v costs using physics-inspired approximations like Hohmann transfers and plane change maneuvers. Depending on the number of stations, I'll switch between exact dynamic programming for small N, iterated local search for medium N, and greedy/metaheuristics with 2-opt for large N, aiming for readable, modular code handling cost matrix creation, initial tours, and iterative improvements.
