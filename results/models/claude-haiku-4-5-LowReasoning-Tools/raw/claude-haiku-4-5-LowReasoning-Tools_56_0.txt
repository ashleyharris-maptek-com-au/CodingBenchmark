I've written a Vehicle Routing Problem solver that:

1. **Uses nearest neighbor heuristic** to build initial routes efficiently, distributing packages across available trucks
2. **Applies 2-opt local optimization** to improve route quality by eliminating crossing paths
3. **Limits optimization iterations** to keep performance reasonable for large inputs
4. **Outputs pure JSON** with no debug logs

The algorithm prioritizes scalability while maintaining solution quality through:
- O(n²) nearest neighbor construction (fast for large n)
- Bounded 2-opt improvements with iteration limits
- Direct index-based operations to minimize memory overhead

The solution handles the benchmark's constraint of increasingly large scenarios by using efficient heuristics rather than exhaustive optimization.
