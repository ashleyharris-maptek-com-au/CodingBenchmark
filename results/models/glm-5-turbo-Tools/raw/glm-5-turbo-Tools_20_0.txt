## Drillhole Assay Data Validator

### Architecture

The program uses a **three-layer spatial outlier detection** strategy:

1. **Global z-score** — compares each value against the median and MAD (Median Absolute Deviation) of all samples across all holes for that property. This catches values that are globally unusual.

2. **Local along-hole z-score** — compares each value against neighboring samples in the same hole (within a sliding window). This catches values that don't fit the local trend even if they're within global range.

3. **Spatial neighborhood z-score** — compares each value against samples from nearby holes within a configurable 3D search radius using a spatial hash grid. This catches values that are inconsistent with the geological neighborhood.

The three z-scores are combined via a **weighted Euclidean norm** (global=1.0, local=0.5, spatial=0.3), and a **piecewise confidence mapping** converts the combined score to [0.5, ~1.0]:

| Combined z | Confidence |
|-----------|-----------|
| < 2       | 0.500     |
| 10        | 0.800     |
| 30        | 0.950     |
| > 30      | → 0.999   |

### Adaptive Configuration

| Dataset Size | z Threshold | Spatial Range | Local Window |
|-------------|-------------|--------------|-------------|
| < 2,000 samples | 2.0 | 200m | 30 |
| < 50,000 samples | 2.5 | 80m | 15 |
| ≥ 50,000 samples | 3.0 | 40m | 10 |

### Key Design Decisions

- **MAD-based robust statistics** (resistant to up to 50% contamination) with IQR and heuristic fallbacks
- **Spatial hash grid** for O(1) amortized neighbor lookups (disabled for >500M cells)
- **Median-of-three quickselect** for efficient O(n) median computation
- **Confidence as u32×1000** for exact integer sorting without floating-point comparison issues
- **NaN propagation** throughout — missing data is gracefully skipped
- **Degenerate input handling** — zero-length holes, empty properties, flat bounding boxes
