const truckCount = data.truckCount;

    const dy = py[i] - depotY;

That is, the route depot → t[b_i] → t[b_i+1] → ... → t[b_{i+1}-1] → depot might be better as depot → t[b_{i+1}-1] → t[b_{i+1}-2] → ... → t[b_i] → depot.

The cost of the route in the original direction:
depotDist(t[b_i]) + segment_length + depotDist(t[b_{i+1}-1])

The cost in the reversed direction:
depotDist(t[b_{i+1}-1]) + segment_length + depotDist(t[b_i])

These are the same! Because the segment length doesn't change, and we just swap the two depot connections. So the direction doesn't matter for a route extracted from the TSP tour.
