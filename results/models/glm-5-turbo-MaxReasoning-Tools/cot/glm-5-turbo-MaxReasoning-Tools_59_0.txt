Actually, wait. I realize the via-start approach for path computation is problematic because the drone doesn't actually go back to the start. The path via start is just a planning artifact. The actual drone position should be the last cell in the path.

But the cost of the via-start path is dist(current, start) + dist(start, target), which could be much larger than the actual shortest path. This means the drone wastes a lot of budget going back to start.

**Better approach: Use BFS from current position**

For each drone, when it needs to move to a target, compute BFS from its current position. This gives the optimal path.

To make this efficient:
- Limit BFS depth to remaining steps
- Skip if Manhattan distance to target > remaining steps

For moderate grids (1000x1000), BFS is 10^6 operations. If each drone visits 100 targets, that's 100 * 10^6 = 10^8 per drone. With 10 drones, 10^9 total. This is about 1-2 seconds, which might be acceptable.

But for 100 drones and 1000 targets each, it's 10^11, too slow.

**Compromise: BFS with depth limit**

If remaining steps = R, limit BFS to depth R. BFS explores O(min(gridSize, 4^R)) nodes. For R = 10 and gridSize = 10^6, BFS explores at most 4^10 ≈ 10^6 nodes. For R = 100, it's still at most 10^6 (grid size is the limit). So BFS is always O(gridSize), but with a tight depth limit, it might explore much fewer nodes.

Actually, BFS with visited set explores each cell at most once, so it's always O(gridSize). The depth limit just prunes early.

**Scalable approach: Guided pathfinding**

Instead of BFS, use A* with Manhattan heuristic. A* explores fewer nodes and is faster for finding paths between two specific cells.

A* with Manhattan heuristic is optimal for grid-based pathfinding and explores far fewer nodes than BFS when the path is short.

```javascript
function aStar(sx, sy, tx, ty, maxDist) {
    if (sx === tx && sy === ty) return [[sx, sy]];
    
    const h = (x, y) => Math.abs(x - tx) + Math.abs(y - ty);
    const g = Array.from({length: H}, () => new Int32Array(W).fill(Infinity));
    const f = Array.from({length: H}, () => new Float64Array(W).fill(Infinity));
    const parent = Array.from({length: H}, () => new Array(W).fill(null));
    const closed = Array.from({length: H}, () => new Uint8Array(W));
    
    g[sy][sx] = 0;
    f[sy][sx] = h(sx, sy);
    
    // Simple priority queue (binary heap would be better but this works)
    const open = [[f[sy][sx], sx, sy]];
    
    while (open.length > 0) {
        // Find min f
        let minIdx = 0;
        for (let i = 1; i < open.length; i++) {
            if (open[i][0] < open[minIdx][0]) minIdx = i;
        }
        const [fv, x, y] = open[minIdx];
        open.splice(minIdx, 1);
        
        if (closed[y][x]) continue;
        closed[y][x] = 1;
        
        if (x === tx && y === ty) {
            // Reconstruct
            const path = [];
            let cx = tx, cy = ty;
            while (cx !== sx || cy !== sy) {
                path.push([cx, cy]);
                const p = parent[cy][cx];
                cx = p[0]; cy = p[1];
            }
            path.push([sx, sy]);
            return path.reverse();
        }
        
        if (g[y][x] >= maxDist) continue;
        
        for (const [dx, dy] of [[0,-1],[0,1],[-1,0],[1,0]]) {
            const nx = x + dx, ny = y + dy;
            if (nx >= 0 && nx < W && ny >= 0 && ny < H && !closed[ny][nx] && prob[ny][nx] !== -1) {
                const ng = g[y][x] + 1;
                if (ng < g[ny][nx]) {
                    g[ny][nx] = ng;
                    f[ny][nx] = ng + h(nx, ny);
                    parent[ny][nx] = [x, y];
                    open.push([f[ny][nx], nx, ny]);
                }
            }
        }
    }
    
    return null; // Not reachable within maxDist
}
```
