```glsl
#version 450

/* ================================================================
 *  Gravitational N-Body Compute Shader
 *  -------------------------------
 *  Each body is stored as two consecutive vec4s in the buffer:
 *      [0] vec4(position.xyz, mass)
 *      [1] vec4(velocity.xyz, 0.0)
 *  So body i occupies indices i*2   (pos+mass) and i*2+1 (vel).
 *
 *  Integration: symplectic Euler (kick-drift)
 *      v_{n+1} = v_n + a(x_n) * dt
 *      x_{n+1} = x_n + v_{n+1} * dt
 *
 *  Dispatch: ceil(N / 256) workgroups in X.
 *  The host ping-pongs input/output buffers across timesteps.
 * ================================================================ */

layout(local_size_x = 256) in;

// ---- Storage buffers ---------------------------------------------------
layout(set = 0, binding = 0) readonly buffer InputBodies {
    vec4 inData[];        // length = N * 2
};

layout(set = 0, binding = 1) buffer OutputBodies {
    vec4 outData[];       // length = N * 2
};

// ---- Uniform parameters ------------------------------------------------
layout(set = 0, binding = 2) uniform Params {
    uvec4 counts;         // counts.x = N  (number of bodies)
    vec4  physics;        // physics.x = dt
                         // physics.y = softening
                         // physics.z = G
                         // physics.w = unused
};

// ---- Convenience macros ------------------------------------------------
#define BODY_COUNT     counts.x
#define DT             physics.x
#define SOFTENING      physics.y
#define GRAV_CONST     physics.z

#define POS_INDEX(i)   ((i) * 2)
#define VEL_INDEX(i)   ((i) * 2 + 1)

// ---- Main --------------------------------------------------------------
void main()
{
    uint i = gl_GlobalInvocationID.x;

    // Guard: this invocation has no work to do
    if (i >= BODY_COUNT)
        return;

    // ------------------------------------------------------------------
    // Read body i's current state
    // ------------------------------------------------------------------
    vec4  body_i_pos_mass = inData[POS_INDEX(i)];
    vec3  pos_i           = body_i_pos_mass.xyz;
    float mass_i          = body_i_pos_mass.w;   // not needed for accel,
                                                  // but kept for clarity

    vec3 vel_i = inData[VEL_INDEX(i)].xyz;

    // ------------------------------------------------------------------
    // Accumulate gravitational acceleration from every other body
    // ------------------------------------------------------------------
    vec3 accel = vec3(0.0);
    float softening2 = SOFTENING * SOFTENING;

    for (uint j = 0u; j < BODY_COUNT; ++j)
    {
        // Skip self-interaction
        if (j == i)
            continue;

        vec4  body_j_pos_mass = inData[POS_INDEX(j)];
        vec3  pos_j           = body_j_pos_mass.xyz;
        float mass_j          = body_j_pos_mass.w;

        // Vector from body i towards body j
        vec3 r_ij = pos_j - pos_i;

        // Softened inverse-distance cubed:
        //   1 / (|r|^2 + eps^2)^{3/2}
        float dist2    = dot(r_ij, r_ij) + softening2;
        float invDist3 = 1.0 / (dist2 * sqrt(dist2));

        // Newton's law of gravitation (vector form)
        accel += GRAV_CONST * mass_j * r_ij * invDist3;
    }

    // ------------------------------------------------------------------
    // Symplectic Euler integration (kick-drift)
    //   1. Kick  : update velocity with acceleration
    //   2. Drift : update position with the *new* velocity
    // ------------------------------------------------------------------
    vec3 vel_new = vel_i + accel * DT;
    vec3 pos_new = pos_i + vel_new * DT;

    // ------------------------------------------------------------------
    // Write updated state to the output buffer
    // ------------------------------------------------------------------
    outData[POS_INDEX(i)] = vec4(pos_new, mass_i);
    outData[VEL_INDEX(i)] = vec4(vel_new, 0.0);
}
```

## Walkthrough

### Data layout
Each body occupies **two consecutive `vec4` slots**:

| Index | Field | Components |
|-------|-------|------------|
| `i*2` | Position + mass | `(x, y, z, mass)` |
| `i*2+1` | Velocity | `(vx, vy, vz, 0)` |

The input is read-only; the output is written to a separate buffer. The host swaps (ping-pongs) them between dispatches.

### Acceleration kernel
The inner loop visits **every other body** `j ≠ i` and accumulates:

```
a_i += G · m_j · (r_j − r_i) / (|r_j − r_i|² + ε²)^{3/2}
```

The **softening²** (`ε²`) is added *inside* the `dot` before taking the square root and cube, which eliminates the 1/r² singularity when two bodies nearly overlap and keeps the shader numerically stable even for tight encounters.

### Integration
A **symplectic Euler** (kick-then-drift) scheme is used:

1. **Kick** — `v += a · Δt`  
2. **Drift** — `x += v_new · Δt`

This is the simplest first-order symplectic integrator; it conserves energy much better than the naive (drift-then-kick) Euler method over long runs, making it a practical choice for a single-pass-per-timestep GPU shader. The leapfrog (Störmer–Verlet) integrator would require two half-step passes, which doesn't fit the stated single-dispatch constraint.

### Dispatch sizing
The shader declares `local_size_x = 256`. The host dispatches:

```c
glDispatchCompute(ceil(N / 256.0), 1, 1);
```

The early `if (i >= BODY_COUNT) return;` guard ensures out-of-range invocations are harmless.

### Performance note
This is the naive **O(N²)** all-pairs kernel — each of the N workgroups reads the entire input buffer. For N ≤ 2048 with the given parameters this runs well within a single frame budget on modern GPUs. Scaling beyond that would motivate a Barnes-Hut or FMM approach, but those are significantly more complex to implement in GLSL.
