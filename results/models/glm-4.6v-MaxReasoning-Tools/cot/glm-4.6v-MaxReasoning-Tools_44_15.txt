I need to write a GLSL fragment shader that implements Gooch warm/cool shading. Let me break down what I need to do:

1. The shader will be for a unit sphere
2. I have inputs from the vertex shader: worldPos, normal, uv, tangent, color
3. I have a UBO with model, view, projection matrices, light position, camera position, and params
4. I need to output a vec4 color with alpha 1.0
5. The Gooch shading should use:
   - Cool color: (0.2, 0.2, 0.75)
   - Warm color: (0.7, 0.7, 0.4)
   - t = dot(N,L) * 0.5 + 0.5

The Gooch shading model is a non-photorealistic shading technique that uses warm and cool colors to simulate lighting. The basic idea is:
- When the surface is facing the light (dot(N,L) > 0), use warm colors
- When the surface is facing away from the light (dot(N,L) < 0), use cool colors
- The transition is controlled by t = dot(N,L) * 0.5 + 0.5, which maps the dot product from [-1,1] to [0,1]

Let me write the shader:

```glsl
#version 450

layout(location = 0) in vec3 worldPos;
layout(location = 1) in vec3 normal;
layout(location = 2) in vec2 uv;
layout(location = 3) in vec3 tangent;
layout(location = 4) in vec3 color;

layout(set = 0, binding = 0) uniform UBO {
    mat4 model;
    mat4 view;
    mat4 projection;
    vec4 lightPos;
    vec4 cameraPos;
    vec4 params;
} ubo;

layout(location = 0) out vec4 outColor;

void main() {
    // Normalize the normal (it might be interpolated and not unit length)
    vec3 N = normalize(normal);
    
    // Get light direction (from surface to light)
    vec3 L = normalize(ubo.lightPos.xyz - worldPos);
    
    // Calculate t as specified: dot(N,L) * 0.5 + 0.5
    float t = dot(N, L) * 0.5 + 0.5;
    
    // Define cool and warm colors
    vec3 coolColor = vec3(0.2, 0.2, 0.75);
    vec3 warmColor = vec3(0.7, 0.7, 0.4);
    
    // Mix between cool and warm colors based on t
    vec3 shadedColor = mix(coolColor, warmColor, t);
    
    // Set alpha to 1.0
    outColor = vec4(shadedColor, 1.0);
}
```
