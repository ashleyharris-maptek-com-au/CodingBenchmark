The user wants me to write a GLSL fragment shader that visualizes the surface normal as color. The requirements are:

1. The shader should map the normalized normal from [-1, 1] to [0, 1] range
2. The output should be RGBA with alpha = 1.0
3. The shader should use the provided inputs and uniform buffer
4. Must follow the specified GLSL requirements

Let me think about this:
- The normal is in world space and is already interpolated from the vertex shader
- I need to normalize it (though it should already be normalized, it's good practice)
- To map from [-1, 1] to [0, 1], I can use: (normal + 1.0) * 0.5
- The alpha should be 1.0

The shader should look like this:

```glsl
#version 450

layout(location = 0) in vec3 worldPos;
layout(location = 1) in vec3 normal;
layout(location = 2) in vec2 uv;
layout(location = 3) in vec3 tangent;
layout(location = 4) in vec3 color;

layout(set = 0, binding = 0) uniform UBO {
    mat4 model;
    mat4 view;
    mat4 projection;
    vec4 lightPos;
    vec4 cameraPos;
    vec4 params;
} ubo;

layout(location = 0) out vec4 outColor;

void main() {
    // Normalize the normal (though it should already be normalized)
    vec3 n = normalize(normal);
    
    // Map from [-1, 1] to [0, 1] range
    vec3 normalColor = (n + 1.0) * 0.5;
    
    // Set alpha to 1.0
    outColor = vec4(normalColor, 1.0);
}
```

This should work correctly. The normal is normalized, then mapped to the [0, 1] range, and alpha is set to 1.0.
