```glsl
#version 450

layout(local_size_x = 256) in;

// Each body is stored as two consecutive vec4s:
//   vec4(position.xyz, mass)
//   vec4(velocity.xyz, 0.0)
// So body i starts at index i*2 in the buffer.

layout(set = 0, binding = 0) readonly buffer InputBodies {
    vec4 inData[];   // length = N * 2
};

layout(set = 0, binding = 1) buffer OutputBodies {
    vec4 outData[];  // length = N * 2
};

layout(set = 0, binding = 2) uniform Params {
    uvec4 counts;    // counts.x = N (number of bodies)
    vec4 physics;    // physics.x = dt, physics.y = softening, physics.z = G
};

void main()
{
    uint idx = gl_GlobalInvocationID.x;
    uint N = counts.x;

    // Guard: out-of-range invocations do nothing
    if (idx >= N) return;

    // Read body i's position, mass, and velocity
    vec4 posMass_i = inData[idx * 2u + 0u];
    vec4 vel_i     = inData[idx * 2u + 1u];

    vec3 pos_i = posMass_i.xyz;
    float mass_i = posMass_i.w;  // not needed for acceleration, but preserved

    float dt         = physics.x;
    float softening  = physics.y;
    float G          = physics.z;

    // Compute gravitational acceleration on body i from all other bodies
    vec3 acc = vec3(0.0);

    float soft2 = softening * softening;

    for (uint j = 0u; j < N; j++)
    {
        // Skip self-interaction
        if (j == idx) continue;

        vec4 posMass_j = inData[j * 2u + 0u];
        vec3 pos_j = posMass_j.xyz;
        float mass_j = posMass_j.w;

        vec3 r = pos_j - pos_i;
        float dist2 = dot(r, r) + soft2;
        float inv_dist = 1.0 / sqrt(dist2);
        float inv_dist3 = inv_dist * inv_dist * inv_dist;

        acc += G * mass_j * r * inv_dist3;
    }

    // Advance velocity and position (symplectic Euler / kick-drift)
    vec3 newVel = vel_i.xyz + acc * dt;
    vec3 newPos = pos_i + newVel * dt;

    // Write both vec4s to output
    outData[idx * 2u + 0u] = vec4(newPos, mass_i);
    outData[idx * 2u + 1u] = vec4(newVel, 0.0);
}
```

## Walkthrough

**Data layout.** Body *i* occupies slots `inData[2i]` (position + mass) and `inData[2i+1]` (velocity). The shader reads both for its assigned body and writes the same pair to `outData`.

**Acceleration loop.** For every other body *j*, we compute the displacement **r** = pos\_j − pos\_i. The softened inverse-cube factor is:

$$\frac{1}{(\|\mathbf{r}\|^2 + \epsilon^2)^{3/2}}$$

which we compute as `(inv_dist)³` where `inv_dist = 1/√(r·r + ε²)`. This avoids a separate `pow` call and is numerically stable. The accumulated acceleration is:

$$\mathbf{a}_i = \sum_{j \neq i} \frac{G\, m_j\, \mathbf{r}_{ij}}{(\|\mathbf{r}_{ij}\|^2 + \epsilon^2)^{3/2}}$$

**Integration (kick-drift Euler).** This is the symplectic Euler / leapfrog-like variant:
1. **Kick:** v ← v + a·Δt  
2. **Drift:** x ← x + v·Δt  (using the *updated* velocity)

Using the updated velocity in the drift step gives better energy conservation than the naive forward Euler (where you'd use the old velocity), and is the standard "kick-drift" form of leapfrog when ping-ponging with a single compute dispatch per timestep.

**Softening.** The softening parameter ε prevents division-by-zero singularities when two bodies are very close. With `soft2 = ε²` precomputed outside the loop, we add it to the squared distance before taking the square root.

**Boundary guard.** If `gl_GlobalInvocationID.x ≥ N`, the invocation returns immediately, avoiding out-of-bounds reads/writes when N is not a multiple of 256.

**Output preservation.** The mass component is carried through unchanged in the position+mass vec4, and the velocity vec4's w component is kept at 0.0 as specified.
