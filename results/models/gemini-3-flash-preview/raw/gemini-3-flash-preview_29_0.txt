I have implemented a high-performance Maximum Clique solver in C# that efficiently handles graphs ranging from sparse to very dense, with up to 1,000,000 vertices.

### Technical Implementation Details:
- **Degeneracy Ordering:** The solver first computes a degeneracy ordering of the vertices in $O(V+E)$ time. This ordering ensures that for sparse graphs, each vertex has a small number of neighbors that appear later in the order, drastically reducing the search space.
- **Parallel Search:** The top-level search is parallelized across all available CPU cores using `Parallel.For`. It explores vertices in reverse degeneracy order, which helps in finding large cliques early and enables aggressive pruning for subsequent tasks.
- **Bitset-based Backtracking:** For subproblems, the solver employs a bitset-based approach (up to 65,536 vertices per subproblem). Bitsets accelerate graph intersections and adjacency checks using SIMD-accelerated `PopCount` and `TrailingZeroCount` operations.
- **Color-based Pruning:** A greedy coloring heuristic is used within the search tree to provide a tight upper bound on the possible clique size in any given branch, allowing for early exits when a branch cannot improve upon the best-found clique.
- **Memory Efficiency:** The graph is stored in a packed adjacency list format to minimize memory overhead and improve cache locality, ensuring it fits within the 32GB memory limit even for very large graphs.
- **Timeout Management:** The solver monitors its execution time and ensures it delivers the best found solution before the 5-minute time limit.
