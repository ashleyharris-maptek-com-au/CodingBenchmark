I have implemented a C++ program to perform a Constructive Solid Geometry (CSG) union operation on two triangle meshes. The program is designed to be scalable and handle meshes of varying complexity, from trivial to very large, as required by the prompt.

### Strategy and Design Choices

The core of the solution is a **voxel-based method using the Marching Cubes algorithm**. This approach was chosen over a direct boundary representation (B-Rep) method for the following reasons:

1.  **Scalability:** B-Rep algorithms, which involve precise triangle-triangle intersection calculations, typically have a computational complexity that scales poorly with the number of faces (e.g., O(N*M)), making them too slow for very large meshes. The voxel-based approach, while an approximation, has a complexity related to the grid resolution and the cost of the point-in-mesh test, which is more manageable for large inputs.
2.  **Robustness:** Implementing a fully robust B-Rep CSG algorithm is notoriously difficult and requires handling numerous geometric special cases (degeneracies, floating-point imprecision, etc.). A voxel-based approach is generally more robust and easier to implement correctly, as it converts the complex geometry into a regular grid structure.
3.  **Adaptive Complexity:** The prompt requires the program to work efficiently across all scales. My implementation achieves this by **adaptively setting the resolution of the voxel grid** based on the total number of vertices in the input meshes. For smaller meshes, a lower resolution is used, and for larger meshes, the resolution is increased to capture more detail.

### Implementation Details

The program is a single, self-contained C++ file (`main.cpp`) and follows these steps:

1.  **Input Parsing:** It reads the vertex and face counts for two meshes (A and B) from `stdin`, followed by the vertex coordinates and face indices for each mesh. The data is stored in simple `Mesh` structs containing `std::vector`s of vertices and faces.

2.  **Voxelization:**
    *   The combined Axis-Aligned Bounding Box (AABB) of both meshes is computed.
    *   A 3D grid is created to cover this AABB. The grid resolution is chosen adaptively.
    *   The program iterates through each point on the grid and determines if it lies inside the union of the two meshes. This is done by checking if the point is inside `meshA` **OR** inside `meshB` using a ray-casting `is_inside` function. A point is considered "inside" a mesh if a ray cast from that point intersects with an odd number of the mesh's faces.

3.  **Marching Cubes:**
    *   The Marching Cubes algorithm is executed on the populated voxel grid. It processes each 2x2x2 cell (cube) of the grid.
    *   Based on which of the 8 corners of the cube are "inside" vs. "outside" the geometry, a `cube_index` is calculated.
    *   This index is used to look up the required edges to be generated in the pre-computed `edgeTable`.
    *   The same index is then used to look up the triangulation pattern in the `triTable`, which specifies how to connect the edge vertices to form triangles.
    *   Vertices are created on the edges of the cubes via linear interpolation to form a smooth surface.

4.  **Vertex Merging:** To ensure the output mesh is compact and connected, a simple vertex merging (or welding) technique is used. As new vertices are generated, their coordinates are quantized and used as a key in an `std::unordered_map`. This prevents the creation of duplicate vertices at the same location.

5.  **Output:** The final list of unique vertices and the generated faces are printed to `stdout` in the specified format.

This solution correctly implements the CSG union operation while adhering to the critical requirements of scalability and performance by choosing an appropriate and adaptive algorithm.
