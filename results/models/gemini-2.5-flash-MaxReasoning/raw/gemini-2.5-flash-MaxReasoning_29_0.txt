The `Program.cs` file has been created and thoroughly optimized for the Maximum Clique problem. The solution implements the Bron-Kerbosch algorithm with pivoting, which is generally one of the most efficient exact algorithms for this NP-hard problem.

Here's a summary of the key features and optimizations:

1.  **Graph Representation:** An adjacency list `HashSet<int>[] adj` is used to store the graph. Using `HashSet<int>` for each vertex's neighbors allows for efficient (average O(1)) lookup of neighbors, which is crucial for the set intersection operations within the Bron-Kerbosch algorithm.
2.  **Bron-Kerbosch (BK2) Algorithm with Pivoting:** The core of the solution is a recursive implementation of the Bron-Kerbosch algorithm. The BK2 variant includes a pivot selection strategy that significantly prunes the search space.
3.  **Optimized Pivot Selection:** The pivot (a vertex chosen from `P U X`) is selected to maximize the number of its neighbors present in `P`. This heuristic helps to reduce the size of the `P` set in subsequent recursive calls, leading to fewer branches in the search tree. The implementation avoids `OrderByDescending` for pivot selection, opting for a manual, more efficient iteration.
4.  **Efficient Set Operations:** `HashSet<int>` is consistently used for `P` (potential candidates) and `X` (excluded vertices) sets. This ensures that set operations like `UnionWith`, `IntersectWith`, `Contains`, `Remove`, and `Add` perform efficiently.
5.  **Input/Output:** The program reads graph data from standard input and writes the maximum clique size and vertices to standard output, adhering to the specified format.

While the Bron-Kerbosch algorithm is exponential in the worst case, these optimizations aim to make it as performant as possible for various graph complexities, including dense instances. The requirement to handle up to 1 million vertices within 5 minutes for an NP-hard problem remains challenging; however, this solution represents a highly optimized exact approach within the given constraints.
