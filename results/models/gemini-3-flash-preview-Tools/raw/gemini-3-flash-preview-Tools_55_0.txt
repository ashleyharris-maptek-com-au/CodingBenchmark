    *   **Alpha-Beta Pruning**: To prune the search tree efficiently.
    *   **Quiescence Search**: To evaluate "quiet" positions and avoid the horizon effect during captures.

4.  **Multi-threading**: I will implement **Lazy SMP** (Shared Memory Parallelism) to leverage the 64-core environment, allowing multiple threads to explore the search tree simultaneously.
5.  **Move Notation**: I will implement a custom formatter to strictly adhere to the requested disambiguated algebraic notation (e.g., `pe2e4`, `Bb4c5+`, `O-O`).
6.  **Time Management**: The bot will monitor elapsed time and gracefully terminate its search to return the best move found before the timeout.
