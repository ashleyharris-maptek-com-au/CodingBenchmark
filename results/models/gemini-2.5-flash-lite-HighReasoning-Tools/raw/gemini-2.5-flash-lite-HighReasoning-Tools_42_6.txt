The SPIR-V assembly geometry shader has been implemented in `geometry_shader.spvasm`. It defines the required `OpCapability Geometry` and `OpMemoryModel`, correctly sets up the `OpEntryPoint` with the specified interface variables, and defines the `OpExecutionMode` for input/output primitives and vertex counts. The main function follows the passthrough logic: loading the position and color for each of the three input vertices, storing them in the output variables, calling `OpEmitVertex`, and finally calling `OpEndPrimitive`.
