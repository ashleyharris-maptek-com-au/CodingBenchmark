# Plan: 2D Lattice Boltzmann (D2Q9) Compute Shader in SPIR-V Assembly

## Objective
Create a functional 2D Lattice Boltzmann (D2Q9) compute shader using raw SPIR-V assembly.

## Background & Motivation
The Lattice Boltzmann Method (LBM) is a powerful technique for simulating fluid dynamics. Implementing it as a compute shader allows for massive parallelism on GPUs. SPIR-V assembly is the lowest-level representation, providing maximum control but requiring precise instruction management.

## Scope & Impact
- **Model:** D2Q9 (2D, 9 velocities).
- **Steps:** Collision (BGK) and Streaming.
- **Output:** A valid SPIR-V assembly file (`.spvasm`).

## Proposed Solution
1.  **Data Structure:** Use a structured buffer to store the 9 population values for each grid cell.
2.  **Kernel Logic:**
    -   Read populations from the input buffer.
    -   Calculate macroscopic density and velocity.
    -   Perform collision (BGK approximation).
    -   Perform streaming (move populations to neighboring cells).
    -   Write updated populations to the output buffer.
3.  **SPIR-V Structure:**
    -   `OpCapability Shader`
    -   `OpMemoryModel Logical GLSL450`
    -   `OpEntryPoint GLCompute`
    -   `OpExecutionMode` (local size)
    -   `OpDecorate` (buffer bindings)

## Implementation Steps
1.  **Define SPIR-V Header:** Set up capabilities, memory model, and entry point.
2.  **Define Types and Constants:** Define float types, vector types, and buffer structures.
3.  **Define Buffers:** Set up `StorageBuffer` for input and output populations.
4.  **Implement Kernel:**
    -   Get global invocation ID.
    -   Load populations.
    -   Implement collision logic (BGK).
    -   Implement streaming logic.
    -   Store results.
