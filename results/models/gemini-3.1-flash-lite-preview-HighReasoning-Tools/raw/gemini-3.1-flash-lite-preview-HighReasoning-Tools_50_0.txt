# Design Document: 3D Boid Flocking Compute Shader

## Objective
Implement a 3D boid flocking simulation using a GLSL compute shader.

## Data Structures
The simulation will use two Shader Storage Buffer Objects (SSBOs) to store boid data, allowing for ping-pong buffering if necessary, or direct updates if the simulation allows.

```glsl
struct Boid {
    vec4 position; // xyz: position, w: unused/radius
    vec4 velocity; // xyz: velocity, w: speed
};

layout(std430, binding = 0) buffer BoidBuffer {
    Boid boids[];
};
```

## Flocking Rules
The compute shader will implement the three classic Reynolds rules:
1.  **Separation:** Steer to avoid crowding local flockmates.
2.  **Alignment:** Steer towards the average heading of local flockmates.
3.  **Cohesion:** Steer to move towards the average position of local flockmates.

## Implementation Steps
1.  **Setup:** Define the compute shader layout and SSBO bindings.
2.  **Compute Logic:**
    *   Iterate through all boids to calculate the influence of neighbors based on a defined radius.
    *   Apply separation, alignment, and cohesion forces.
    *   Update velocity based on forces and speed limits.
    *   Update position based on velocity.
    *   Apply boundary conditions (e.g., wrap-around or bounce).
3.  **Integration:** Use Euler integration for position updates.

## Verification
*   Verify shader compilation.
*   Validate that boids maintain flocking behavior within the defined 3D space.
