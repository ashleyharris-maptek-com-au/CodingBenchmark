### Proposed Strategy: Heuristic-Based Greedy Search
Instead of a computationally expensive look-ahead (Minimax), we will implement a robust evaluation function to score all possible moves (rotations and positions) for each incoming piece.

**Evaluation Function Metrics:**
*   **Aggregate Height:** The sum of heights of all columns. Minimizing this keeps the board low.
*   **Complete Lines:** The number of full lines that would be cleared. Maximizing this is the primary goal.
*   **Holes:** The number of empty spaces trapped beneath filled cells. These are costly to clear; minimizing them is critical.
*   **Bumpiness:** The sum of absolute differences in height between adjacent columns. Minimizing this promotes a flat board surface.

**Pros:**
*   **Speed:** Extremely efficient; will easily meet the 5-second response time.
*   **Robustness:** Well-suited for long game sequences.
*   **Simplicity:** Highly maintainable C# implementation.

**Cons:**
*   **Myopia:** Does not plan beyond the current piece, though this can be mitigated by high-quality weighting of the heuristics.

**Implementation Plan:**
1.  **Board Data Structure:** A 2D array or bitboard for efficient state representation.
2.  **Move Generator:** A generator for all possible rotations and column positions of the current piece.
3.  **Evaluator:** A weighted scoring function based on the metrics above.
4.  **Main Loop:** Efficient I/O handling to parse stdin and output moves to stdout.
