### SPIR-V Assembly (Passthrough Geometry Shader)

```assembly
; SPIR-V Assembly - Passthrough Geometry Shader
; Assembled with: spirv-as
;
; Header
OpCapability Geometry
OpMemoryModel Logical GLSL450

; Entry Point & Interface
OpEntryPoint Geometry %main "main" %in_pos %out_pos

; Execution Modes
OpExecutionMode %main InputPoints
OpExecutionMode %main OutputPoints
OpExecutionMode %main OutputVertices 1
OpExecutionMode %main Invocations 1

; Types & Variables
%float = OpTypeFloat 32
%v4float = OpTypeVector %float 4
%ptr_input_v4float = OpTypePointer Input %v4float
%ptr_output_v4float = OpTypePointer Output %v4float

; Interface Variables
%in_pos = OpVariable %ptr_input_v4float Input
%out_pos = OpVariable %ptr_output_v4float Output

; Main Function
%void = OpTypeVoid
%func = OpTypeFunction %void
%main = OpFunction %void None %func
%label = OpLabel

; Passthrough Logic
%val = OpLoad %v4float %in_pos
OpStore %out_pos %val
OpEmitVertex
OpEndPrimitive

OpReturn
OpFunctionEnd
```

### Key Components

*   **`OpCapability Geometry`**: Enables the geometry shader stage.
*   **`OpExecutionMode`**: Defines how the geometry shader processes input primitives (`InputPoints`), what it outputs (`OutputPoints`), how many vertices are produced per invocation (`OutputVertices 1`), and the number of invocations per primitive (`Invocations 1`).
*   **`OpEmitVertex` / `OpEndPrimitive`**: Essential geometry shader instructions to emit a vertex to the primitive assembly and terminate the current primitive strip.
