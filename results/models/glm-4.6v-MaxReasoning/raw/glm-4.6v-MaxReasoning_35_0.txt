
**Solution Explanation**

The Minimum Dominating Set (MDS) problem asks for a smallest set of vertices `S` such that every vertex of the graph is either in `S` or adjacent to a vertex in `S`.  
The problem is NP‑hard, so we use a fast greedy heuristic that works well in practice and easily satisfies the 5‑minute time limit for large inputs.

### Greedy Heuristic
1. **Initial coverage** – each vertex can cover itself and all its neighbours.  
   For vertex `v` let `uncovered(v)` be the number of vertices that are still uncovered and are either `v` itself or a neighbour of `v`.
2. **Select the best vertex** – repeatedly pick the vertex with the largest `uncovered(v)` (i.e. the vertex that covers the most still‑uncovered vertices).  
   After selecting a vertex `v` we mark `v` and all its neighbours as covered.
3. **Update counts lazily** – when a vertex `u` becomes covered we must decrease `uncovered(w)` for every neighbour `w` of `u` (because `u` is no longer an uncovered neighbour).  
   We store the current `uncovered` count for each vertex in a max‑heap (implemented with `BinaryHeap`).  
   When a vertex is popped from the heap we first check whether it is still uncovered; if not we skip it (lazy update).

The algorithm stops when every vertex is covered. The resulting set is a dominating set; its size is a heuristic approximation of the optimum.

### Complexity
* Let `n` be the number of vertices and `m` the number of edges.  
* Building the adjacency list: `O(n + m)`.  
* Each time a vertex is added to the dominating set we iterate over its neighbours (`deg(v)`) and for each neighbour we iterate over its neighbours (`deg(u)`).  
  In the worst case this is `O( Σ_{v∈S} deg(v)·deg(u) )`.  
  For sparse graphs (average degree ≈ constant) this is linear in `m`.  
* The heap operations are `O(log n)` per update, but the total number of updates is bounded by `O(m)`.  

Overall the algorithm runs in `O(m·α)` where `α` is a small constant (typically ≤ 10) for practical graphs, easily fitting into the 5‑minute limit on the given hardware.

### Implementation Details
* Input is read from `stdin`.  
* The graph is stored as an adjacency list (`Vec<Vec<usize>>`).  
* `covered` – a boolean vector marking already dominated vertices.  
* `uncovered_cnt` – the current number of uncovered vertices that a vertex can dominate.  
* A max‑heap (`BinaryHeap`) stores pairs `(-uncovered_cnt, vertex)` (negative to turn `BinaryHeap` into a max‑heap).  
* After each selection we update `covered` and `uncovered_cnt` for affected vertices and push the updated counts into the heap.

The program prints the size of the dominating set and the vertices (in any order).

```rust
use std::io::{self, BufRead};
use std::collections::BinaryHeap;

fn main() {
    // Read the first line: number of vertices (n) and edges (m)
    let stdin = io::stdin();
    let mut lines = stdin.lock().lines();
    let first_line = lines.next().expect("missing first line");
    let first_line = first_line.expect("failed to read first line");
    let mut parts = first_line.split_whitespace();
    let n: usize = parts.next().expect("missing n").parse().expect("invalid n");
    let m: usize = parts.next().expect("missing m").parse().expect("invalid m");

    // Build adjacency list
    let mut adj = vec![Vec::new(); n];
    for _ in 0..m {
        let line = lines.next().expect("missing edge line").expect("failed to read edge line");
        let mut parts = line.split_whitespace();
        let u: usize = parts.next().expect("missing u").parse().expect("invalid u");
        let v: usize = parts.next().expect("missing v").parse().expect("invalid v");
        adj[u].push(v);
        adj[v].push(u);
    }

    // Compute initial uncovered count: degree + 1 (itself)
    let mut degree = vec![0; n];
    for i in 0..n {
        degree[i] = adj[i].len();
    }
    let mut uncovered_cnt = degree.clone();
    for i in 0..n {
        uncovered_cnt[i] += 1;
    }

    // Data structures for the greedy algorithm
    let mut covered = vec![false; n];
    let mut heap = BinaryHeap::new();
    for i in 0..n {
        heap.push((-uncovered_cnt[i] as isize, i));
    }

    let mut dominating_set = Vec::new();
    let mut covered_count = 0;

    // Greedy selection
    while covered_count < n {
        // Pop the vertex with the largest uncovered count
        let (cnt, v) = heap.pop().expect("heap empty before covering all vertices");
        if covered[v] {
            // Stale entry – skip
            continue;
        }

        // Add to dominating set and mark as covered
        dominating_set.push(v);
        covered[v] = true;
        covered_count += 1;

        // Cover all neighbours of v
        for &u in &adj[v] {
            if !covered[u] {
                covered[u] = true;
                covered_count += 1;

                // For each neighbour w of u, decrease its uncovered count
                for &w in &adj[u] {
                    if !covered[w] {
                        uncovered_cnt[w] -= 1;
                        // Push updated count (lazy update)
                        heap.push((-uncovered_cnt[w] as isize, w));
                    }
                }
            }
        }
    }

    // Output the result
    println!("{}", dominating_set.len());
    let mut out = String::new();
    for &v in &dominating_set {
        out.push_str(&v.to_string());
        out.push(' ');
    }
    // Remove trailing space if any
    if !out.is_empty() {
        out.pop();
    }
    println!("{}", out);
}
```

The program reads the graph, runs the greedy heuristic, and prints a dominating set. It is fully self‑contained, uses only the Rust standard library, and meets all the specified requirements.
