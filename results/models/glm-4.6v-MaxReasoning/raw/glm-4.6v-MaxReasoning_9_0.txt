
To solve the 1D Cutting Stock Problem efficiently, we use a **First-Fit Decreasing (FFD)** heuristic. This approach sorts the cuts in descending order and places each cut into the first stock that can accommodate it, ensuring a feasible solution with minimal stocks. The algorithm leverages a sorted data structure to quickly locate the appropriate stock for each cut, achieving an efficient \(O(N \log N)\) time complexity.

### Approach
1. **Input Reading**: Read the number of cuts \(N\) and the stock length.
2. **Sorting**: Sort the cuts in descending order while preserving their original indices.
3. **Bin Management**: Use a `SortedSet` to maintain stocks (bins) sorted by their remaining capacity. This allows efficient lookup of the first bin that can fit a cut.
4. **Assignment**: For each cut, find the first bin with sufficient remaining capacity. If found, update the bin's remaining capacity; otherwise, create a new bin.
5. **Output**: Group cuts by their assigned bin and print the results.

### Solution Code
```csharp
using System;
using System.Collections.Generic;
using System.Linq;

public class CuttingStockSolver
{
    public static void Main()
    {
        var input = Console.ReadLine().Split();
        int N = int.Parse(input[0]);
        int stockLength = int.Parse(input[1]);
        var cuts = Console.ReadLine().Split().Select(int.Parse).ToArray();

        // Pair each cut with its original index and sort descending by length
        var indexedCuts = cuts.Select((len, idx) => (len, idx)).OrderByDescending(x => x.len).ToArray();

        // Array to store the bin index for each cut
        int[] binAssignment = new int[N];
        // SortedSet to manage bins by remaining capacity
        var bins = new SortedSet<Bin>();
        int binId = 0;

        foreach (var (len, idx) in indexedCuts)
        {
            // Find the first bin with remaining capacity >= len
            var view = bins.GetViewBetween(new Bin(len, 0), new Bin(int.MaxValue, 0));
            if (view.Count > 0)
            {
                var bin = view.First();
                bins.Remove(bin);
                // Update the bin's remaining capacity
                bin.Remaining -= len;
                bins.Add(bin);
                binAssignment[idx] = bin.Id;
            }
            else
            {
                // Create a new bin
                var newBin = new Bin(stockLength - len, binId);
                bins.Add(newBin);
                binAssignment[idx] = binId;
                binId++;
            }
        }

        // Group cuts by bin index
        var binGroups = new Dictionary<int, List<int>>();
        for (int i = 0; i < N; i++)
        {
            int binIdx = binAssignment[i];
            if (!binGroups.ContainsKey(binIdx))
            {
                binGroups[binIdx] = new List<int>();
            }
            binGroups[binIdx].Add(i);
        }

        // Output the results
        Console.WriteLine(binGroups.Count);
        foreach (var binIdx in binGroups.Keys.OrderBy(x => x))
        {
            var cutsInBin = binGroups[binIdx];
            Console.WriteLine(string.Join(" ", cutsInBin));
        }
    }

    // Bin class to store remaining capacity and a unique ID
    private class Bin : IComparable<Bin>
    {
        public int Remaining { get; set; }
        public int Id { get; }

        public Bin(int remaining, int id)
        {
            Remaining = remaining;
            Id = id;
        }

        public int CompareTo(Bin other)
        {
            int cmp = Remaining.CompareTo(other.Remaining);
            if (cmp != 0) return cmp;
            return Id.CompareTo(other.Id);
        }
    }
}
```

### Explanation
1. **Input Handling**: The input is read and parsed into the number of cuts \(N\) and the stock length.
2. **Sorting Cuts**: Cuts are paired with their original indices and sorted in descending order to prioritize larger cuts first, which helps in minimizing the number of stocks.
3. **Bin Management**: A `SortedSet` of `Bin` objects is used, where each `Bin` tracks its remaining capacity and a unique ID. The set is sorted by remaining capacity to enable efficient lookup.
4. **Cut Assignment**: For each cut, the algorithm checks the `SortedSet` for the first bin with sufficient remaining capacity. If found, the bin's capacity is updated; otherwise, a new bin is created.
5. **Output Preparation**: Cuts are grouped by their assigned bin index, and the results are printed in the required format, showing the number of stocks and the cuts assigned to each stock.

This approach efficiently handles large input sizes by leveraging sorting and a balanced binary search tree, ensuring optimal performance for millions of cuts.
