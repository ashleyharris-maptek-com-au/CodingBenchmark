
Here is a complete Node.js program that solves the scheduling problem using a heuristic approach. The program reads the input JSON, computes a topological order respecting dependencies, then schedules tasks using a priority rule (modified due date) to minimize weighted lateness and makespan. It outputs a schedule with start times and machine assignments.

```javascript
const fs = require('fs');
const input = fs.readFileSync(0, 'utf8');
const data = JSON.parse(input);

const { machineCount, tasks } = data;

// Map task IDs to indices for easier access
const taskMap = new Map();
tasks.forEach((task, idx) => {
  taskMap.set(task.id, idx);
});

const n = tasks.length;
const adj = new Array(n).fill().map(() => []);
const inDegree = new Array(n).fill(0);

// Build adjacency list and compute in-degrees
tasks.forEach((task, idx) => {
  task.deps.forEach(depId => {
    const depIdx = taskMap.get(depId);
    adj[depIdx].push(idx);
    inDegree[idx]++;
  });
});

// Compute earliest start times (ES) using forward pass
const ES = new Array(n).fill(0);
const queue = [];
for (let i = 0; i < n; i++) {
  if (inDegree[i] === 0) {
    queue.push(i);
  }
}

while (queue.length > 0) {
  const u = queue.shift();
  tasks[u].deps.forEach(depId => {
    const v = taskMap.get(depId);
    if (ES[v] < ES[u] + tasks[u].duration) {
      ES[v] = ES[u] + tasks[u].duration;
    }
  });
  adj[u].forEach(v => {
    inDegree[v]--;
    if (inDegree[v] === 0) {
      queue.push(v);
    }
  });
}

// Compute modified due dates (MDD) for priority
const MDD = new Array(n);
for (let i = 0; i < n; i++) {
  MDD[i] = Math.max(tasks[i].deadline, ES[i] + tasks[i].duration);
}

// Topological sort with priority based on MDD
const topoOrder = [];
const inDegreeCopy = [...inDegree];
const priorityQueue = [];

for (let i = 0; i < n; i++) {
  if (inDegreeCopy[i] === 0) {
    priorityQueue.push({ idx: i, mdd: MDD[i] });
  }
}

priorityQueue.sort((a, b) => a.mdd - b.mdd);

while (priorityQueue.length > 0) {
  const { idx } = priorityQueue.shift();
  topoOrder.push(idx);
  adj[idx].forEach(v => {
    inDegreeCopy[v]--;
    if (inDegreeCopy[v] === 0) {
      priorityQueue.push({ idx: v, mdd: MDD[v] });
    }
  });
  priorityQueue.sort((a, b) => a.mdd - b.mdd);
}

// Initialize machine finish times
const machineFinish = new Array(machineCount).fill(0);
const schedule = [];

// Schedule tasks in topological order
for (const idx of topoOrder) {
  const task = tasks[idx];
  const earliestStart = ES[idx];
  
  // Find machine with earliest finish time after scheduling this task
  let bestMachine = 0;
  let bestFinish = Infinity;
  
  for (let m = 0; m < machineCount; m++) {
    const start = Math.max(machineFinish[m], earliestStart);
    const finish = start + task.duration;
    if (finish < bestFinish) {
      bestFinish = finish;
      bestMachine = m;
    }
  }
  
  const start = Math.max(machineFinish[bestMachine], earliestStart);
  machineFinish[bestMachine] = start + task.duration;
  
  schedule.push({
    id: task.id,
    start: start,
    machine: bestMachine
  });
}

// Output the schedule
const output = { schedule };
console.log(JSON.stringify(output));
```

### How It Works:
1. **Input Parsing**: Reads the JSON input and maps task IDs to array indices.
2. **Dependency Graph**: Builds an adjacency list and computes in-degrees for topological sorting.
3. **Earliest Start Times (ES)**: Performs a forward pass to compute the earliest possible start time for each task.
4. **Modified Due Date (MDD)**: Calculates MDD = max(deadline, ES + duration) to prioritize urgent tasks.
5. **Topological Sorting**: Sorts tasks by MDD to respect dependencies while prioritizing critical tasks.
6. **List Scheduling**: Assigns each task to the machine that becomes available the earliest, ensuring dependencies are met.
7. **Output**: Returns the schedule with start times and machine assignments.

This heuristic balances load across machines and prioritizes tasks with tighter deadlines, aiming to minimize weighted lateness and makespan. The algorithm scales efficiently for large instances using O(n + m) time for topological sorting and O(n * machineCount) for scheduling.</think>
