
**Solution Explanation**

The task is to plan paths for multiple drones that maximize the total unique probability covered on a grid.
Each drone starts at a given cell and can move up/down/left/right or stay in place.
Obstacles (`-1`) must be avoided and the drones must stay inside the grid.
The score is the number of *unique* probability cells covered, compared to a greedy reference.

The reference (greedy) algorithm simply moves each drone to the highest‑probability adjacent cell
that is not an obstacle and not already covered by any drone.  
Our solution improves on this by looking ahead: at every step each drone chooses the best
target cell that can be reached within its remaining steps, using a breadth‑first search (BFS)
to compute the shortest path to that target. This ensures that high‑value cells that are
farther away are still considered, not just the immediate neighbours.

The algorithm works as follows:

1. **Pre‑processing**  
   - Read the input JSON.
   - Create a set `covered` that tracks which cells have already been covered.
   - Initialise each drone's path with its start position.

2. **Iterative planning**  
   For each step (from `0` to `maxStepsPerDrone - 1`):
   - For every drone that still has steps left:
     - Run a BFS from the drone's current position, limited to the remaining steps.
     - Collect all reachable cells that contain positive probability and are not yet covered.
     - Choose the reachable cell with the highest probability (break ties by Manhattan distance
       to the drone's current position to favour closer cells).
     - If a target is found, compute the shortest path to it (the BFS already provides the parent
       pointers) and append the first move of that path to the drone's path.
     - Mark the target cell as covered.
   - If no reachable target exists, the drone stays in place.

3. **Output**  
   - Convert the list of paths to the required JSON format and print it.

The BFS is bounded by the remaining steps, so its cost is `O(k * (n*m))` where `k` is the number
of steps left and `n*m` is the grid size. In practice this is fast because each drone only
performs a BFS when it has steps left, and the BFS stops as soon as the depth exceeds the
remaining steps. The overall complexity is linear in the total number of steps and grid cells,
making it suitable for large scenarios.

**Why this is better than the greedy reference**  
The greedy reference only looks at immediate neighbours, potentially missing high‑value cells
that are a few steps away. By using BFS to consider all reachable cells within the remaining
steps, our algorithm can plan ahead and reach those valuable cells, increasing the total
unique probability covered.

**Complexity Analysis**

- Let `S` be the total number of steps across all drones (`S = drones * maxStepsPerDrone`).
- Let `G = n * m` be the number of grid cells.
- Each BFS runs in `O(k * G)` where `k` is the remaining steps (at most `maxStepsPerDrone`).
- In the worst case, each drone performs `maxStepsPerDrone` BFS, leading to `O(S * G)`.
- In practice, the BFS stops early when the depth exceeds the remaining steps, and many
  drones will stop early when they run out of steps, so the average runtime is much lower.
- Memory usage is `O(G)` for the grid and `O(G)` for the BFS queue, which is acceptable for
  large grids (e.g., 1000x1000).

**Reference Implementation (Node.js)**

```javascript
const fs = require('fs');
const input = fs.readFileSync(0, 'utf8');
const data = JSON.parse(input);

// Grid dimensions
const rows = data.probability.length;
const cols = data.probability[0].length;

// Drones: each has id, start x, start y, and remaining steps
const drones = data.drones.map(d => ({
  id: d.id,
  x: d.x,
  y: d.y,
  stepsLeft: data.maxStepsPerDrone,
  path: [[d.x, d.y]] // start position
}));

// Set of covered cells (stored as string "x,y")
const covered = new Set();

// Helper: check if a cell is inside the grid and not an obstacle
function isValid(x, y) {
  return x >= 0 && x < rows && y >= 0 && y < cols && data.probability[x][y] !== -1;
}

// Helper: Manhattan distance
function manhattan(x1, y1, x2, y2) {
  return Math.abs(x1 - x2) + Math.abs(y1 - y2);
}

// Main planning loop
for (let step = 0; step < data.maxStepsPerDrone; step++) {
  // Process each drone that still has steps left
  for (const drone of drones) {
    if (drone.stepsLeft <= 0) continue;

    // BFS to find best reachable target
    const queue = [];
    const visited = new Array(rows);
    for (let i = 0; i < rows; i++) {
      visited[i] = new Array(cols).fill(false);
    }
    const parent = new Map(); // maps cell string to parent cell string
    const dist = new Map();   // maps cell string to distance from drone

    const startKey = `${drone.x},${drone.y}`;
    queue.push([drone.x, drone.y, 0]); // x, y, distance
    visited[drone.x][drone.y] = true;
    dist.set(startKey, 0);

    let bestTarget = null;
    let bestProb = -1;
    let bestDist = Infinity;

    while (queue.length > 0) {
      const [x, y, d] = queue.shift();
      const key = `${x},${y}`;

      // If we've exceeded remaining steps, stop expanding this branch
      if (d >= drone.stepsLeft) continue;

      // Check if this cell is a candidate (positive probability and not covered)
      const prob = data.probability[x][y];
      if (prob > 0 && !covered.has(key)) {
        // Prefer higher probability; break ties by smaller distance
        if (prob > bestProb || (prob === bestProb && d < bestDist)) {
          bestTarget = key;
          bestProb = prob;
          bestDist = d;
        }
      }

      // Explore neighbours
      const dirs = [[-1, 0], [1, 0], [0, -1], [0, 1]];
      for (const [dx, dy] of dirs) {
        const nx = x + dx;
        const ny = y + dy;
        const nKey = `${nx},${ny}`;
        if (isValid(nx, ny) && !visited[nx][ny]) {
          visited[nx][ny] = true;
          queue.push([nx, ny, d + 1]);
          parent.set(nKey, key);
          dist.set(nKey, d + 1);
        }
      }
    }

    // If we found a target, reconstruct the path and move
    if (bestTarget) {
      // Reconstruct path from target back to drone's current position
      const path = [];
      let cur = bestTarget;
      while (cur !== startKey) {
        const [x, y] = cur.split(',').map(Number);
        path.push([x, y]);
        cur = parent.get(cur);
      }
      path.reverse(); // now from start to target

      // We only need the first step (or stay if path is empty)
      if (path.length > 0) {
        const move = path[0];
        drone.x = move[0];
        drone.y = move[1];
        drone.path.push(move);
        covered.add(bestTarget);
      }
      // Mark the target as covered (even if we don't move all the way)
      covered.add(bestTarget);
    }

    drone.stepsLeft--;
  }
}

// Prepare output: only the paths, in the same order as input drones
const output = {
  paths: drones.map(d => d.path)
};

console.log(JSON.stringify(output));
```

**How to run**

Save the code to a file (e.g., `drone_planner.js`) and run:

```bash
node drone_planner.js < input.json
```

The program reads the JSON from standard input, computes the drone paths, and prints the
resulting JSON to standard output. The implementation uses only built‑in Node.js modules,
scales to large grids, and follows all movement constraints.
