We need to write a SPIR-V assembly geometry shader for Vulkan 1.0 that takes InputPoints (so each invocation gets 1 vertex) and expands each point into a small colored quad (4 vertices as triangle strip). The input is a 5x5 grid of points in [-0.8, 0.8] with colors, but we don't need to generate the grid; we just need to expand each incoming point into a quad.

We need to follow the requirements:

- OpCapability Geometry
- OpMemoryModel Logical GLSL450
- OpEntryPoint Geometry %main "main" listing ALL interface variables
- OpExecutionMode %main InputPoints
- OpExecutionMode %main OutputTriangleStrip
- OpExecutionMode %main OutputVertices 4
- OpExecutionMode %main Invocations 1

Input from vertex shader:
- gl_PerVertex block with gl_Position (array size 1 for InputPoints)
- User-defined inputs: worldPos (vec3), normal (vec3), color (vec3) - all arrays of size 1.

Output:
- gl_PerVertex block for gl_Position
- outColor (vec3) for fragment shader.

We have a UBO (set=0, binding=0) but we don't need it for this task.

We need to:
1. Read gl_in[0].gl_Position (which is a vec4)
2. Read color[0] (vec3)
3. Define half-size h = 0.04
4. Emit 4 vertices forming a quad centered on the point:
   - (x-h, y-h, 0, 1)
   - (x+h, y-h, 0, 1)
   - (x-h, y+h, 0, 1)
   - (x+h, y+h, 0, 1)
   - All with the same color.

We must emit as a triangle strip (so 4 vertices) and end the primitive after emitting 4 vertices.

We need to write the complete SPIR-V assembly text.

Let's outline the steps:

1. Define the types:
   - %v4float = OpTypeVector %float 4
   - %v3float = OpTypeVector %float 3
   - %float = OpTypeFloat 32
   - %int = OpTypeInt 32 0
   - %uint = OpTypeInt 32 1

2. Define the gl_PerVertex input and output structures:
   - %gl_PerVertex_in = OpTypeStruct %v4float
   - OpDecorate %gl_PerVertex_in Block
   - OpMemberDecorate %gl_PerVertex_in 0 BuiltIn Position
   - %arr_pv_in = OpTypeArray %gl_PerVertex_in %uint_1   (since InputPoints: array size 1)
   - %ptr_in_arr_pv = OpTypePointer Input %arr_pv_in
   - %gl_in = OpVariable %ptr_in_arr_pv Input

   - %gl_PerVertex_out = OpTypeStruct %v4float
   - OpDecorate %gl_PerVertex_out Block
   - OpMemberDecorate %gl_PerVertex_out 0 BuiltIn Position
   - %ptr_out_pv = OpTypePointer Output %gl_PerVertex_out
   - %gl_out = OpVariable %ptr_out_pv Output

3. Define the user-defined input variables:
   - %worldPos = OpVariable %ptr_in_arr_pv Input? Actually, we need to define the type for the input arrays.

   We have:
   - worldPos: vec3 array of size 1
   - normal: vec3 array of size 1
   - color: vec3 array of size 1

   So:
   - %v3float = OpTypeVector %float 3
   - %arr_v3float_1 = OpTypeArray %v3float %uint_1
   - %ptr_in_arr_v3 = OpTypePointer Input %arr_v3float_1

   Then:
   - %worldPos = OpVariable %ptr_in_arr_v3 Input
   - %normal = OpVariable %ptr_in_arr_v3 Input
   - %color = OpVariable %ptr_in_arr_v3 Input

   But note: the vertex shader must have set these locations. We assume location 0 for worldPos, location 1 for normal, location 2 for color.

4. Define the output variable:
   - %outColor = OpVariable %ptr_out_v3 Output   (where %ptr_out_v3 is a pointer to vec3 in output)

   But we need to define the output structure for the user-defined outputs? Actually, in SPIR-V, we can have separate output variables. We'll define:

   - %ptr_out_v3 = OpTypePointer Output %v3float
   - %outColor = OpVariable %ptr_out_v3 Output

5. Define the main function:
   - %main = OpFunction %void None %void_type
   - %entry = OpLabel

   Then:
   - Load gl_in[0].gl_Position: 
        %pos_ptr = OpAccessChain %ptr_in_arr_pv %gl_in %int_0 %int_0
        %pos = OpLoad %v4float %pos_ptr

   - Load color[0]:
        %color_ptr = OpAccessChain %ptr_in_arr_v3 %color %int_0 %int_0
        %color_val = OpLoad %v3float %color_ptr

   - Extract x, y from the position: 
        %x = OpVectorExtractDynamic %float %pos %int_0   (or use OpCompositeExtract)
        %y = OpVectorExtractDynamic %float %pos %int_1

   Alternatively, we can use OpCompositeExtract for the x and y components.

   - Define half-size: 
        %h = OpConstant %float 0.04

   - Compute the four vertices:

        Vertex 0: (x-h, y-h, 0, 1)
        Vertex 1: (x+h, y-h, 0, 1)
        Vertex 2: (x-h, y+h, 0, 1)
        Vertex 3: (x+h, y+h, 0, 1)

   We can compute each vertex as a vec4.

   - Then, for each vertex, store the position in gl_out and the color in outColor, then emit.

   Steps for vertex 0:
        %vx0 = OpFSub %float %x %h
        %vy0 = OpFSub %float %y %h
        %vz0 = OpConstant %float 0.0
        %vw0 = OpConstant %float 1.0
        %pos0 = OpCompositeConstruct %v4float %vx0 %vy0 %vz0 %vw0

        %pos_out_ptr = OpAccessChain %ptr_out_pv %gl_out %int_0
        OpStore %pos_out_ptr %pos0

        %color_out_ptr = OpAccessChain %ptr_out_v3 %outColor %int_0
        OpStore %color_out_ptr %color_val

        OpEmitVertex

   Similarly for vertex 1, 2, 3.

   After emitting the fourth vertex, we call OpEndPrimitive.

6. We must also define the constants: %int_0, %int_1, %uint_1, %float_0_04, etc.

For the output:
