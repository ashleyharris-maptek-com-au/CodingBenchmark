Then we iterate up to 20 steps, escape when dot(z,z) > 4, and color with t=iterations/20 as (t, t*0.5, 1.0-t), alpha 1.0.

We have inputs: worldPos, normal, uv, tangent, color. But we don't need them for the Mandelbrot calculation. We only need the uv.

The UBO is available but we don't need it for this task.

The output is outColor.

We need to compute the Mandelbrot set.

Steps:

1. Get uv from input.
2. Map uv to c: c.x = uv.x * 3 - 2; c.y = uv.y * 2 - 1;
3. Initialize z = vec2(0.0, 0.0);
4. Iterate up to 20 steps:
   - z = vec2(z.x*z.x - z.y*z.y, 2.0*z.x*z.y) + c;
   - Check if dot(z,z) > 4.0, then break.
5. Compute t = iterations / 20.0;
6. Color: vec3(t, t*0.5, 1.0 - t);
7. outColor = vec4(color, 1.0);

Note: The UVs are in [0,1], so mapping to the region [-2,1] in x and [-1,1] in y.

We need to write the complete shader.

Also, note that the input uv is a vec2, so we can use it directly.

We don't need to use the other inputs (worldPos, normal, tangent, color) for this task.

So the shader will be:

```glsl
#version 450

layout(location = 0) in vec3 worldPos;
layout(location = 1) in vec3 normal;
layout(location = 2) in vec2 uv;
layout(location = 3) in vec3 tangent;
layout(location = 4) in vec3 color;

layout(set = 0, binding = 0) uniform UBO {
    mat4 model;
    mat4 view;
    mat4 projection;
    vec4 lightPos;
    vec4 cameraPos;
    vec4 params;
} ubo;

layout(location = 0) out vec4 outColor;

void main() {
    vec2 c = vec2(uv.x * 3.0 - 2.0, uv.y * 2.0 - 1.0);
    vec2 z = vec2(0.0, 0.0);
    int iterations = 0;
    const int maxIterations = 20;
    for (int i = 0; i < maxIterations; ++i) {
        z = vec2(z.x * z.x - z.y * z.y, 2.0 * z.x * z.y) + c;
        if (dot(z, z) > 4.0) {
            iterations = i + 1;
            break;
        }
    }
    float t = float(iterations) / float(maxIterations);
    vec3 mandelbrotColor = vec3(t, t * 0.5, 1.0 - t);
    outColor = vec4(mandelbrotColor, 1.0);
}
```
