Let me check the inputs:
- worldPos: world-space position
- normal: world-space normal (needs normalize)
- uv: texture coordinates
- tangent: tangent vector
- color: per-vertex color (but may not be used if not specified)

UBO has:
- lightPos: (5.0, 5.0, 5.0, 1.0) - this is the light position in world space (since w=1.0, it's a point light position)
- cameraPos: (0.0, 0.0, 3.0, 1.0)
- params: (0.0, 0.5, 0.0, 0.0)

For Gooch shading:
1. Get the normal N (normalize it)
2. Get the light direction L (from surface to light, so L = normalize(lightPos.xyz - worldPos))
3. Calculate t = dot(N, L) * 0.5 + 0.5
4. Mix cool and warm colors based on t
5. Output with alpha 1.0

Let me write the shader:

```glsl
#version 450

layout(location = 0) in vec3 worldPos;
layout(location = 1) in vec3 normal;
layout(location = 2) in vec2 uv;
layout(location = 3) in vec3 tangent;
layout(location = 4) in vec3 color;

layout(set = 0, binding = 0) uniform UBO {
    mat4 model;
    mat4 view;
    mat4 projection;
    vec4 lightPos;
    vec4 cameraPos;
    vec4 params;
} ubo;

layout(location = 0) out vec4 outColor;

void main() {
    vec3 N = normalize(normal);
    vec3 L = normalize(ubo.lightPos.xyz - worldPos);
    
    float t = dot(N, L) * 0.5 + 0.5;
    
    vec3 coolColor = vec3(0.2, 0.2, 0.75);
    vec3 warmColor = vec3(0.7, 0.7, 0.4);
    
    vec3 finalColor = mix(coolColor, warmColor, t);
    
    outColor = vec4(finalColor, 1.0);
}
```
