The task is to write a C++ program that solves the Travelling Salesman Problem for up to 10 million cities, but it must be adaptive based on the number of cities N.

Input: N, then N lines of x y coordinates (floats).

Output: One line with N integers: permutation of  0 to N-1 starting with 0, the visit order (no return to 0).

It must be a valid permutation starting with 0.

Scalability: works for N from 10 to 10M+, time <300s even for largest.

Quality: reasonable routes at all scales.

Environment: 64 cores, 128GB mem, MSVC C++20.

I need complete compilable code with main().

Adaptive logic based on N.

For small N (say <=20): exact solution using DP (Held-Karp) or brute force permutations.
