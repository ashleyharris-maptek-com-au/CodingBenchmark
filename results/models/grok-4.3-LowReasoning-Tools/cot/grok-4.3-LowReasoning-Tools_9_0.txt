The problem is the 1D Cutting Stock Problem. I need to write a C# program that reads from stdin: first line has N and stock_length, second line has N cut lengths.

Output: number of stocks used, then for each stock, the indices (0-based) of the cuts assigned to it.

 - Constraints: cuts up to millions, need to be efficient.

It's like bin packing where bins are of size stock_length, items are the cut lengths.

I need an efficient way to pack them.

Since N can be millions, I need something linear or n log n at worst.

A simple greedy approach like First Fit Decreasing should work well.

But for millions, need to be careful with implementation.

First, I need to read the input efficiently.

In C#, for large N, use fast input.

Plan:

- Read N and L (stock_length)

- Read array of lengths, say int[] cuts = new int[N];

 But indices matter, so I need to keep track of original indices.

  I should create a list of items with length and index.

  Sort them descending for FFD.

  Then, use a list of "bins", each bin has current remaining length and list of indices.
