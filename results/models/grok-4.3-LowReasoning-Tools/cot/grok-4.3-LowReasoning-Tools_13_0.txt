The task is to write a C++ program that finds the Longest Common Substring (LCS) among N strings from stdin.
The problem is to find the Longest Common Substring among N strings, where N can be up to 100 and strings up to 10MB each. Time limit 300s, plenty of memory.

I need an efficient algorithm that scales.

Standard DP won't work for large sizes.

Best way is binary search on the length L, and use rolling hashes to check for existence of common substring of length L, and keep track of the maximum L and one such substring.

To get the actual substring, during the check for max L, I need to find one position in one string that matches.

Plan:

- Read N

- Read vector strings

- Find min_len = min length, and idx of shortest string, say S = strings[0]

- If N==1, output the string itself.

- Use double rolling hash to avoid collisions.
