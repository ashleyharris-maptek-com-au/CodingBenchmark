The task is to write a Node.js program that chooses where to cut firebreaks to maximize the saved town value from a wildfire.
The task is to write a complete Node.js JavaScript program that reads the input JSON from stdin, computes firebreaks (cut cells) within the cutBudget, and outputs the JSON with the firebreaks list to stdout. The goal is to maximize the saved town value fraction.

First, I need to understand the input structure 100%:

- width, height: integers

- grid: array of strings, each of length width, rows height, '.' burnable, '~' not

- ignitions: array of [x,y] integers, starting fire positions (x column, y row?)

- towns: array of objects {id, x, y, value} id string?, x,y coords, value number

- cutBudget: integer max cuts

Assume coords: x horizontal 0 to width-1, y vertical 0 to height-1, grid[y][x]

Fire spreads up,down,left,right to '.' cells.

Cut cells become '~' effectively, block spread.

Towns burn if fire reaches their (x,y) cell, I assume even if town on '.', and value saved if not burned.

If town on '~' or cut, probably doesn't burn.

To maximize sum values of saved towns / total possible.

But output only the firebreaks list [[x,y],..] up to budget, no duplicates I guess.

For algorithm: since scales, need efficient.

I'll implement:

- Use BFS to simulate spread.

- To choose cuts: do multi-source BFS from all ignitions, compute dist to each cell.

- Collect all threatened towns: those with dist < 1e9 i.e. reachable.

- But for cuts: collect all burnable '.' cells that are reachable (dist finite), sort by dist ascending (closest to fire first), pick first min(cutBudget, num) as cuts.

This way, cut early to contain.
