It needs to handle graphs from small to very large sizes, up to 200,000 nodes, but the challenge mentions ...The task is to write a Rust program that reads a graph from stdin and outputs node coordinates that minimize edge crossings as much as possible, handling graphs up to 200,000 nodes efficiently within 30 seconds.

I need to implement a graph layout algorithm in Rust that works for all scales.

Key points:

- Input: 1st line N M, then M lines of u v (0-indexed undirected edges)

For reasonable quality, use force-directed with approximation for large N.
