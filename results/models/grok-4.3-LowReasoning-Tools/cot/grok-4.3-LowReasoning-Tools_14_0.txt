The task is to write a Minesweeper solver in C# that handles boards from small (5x5) to massive (10000x10000). It must find provably safe cells without guessing.
The problem is to write a C# program that solves Minesweeper by finding all provably safe cells (unknown cells that must be safe) without guessing, for boards up to 10000x10000.

Input: H W, then H lines of W chars each: ' ', '0'-'8', '*'

Output: N, then N lines of x y (col row, 0-based) for safe unknowns.

I need to implement deduction logic.

Basic local deduction:

For each revealed cell with number K at (i,j):

Look at its 8 neighbors.

Count how many are '*" (mines known)

 F = count of '*'

  U = count of ' ' (unknown)

  If K - F == 0, then all U neighbors are safe (not mines)

  If K - F == U, then all U are mines.

To propagate, I need to mark deduced mines as known, and also collect safes.

Since safes are not marked as known_mine, but for output we collect them.

For propagation, when we deduce a mine, we set it as known_mine, so it counts as F for other cells.

If we deduce a safe, we can collect it, but don't need to mark it known_mine (since it's not mine).

To avoid infinite loop, use a queue: when we deduce new mines, enqueue the affected revealed cells.
