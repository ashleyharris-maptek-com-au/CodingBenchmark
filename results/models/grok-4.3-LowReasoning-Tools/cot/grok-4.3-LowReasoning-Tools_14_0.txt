The task is to write a Minesweeper solver in C# that handles boards from small to huge, like 10000x10000. It must be efficient.
The problem is to write a C# program that reads a Minesweeper board from stdin and outputs the provably safe cells that are unknown (' ') and must be safe based on the constraints.

I need to implement logic to deduce safe cells without guessing.

The basic local rule:

For each revealed cell with number N at 0-8:

- Count F: number of '*' in adjacent cells (flags)

- Count U: number of ' ' in adjacent cells

- Total adjacent mines should be N, so remaining mines = N - F

- If remaining_mines == 0, all U are safe.

- If remaining_mines == U, all U are mines.

We collect all positions where U are safe from such constraints.

Since deductions can chain (if one is deduced safe, it affects other counts), we need propagation: repeat until no change.

For large boards, need efficient way.
