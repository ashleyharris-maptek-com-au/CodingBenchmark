The task is to write a C++ program that solves the Minimum Edge Cut problem for an undirected graph. It needs to find the smallest set of edges whose removal disconnects the graph, and output the size...The problem is to find the minimum edge cut in an undirected connected graph, meaning the smallest number of edges to remove to disconnect it, and output those edges.

It's the edge connectivity λ(G), and one such cut.

Constraints: must work for any size, even multi-GB, so N and M can be huge, 64GB RAM, so probably N up to 10^7 or so depending on density.

But input size would be large, M up to billions maybe, but practically for time, need efficient algo.

I need an algorithm that can handle large sparse graphs.

Stoer-Wagner is O(N M log N) or O(N^2 M) worst, but for sparse good, but for huge N slow.

Karger's is Monte Carlo, randomized, fast for approx, but for exact need many runs.

For exact, there is deterministic versions but slow.

Since we need exact, and output the edges, perhaps use flow network techniques with multiple source sink.

A reliable way: use the fact that min cut = min over all v of the min cut separating v from some other, but still.

A standard efficient practical way for sparse graphs is to use max-flow with multiple calls, but better use push-relabel or dinic for unit capacities or something .

Since undirected unweighted (for connectivity), capacities 1.

To find global min cut, one efficient way is to pick a source s, compute min cut to all other nodes by running one maxflow, but actually to find the smallest cut, I need the smallest among all possible s-t maxflows.
