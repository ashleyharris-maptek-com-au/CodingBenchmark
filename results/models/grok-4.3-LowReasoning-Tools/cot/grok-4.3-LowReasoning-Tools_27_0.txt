The task is to write a C++ program that colors a graph with k colors, outputting the color assignment for each vertex.

Input: first line num_vertices num_edges num_colors, then num_edges lines of u v (0-based).

Output: space-separated colors for 0 to n-1.

It must be valid 0 to k-1 coloring, no adj same color.

For large n up to 1e6, k=60, need efficient algo within 5 min.

Since NP-hard, use heuristic.

I'll use greedy coloring with vertices ordered by decreasing degree (Welsh-Powell like).

First, build adj list 0 to n-1.

Compute degrees.

Then create vector of vertices sorted by deg descending.

Then color array init -1.
