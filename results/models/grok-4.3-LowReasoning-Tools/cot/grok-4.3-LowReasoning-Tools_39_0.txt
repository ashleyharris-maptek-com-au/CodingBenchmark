The task is to write Rust code for the Graph Bisection problem. Given an undirected graph with even number of vertices, split into two equal parts minimizing the cut edges.
The task is to write Rust code that solves the Graph Bisection problem exactly, finding a minimum cut bisection for an undirected graph with even n vertices.

Since n is even, split into two sets of n/2 vertices each, minimizing the number of edges crossing the cut.

The code must output the cut size and 1 list of n/2 vertices in one partition (the other is complement).

It must be the minimum possible cut, as per strict evaluator.

No constraints given, but for exact solution in Rust (no external crates), assume n small, e.g., <=20, so we can use bitmasks to enumerate all 2^n possibilities, skipping those not with exactly n/2 bits set.

C(n,n/2) for n=20 is ~184k, feasible.
