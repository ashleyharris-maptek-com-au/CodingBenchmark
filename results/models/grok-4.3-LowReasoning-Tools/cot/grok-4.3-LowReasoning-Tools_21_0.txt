The task is to write a C++ program that solves an N-dimensional Snake game for any N from 2 to 20. It must read input from stdin in the specified format and output moves to stdout.

First, I need to clarify the input format. It's given as:

D max_turns (D = number of dimensions, max_turns = turn limit)

bounds (D integers: size of space in each dimension)

snake_length start_pos (length, then D integers for head position)

F (number of food items)

F lines: food positions (D integers each)

O (number of obstacles)

O lines: obstacle positions (D integers each)

For snake, snake_length and start_pos: probably snake_length is initial length, but body not given. I think it's likely that snake starts with length 1 at start_pos, and snake_length is 1 or to be read but ignored for body. To make it work, I'll assume initial snake is just the head at start_pos, length=1, and body empty.

For implementation, use std::vector<int> for positions, with D up to 20.

Bounds: vector<int> bounds(D);

Food: vector<vector<int>> foods;

Obstacles: vector<vector<int>> obs;

Snake: deque or vector of Pos, but deque for efficient pop_front.

For state, to check collision, use set<Pos> for snake body.

For N-D BFS: from current head to a target food, avoiding obs and snake positions (except head).

Since snake grows when eating, the path must not hit body except at 0 length.

BFS in N-D grid: each state is Pos, neighbors by changing one coord by ±1, within bounds.

Bounds: 0 to bounds[i]-1 I assume.

For visited: need hash for Pos.

In C++, I can use unordered_set with custom hash for vector<int>.
