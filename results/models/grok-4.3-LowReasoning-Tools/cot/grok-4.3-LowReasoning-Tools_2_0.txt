First, recall what the Chinese Postman Problem (CPP) is for undirected graphs: find the shortest closed walk that traverses each edge at least once.

Standard algorithm:

1. Check if the graph is connected. If not, it's impossible, but assume it is.

2. Find all vertices with odd degree.

3. Compute the minimum cost perfect matching on these odd-degree vertices, where cost between two is the shortest path distance in the graph.

4. For each matched pair, duplicate the edges along the shortest path.

5. The resulting multigraph is Eulerian (all degrees even, connected).
