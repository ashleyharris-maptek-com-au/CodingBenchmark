For algorithm that works for large scales:

One standard way without matching: use Hierholzer's algorithm but start by adding duplicate edges arbitrarily to make all degrees even, but how.

To do it efficiently:

I can implement a modified Hierholzer's algorithm that allows 2-trails or something.

But let's search my knowledge: actually, there is a way to find a covering tour using DFS with backtracking, but slow.

For efficiency, the way is:

1. Check if graph is connected (using DFS or BFS from 0, ensure all nodes visited, and all edges used 0).

2. Find odd degree nodes.

3. Pair the odd degree nodes arbitrarily (e.g., sort them and pair consecutive).

4. For each pair, find a shortest path or any path between them using BFS (since weights not needed for non-optimal), and duplicate the edges along that path (i.e., 2x the multiplicity on those edges).

5. Then the graph becomes Eulerian (connected, all even degrees).

6. Then run Hierholzer's algorithm to find the Euler circuit.

7. The Euler circuit will be the route, and the duplicated edges mean traversed twice.

This works, and for large N, 1M edges, if number of odd nodes is small, say <20, then few BFS, each O(M), fine.

But worst case all nodes odd, 50k pairs, each BFS O(M), total O(50k *1M) = 5e10, way too slow.

So for 100k nodes, if dense or sparse, but for extreme scale, if many odds, bad.

So need a better way that doesn't depend on number of odds.

A simple way that always works and scales: duplicate ALL edges, make multiplicity *2, then all degrees *2 even, 2M edges, then Hierholzer on that, the tour traverses each original edge exactly twice.
