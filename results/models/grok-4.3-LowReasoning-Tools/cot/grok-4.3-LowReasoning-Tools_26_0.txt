The task is to write a complete, compilable Rust program that clusters 3D points into exactly N clusters using k-means, handling various scales with binary I/O, and adapting to different dataset sizes.

First, understand the input/output:

- Input from stdin: First, probably a line with num_points num_clusters ( 2 numbers separated by space), then binary float32 triples (little-endian, 12 bytes per point: x,y,z).

- Output to stdout: binary uint32 (little-endian, 4 bytes per point) for cluster indices.

The program must be efficient for up to 1B points, within  5 min, respect memory.

Since stdin is a stream, for large datasets, we need a streaming algorithm that processes points sequentially without loading all into memory.

For small datasets, we could load all, but to handle all scales, implement a streaming mini-batch k-means that processes in chunks.

But for true scalability to 1B, one-pass streaming.

I'll implement an online k-means: initialize centroids with first N points, then for each point, assign to closest centroid, update centroid as weighted average (simple online update).

To ensure exactly N clusters, if a cluster empty at end, reassign or something, but for simplicity, allow 0 count, but output uses 0 to N-1.

For convergence, since one pass, no detection.

For adaptive: based on num_points, but since can't load, always streaming.
