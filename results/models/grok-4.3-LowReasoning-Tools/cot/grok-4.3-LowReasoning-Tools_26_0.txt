The task is to write a Rust program for k-means clustering on 3D points, handling scales from small to massive datasets, with streaming for large ones.
The task is to write a Rust program that performs k-means clustering on 3D points from stdin and outputs cluster indices to stdout in binary format. It must handle from small to very large datasets adaptively, with streaming for large ones.

First, understand the input/output:

- Input: First line: num_points num (probably space-separated, num_points as u64, num_clusters as u32)
- Then binary stream: for each point, 12 bytes little-endian f32 x, y, z
- Output: binary stream: for each point, 4 bytes little-endian u32 cluster index (0 to N 1)

Program must be compilable with rustc (2021 edition), read from stdin, write to stdout.

Adaptive strategy based on size.

For small: load all, standard k-means.

For large: streaming, but challenge with multiple passes.

For streaming k-means, common is mini-batch online k-means: process 1 pass, update centroids incrementally with batches.

For assignment, can assign on the fly and update, but for final assignment, need final centroids, so can do one pass to update centroids, but then to assign, need second pass.
