The task is to write Rust code for the Minimum Vertex Cover problem. It needs to handle graphs up to 2M vertices and 15M edges, which is large, so it must be efficient.
The problem is Minimum Vertex Cover, NP-hard, so for large graphs like 2M vertices, 15M edges, I need an approximation algorithm that's efficient.

A standard greedy algorithm for Vertex Cover: repeatedly select the vertex with the highest degree in the remaining graph, add it to the cover, remove it and its incident edges.

Implementing this naively would be too slow for large n,m because finding max degree each time is costly.

A better way: use a maximal matching approach. Compute a maximal matching (greedily), then the vertices in the matching form a vertex cover? No, the endpoints of the matching cover the matched 2*size edges, but for unmatched edges? No.

Recall: in any graph, a maximal matching gives a vertex cover by taking all vertices incident to the matching edges. That is, all matched vertices. Since matching is maximal, no edge between unmatched, so all edges are incident to at least one matched vertex. Yes! Because if an edge between two unmatched, it could be added to matching.

Perfect! So VC size = 2 * |M|, and it's a 2-approx to OPT.

And very efficient: O(E) time.
