The task is to solve the 1D Cutting Stock Problem in C#. 
The problem is the 1D Cutting Stock Problem, which is essentially the bin packing problem. We have stock pieces of length S (stock_length), and N items with lengths L[i], and we need to pack them into as few stocks as possible without exceeding S in each, and output the assignment by indices.

N 100
30 40 50

No, input is:

First line: N S

Second line: L[0] L[1] ... L[N-1]

In example:

3 100

30 40 50

Yes, N=3, S=100, cuts=[30,40 50]

Output:

2

0 1

2

So one stock with 0 and 1 (30+40=70<=100), one with 2 (50).

To minimize number of stocks.

For large N (millions), need efficient heuristic.

Since exact is NP-hard, use approximation algorithm.

 To implement in C# efficiently.

I need an algorithm that runs fast for N=1e6.

Standard FFD is O(N log N for sort + N * B) where B number of bins, bad if B large.

Better way: use First Fit with a data structure for remaining space.

But in C #, I can use a SortedSet or PriorityQueue for remaining capacities, but need to associate with bin index.

I can use a multiset of remaining capacities, but when placing, I need to find the one with smallest remaining >= size for Best Fit, or any for First Fit.

For First Fit, it's harder because order matters.

For good approximation, Best Fit Decreasing (BFD) is good, and can be implemented with priority queue.

Let's go with BFD: sort items descending by size, then for each item, place it in the bin with the smallest remaining capacity that is still >= item size (best fit).

To implement:

- Create list of items: tuple (length, index), sort by length descending.

- Have a list of current remaining capacities for each bin, and list of assignments (List<int> for indices per bin).

- To place fast, maintain a SortedDictionary<int, List<int>> where key = remaining, value = list of bin ids with that remaining.

SortedDictionary in C# is sorted by key.

For each item s:

- Find the smallest key >= s : use SortedDictionary's lower_bound like, by iterating or since sorted, I can use .Keys.GetEnumerator or better use TreeMap but C# SortedDictionary has no direct lower_bound, but I can use .Where or but slow.

SortedDictionary is red-black tree, to find min key >=s, I can use a loop or keep a separate SortedSet for keys, but complicated.

Since for speed, perhaps use a priority queue (MaxHeap) but for min remaining >= , it's tricky.

For Best Fit, I need the smallest remaining that >=s, so min among those >=s.

So need ordered set of remainings.

using System;

using System.Collections.Generic;

using System.IO;

class Program {

static void Main() {

  string[] all = Console.In.ReadToEnd().Trim().Split(new char[] {' ', '\n', '\r', '\t'}, StringSplitOptions.RemoveEmptyEntries);

  int N = int.Parse(all[0]);

  int S = int.Parse(all[1]);

  int[] cuts = new int[N];

  for(int i=0; i<N; i++) {

    cuts[i] = int.Parse(all[2+i]);

  }

  // prepare sorted indices desc

  int[] indices = new int[N];

  for(int i= 0; i<N; i++) indices[i] = i;

  Array.Sort(indices, (x,y) => cuts[y] - cuts[x]); // desc , note if equal ok

  // now bins

  List<int> remainings = new List<int>();

  List<List<int>> assignments = new List<List <int>>();

  for(int k=0; k<N; k++) {

    int i = indices[k];

    int size = cuts[i];

    bool placed = false;

    for(int b=0; b<remainings.Count; b++) {

      if(remainings[b] >= size) {

 remainings[b] -= size;

  assignments[b].Add(i);

  placed = true;

  break;

      }

    }

    if(!placed) {

      remainings.Add(S - size);

      assignments.Add(new List<int>{i});

    }

  }

  // output

  Console.WriteLine(assignments.Count );

  for(int b=0; b<assignments.Count; b++) {

    assignments[b].Sort(); // sort indices asc

    Console.WriteLine(string.Join(" ", assignments[b]));

  }

}

}
