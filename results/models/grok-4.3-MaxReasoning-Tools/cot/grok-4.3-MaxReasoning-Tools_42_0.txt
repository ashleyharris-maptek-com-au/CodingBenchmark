The task is to write a SPIR-V assembly geometry shader for Vulkan 1.0.
The task is to write a SPIR-V assembly for a geometry shader that expands each point into a small colored quad.

Input primitive: InputPoints, so array size 1.

Output: OutputTriangleStrip with 4 vertices.

I need to read gl_in[0].gl_Position, color[0], and expand 4 points around it.

The quad: (x-h, y-h), (x+h, y-h), (x+h, y+h), (x-h, y+h), (x+h, y-h) wait, for triangle strip order: typically (x-h,y-h), (x+h,y-h), ( x-h,y+h), (x+h,y+h), (x-h,y+h), (x-h,y+h), (x-h,y+h), (x+h,y+h), (x-h,y+h), (x-h,y+h), (x+h,y+h) no.

For triangle strip quad: vertex1: bottom left, vertex2: bottom right, vertex3: top left, vertex4: top right.

So order: (-h,-h), (+h,-h), (-h,+h), (+h,+h)

And each with z=0, w=1, same color.

h=0.04

Since 4 vertices, OpEmitVertex four times, then OpEndPrimitive.

Since one invocation per point, and max 4 verts.

Also need to use the color from color[0].

Note: worldPos and normal are declared but not used, but task says include them? But task says "Read color[0]", but to be complete, declare all.
