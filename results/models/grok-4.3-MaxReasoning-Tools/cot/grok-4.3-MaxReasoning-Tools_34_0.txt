Let's look for a simple Python CVRP heuristic code in my knowledge.

A simple 2-phase: 

1. Cluster customers into groups of size that fit capacity, using greedy.

2. For each cluster, order the customers using nearest neighbor from depot.

To limit to K:

- Use greedy assignment to K bins (routes), using FFD or best fit.

- Sort customers by demand desc.

- For each customer, assign to the route with current largest remaining capacity that can fit it (best fit decreasing).

- Then, for each route's customer list, sort them by angle or use nearest neighbor to order them.

This will use exactly K routes or less if some empty.

If total demand > K*CAP, it may not fit, but assume fits.
