Use list scheduling with priority rule.

To implement:

- Use a priority queue for ready operations.

- Each operation has job, position in job.

- Maintain current time for each machine.

- Maintain for each job the current position (next task to do), and the ready time for that position.

- Initially, all jobs' first tasks are ready at time 0.

- Then, use a priority queue to pick next task to schedule: priority based on e.g. duration (SPT), or machine id, etc.

- For each pick: compute start = max( ready_time of this task, avail_time of machine)

- Record the start and machine for that task.

- Update avail of machine to start+ dur

- Update ready_time of next task in same job to start+dur

- If next task, add it to ready queue.

This is standard list scheduling with dispatching rule.

  // schedule

  auto [m,d] = jobs[best_j][curr_task[best_j]] ;
