The task is to write a SPIR-V assembly geometry shader for rendering wireframe by emitting thin quads for each edge of a triangle with adjacency.
The task is to write the complete SPIR-V assembly for the geometry shader.

Key points:

- InputPrimitive: InputTrianglesAdjacency

- OutputPrimitive: OutputTriangleStrip

- OutputVertices: 12 (3 edges * 4 verts)

- For each of 3 edges: 0-2,  2-4,   4-0

- For each edge, emit 4 vertices as triangle strip.

- Thickness 0.008

- Edge color = average of two endpoint colors.

- Use worldPos? But to compute thickness, need to know direction.

To compute perpendicular for thickness:

Since it's 3D, to make quad perpendicular to edge:

- Need a normal to the edge.

- But to make it screen-space thickness, need to offset in direction perpendicular to edge in view space or NDC.

This requires some math.

Probably, the worldPos are in world space, but to offset, we need view or projection.

The UBO is available: same as test 41 (model/view/proj matrices, lightPos, cameraPos, params).

So, probably use cameraPos or view matrix to compute view space, then offset perpendicular in view space, then project.

But that would be complex for SPIR-V assembly.

Perhaps 2D offset in clip space.

Let's try to write the code in GLSL first, then translate to SPIR-V.

In GLSL for geometry shader:

#version 450

layout(triangles_adjacency) in;

layout(triangle_strip, max_vertices=12) out;

in vec3 color[]; // wait, in geometry, the inputs are gl_in[].color etc.

layout(location=0) in vec3 worldPos[]; no, in GS, the inputs are:

in gl_PerVertex gl_in[6];

in vec3 color[6];

etc.

Then:

void main() {

  // for each edge

  int indices [3] = {0,2,4};

  for(int e=0; e<3; e++) {

    int i = indices[e];

    int j = indices[(e+1)%3];

    vec4 posA = gl_in[i].gl_Position;

    vec4 posB = gl in[j].gl_Position;

    vec3 colA = color[i];

    vec3 colB = color[j];

    vec3 edgeColor = (colA + colB) / 2.0;

    // compute perp offset

    vec2 ndcA = posA.xy / posA.w;

 vec2 ndcB = posB.xy / posB.w;

    vec2 dir = ndcB - ndcA;

    float len = length(dir);

    if(len == 0.0) continue;

    dir /= len;

    vec2 perp = vec2(-dir.y, dir.x);

 vec2 offset = perp * 0.008;

    // now the 4 points

    // A+ , A- , B+ , B-

    // but to add offset in NDC

    // define function to add offset to pos

    vec4 add_offset(vec4 p, vec2 off ) {

      return vec4(p.xy + off * p.w, p.zw);

    }

    vec4 Aplus = add_offset(posA, offset);

    vec4 Aminus = add_offset(posA, -offset);

    vec4 Bplus = add_offset(posB, offset);

    vec4 B minus = add_offset(posB, -offset);

    // order for triangle strip: Aplus, Aminus, Bplus, Bminus

    // Emit

    gl_Position = Aplus;

    outColor = edgeColor;

    EmitVertex();

    gl_Position = Aminus;

    outColor = edgeColor ; wait, outColor is vec3, but need to set for all

    EmitVertex();

    gl_Position = Bplus;

    EmitVertex();

    gl_Position = Bminus;

    EmitVertex();

    EndPrimitive();

  }

}

Yes, this is the idea.

Note: outColor is per vertex, same for the quad.

Let's write the full text.

The SPIR-V assembly text:

OpCapability Geometry

OpCapability Shader

OpMemoryModel Logical GLSL450

OpEntryPoint Geometry %main "main" %gl %gl_out %in_color %outColor

; need to decorate locations

OpDecorate %gl_PerVertex_in Block

OpMemberDecorate %gl_PerVertex_in 0 BuiltIn Position

OpDecorate %gl_PerVertex_out Block

OpMemberDecorate %gl_PerVertex_out 0 BuiltIn Position

OpDecorate %in_color Location 2

OpDecorate %outColor Location 0

; Execution modes

OpExecutionMode %main InputTrianglesAdjacency

OpExecutionMode %main OutputTriangleStrip

OpExecutionMode %main OutputVertices 12

OpExecutionMode %main Invocations 1 ; since one triangle

; Ext inst

%glsl = OpExtInstImport "GLSL.std.450"

; Types

%void = OpTypeVoid

%float = OpTypeFloat 32

%int = OpTypeInt 32 1

%uint = OpTypeInt 32 0 ; for array size

%v2float = OpTypeVector %float 2

%v3float = OpTypeVector %float 3

%v4float = OpTypeVector %float 4

%uint_6 = OpConstant %uint 6

%int_0 = OpConstant % int 0

%int_1 = OpConstant %int 1

%int_2 = OpConstant %int 2

%int_4 = OpConstant %int 4

%float_0 = OpConstant %float 0

%float_0_008 = OpConstant %float  0.008

%float_0_5 = OpConstant %float 0.
