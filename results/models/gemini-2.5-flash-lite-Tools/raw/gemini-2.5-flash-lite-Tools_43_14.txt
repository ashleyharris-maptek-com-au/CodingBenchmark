# HLSL for Vulkan Pixel Shader

This document outlines the requirements and conventions for writing HLSL pixel (fragment) shaders designed to be compiled for Vulkan.

## Compilation Strategy
- **Compiler:** Use the **DirectX Shader Compiler (DXC)** for all HLSL to SPIR-V compilation.
- **Compilation Flags:** Must include:
  - `-spirv`: Enables SPIR-V backend.
  - `-fspv-target-env=vulkan1.2` (or higher, depending on project requirement).
  - `-T ps_6_0`: Specifies the Pixel Shader 6.0 profile.

## Vulkan-Specific Syntax
Use the `[[vk::...]]` attribute namespace for explicit layout control.

### Inputs and Outputs
- Use `[[vk::location(N)]]` for vertex-to-fragment inputs and fragment shader outputs (e.g., `SV_Target`).

### Descriptors
- Use `[[vk::binding(N, M)]]` to explicitly map descriptors to Binding `N` and Descriptor Set `M`.
- Alternatively, standard HLSL `register` semantics map to Vulkan:
  - `register(bN, spaceM)` $
ightarrow$ Binding `N`, Set `M` (Uniform buffers).
  - `register(tN, spaceM)` $
ightarrow$ Binding `N`, Set `M` (Texture/SRV).
  - `register(sN, spaceM)` $
ightarrow$ Binding `N`, Set `M` (Sampler).

### Constants
- **Push Constants:** Use `[[vk::push_constant]]` attribute.
- **Specialization Constants:** Use `[[vk::constant_id(N)]]` attribute.

## Important Considerations
- **Coordinates:** Vulkan viewport Y points down. Shader logic must account for this if reusing assets designed for DirectX (Y-up).
- **Depth Range:** Default Vulkan depth range is `[0, 1]`.
- **Organization:** Align `space` indices in HLSL with the project's `VkDescriptorSetLayout` structure.
