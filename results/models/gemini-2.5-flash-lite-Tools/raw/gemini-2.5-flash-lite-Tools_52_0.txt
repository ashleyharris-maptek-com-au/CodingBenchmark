The autopilot system is implemented in Python, encapsulating sensor fusion, PID control, and decision-making logic within a single `autopilot_step` function.

**Sensor Fusion and Validation:**
A `SensorManager` class is responsible for ingesting raw sensor data, handling validity flags, and performing basic fusion using median-based approaches to mitigate flickering and sensor failures. It fuses data from redundant sensors (e.g., wheel speed, GPS, gyroscopes, accelerometers, vision systems, lidar) to provide a more robust and reliable estimate for vehicle speed, yaw rate, acceleration, steering angle, lane information (offset, curvature, line distances), obstacle detection, and speed limits. Historical data is maintained to assist in filtering and graceful degradation.

**Control Logic:**
The `AutopilotControl` class orchestrates the vehicle's behavior:
- **Speed Control:** A PID controller manages throttle and brake inputs to maintain a `target_speed_kmh`, adjusted by detected `speed_limit_kmh` and in response to obstacles. Aggressive braking is applied if an obstacle is very close.
- **Lane Keeping:** A separate PID controller guides steering input to keep the vehicle centered in its lane, primarily using vision-based lane offset and road curvature information.
- **Obstacle Avoidance:** If obstacles are detected by vision or lidar, the system prioritizes reducing speed. A simplified approach is taken for potential lane changes to avoid obstacles or reach a `target_lane`.
- **Traction Management:** The system dynamically adjusts the aggressiveness of throttle, brake, and steering inputs based on `estimated_mu` (tire-road friction) to ensure safer operation on low-friction surfaces.

**Graceful Degradation:**
The median-based fusion in the `SensorManager` inherently provides a degree of graceful degradation by allowing the system to continue functioning even if some redundant sensors fail or flicker. Fallback mechanisms are in place for critical sensor data (e.g., using historical data for speed if all sensors fail temporarily).

**Persistence:**
The `SensorManager` and `PIDController` instances are maintained as module-level variables within the `AutopilotControl` class, ensuring that their internal states (e.g., historical readings, integral terms) persist across `autopilot_step` calls.

The `autopilot_step` function serves as the main entry point, calling the `AutopilotControl`'s `run_step` method to process the current state and return the calculated steering, throttle, and brake commands.
