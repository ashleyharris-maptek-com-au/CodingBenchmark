The task is to write a C++ solver for the Longest Common Substring (LCS) problem for multiple strings. It must handle from small to very large inputs efficiently.
The problem is Longest Common Substring for multiple strings. I need to find the longest string that appears as a contiguous substring in all N strings.

Input: N, then N strings.

Output: one such longest substring, or empty if none.

It has to be scalable, adaptive algorithm based on size.

For small cases 2-3 short strings: easy.

Since we need any one, and to find max L, we can choose the shortest string as base, and try to find the longest L such that some substr of base is in all others.

To find the max over all possible substrings of base, the max L where it is common.

But how: for each possible start i in base, find the max L such that base[i..i+L) is substring of all other strings.

That is, for each i, L_max for that prefix from i is min over other strings = min over j=1 to base.size()-i , min over j, the max match length that is common prefix match with all others.

To compute for each other string's substring.

For each other string, min over all other strings' longest match length where the substr matches exactly from that position.

For each position i in base, for each other string, compute the max k such that base[i..i+k) == other.substr(p, k) for some p, i.e. the longest substring starting at i in base that appears somewhere in other.

That is longest common substring starting at i in base with the whole other string.

To compute that, for each i, for each other, use string::find or better KMP or Z algo on base[i..] + separator + other.

But time consuming.
