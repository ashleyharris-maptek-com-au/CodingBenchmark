The task is to write a complete Node.js JavaScript program for planning grid paths for search drones.
The task is to write a Node.js program that reads a JSON input with probability grid, drones starting positions, and maxStepsPerDrone, then outputs paths for each drone.

I need to plan paths for drones to search the grid, covering unique probabilities.

The score is "unique probability covered compared with a greedy reference search " so probably to match or exceed a greedy baseline.

I need to implement something reasonable.

Assume input structure:

data = {

probability: [[numbers]],

drones: [{id, x, y}, ...],

maxStepsPerDrone: number  // assume same for all

}

Output: {"paths": [[[x 0,y0], [x1,y1], ...], ...]} for each drone's path.

Path length probably maxSteps +1, including start.

Moves: U D L R or stay (so 5 options).

Stay inside bounds, no -1.

To implement a good planner.

Since scalable, and Node 1 thread, need efficient algo.

Idea: use greedy coverage.

- Identify all valuable cells: list of positions with prob >0

- Sort them by value descending.

- Maintain current position for each drone, initially start.

- Maintain visited set global for unique coverage.

- But since paths planned in advance, need to plan the sequence.

One way: simulate time steps, at each global step, assign moves to drones to cover new high value.

But better: use a priority queue for uncovered cells by value.

For each drone, plan its path by repeatedly moving to the highest value uncovered cell that is reachable within remaining steps, using BFS for shortest path to it, then move along that path, mark visited along the way.

But if many cells, and many drones, it could be slow if not careful.

If grid 50x50=2500 cells, drones 10, maxsteps 100, then for each step, BFS O(grid), 2500*100*10 ~ 2.5M ops, ok for Node.

But optimize.

Let's code a simple greedy local for each drone sequentially.

Plan:

- Read data

- prob = data.probability

- H = prob.length, W=prob[0].length

- drones = data.d rones

- maxSteps = data.maxStepsPerDrone

- Assume maxSteps is number.

- Create global covered: 2D array bool, initially false

- For each drone i:

  path = []

  cur_x, cur_y = drones[i].x, drones[i].y

  path.push [cur_x, cur_y]

  covered[cur_y][cur_x] = true

  for step in 1 to maxSteps:

    best_score = -inf

    best_nx, best_ny = cur_x, cur_y  // stay if no better

    for each dir in 5 dirs (dx,dy) : nx=cur+dx etc

      if valid: 0<=nx<W, 0<=ny<H, prob[ny][nx] != -1

        score = (covered[ny][nx] ? -10000 : 0) + prob[ny][nx]

 if score > best_score:

    best = nx,ny

  Then append best to path

  cur = best

  covered[best y][best x] = true

This way, each drone greedily chooses next cell with highest prob, preferring uncovered (by giving penalty to covered).

Since sequential, first drone takes high values, later ones avoid them.

Good for unique.

If a drone is far, it may not reach high value if not adjacent, because it only moves to adjacent each step.

But since it does maxSteps steps, it can travel distance maxSteps.
