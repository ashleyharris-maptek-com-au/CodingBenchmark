The task is to solve the 1D Cutting Stock Problem in C#. The goal is to cut pieces from stock lengths with minimal waste, but here it's about minimizing the number of stock pieces used.
The problem is the 1D Cutting Stock Problem. I need to assign cut lengths to stock pieces of length stock_length, minimizing the number of stocks used, and output the assignment with indices.

It's essentially the bin packing problem where each bin has capacity stock_length, and items are the cut lengths. I need to pack 1D items into bins without exceeding capacity, and report which items in which bin.

Constraints: N can be millions, so need efficient algorithm, heuristic is fine since exact is NP-hard.

I should use First Fit Decreasing (FFD) or Best Fit Decreasing (BFD) for good approximation.

For efficiency with 1e6 items:

- Sorting is O(N log N), ok for 1e6.

- For placement: need data structure that allows fast "find first bin that fits" or "best fit".

In C#, SortedSet can be used for Best Fit: keep sorted by remaining capacity.

For Best Fit: 1. Sort items descending. 2. Maintain a multiset of remaining capacities. For each item, find the smallest remaining >= size (lower_bound or ceiling), assign to it, update remaining.

But to track which bin, need to associate bin id with remaining.

So use a SortedDictionary<int, List<int>> // key: remaining, value: list of bin ids with that rem

Also, a List<List<int>> bins; // to store the indices for each bin

When placing item i with size s:

- Find the smallest key >= s in SortedDictionary.

- If found, take a bin_id from the list, remove that bin from the list (if empty remove key), add s to used, new_rem = old_rem - s, add bin_id with new_rem.

- If no such, create new bin, add it, with rem = L - s, assign indices.
