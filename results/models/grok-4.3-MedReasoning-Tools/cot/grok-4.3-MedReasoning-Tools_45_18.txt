The task is to create a SPIR-V fragment shader as raw binary data, encoded as a hex string. It must be valid SPIR-V that passes spirv-val and renders on a unit sphere.
The task is to write a SPIR-V fragment shader as a hex string. It needs to render a heatmap on a unit sphere based on NdotV.

First, understand what NdotV is: max(dot(N, V), 0), where N is normal, V is view direction from camera to point.

 V = normalize(cameraPos - worldPos), but since unit sphere, worldPos is position vector.

Inputs: location 0: worldPos (vec3), location 1: normal (vec3), etc.

UBO: mat4 model, view, proj, lightPos, cameraPos, params.

For view direction, use cameraPos and worldPos.

Output: location 0: vec4 outColor with alpha 1.0

The color calculation:

NdotV = max(dot(N, V), 0)

if NdotV < 0.5:

  r=1, g=2*NdotV (so 0 to 1), b=0

else:

  r = g = 1 - (NdotV - 0.5)*2   (from 1 to 0 as NdotV from 0.5 to 1)

  b = (NdotV - 0 .5)*2   (0 to 1)

So basically, when NdotV low, blue=0, red=1, green low; when high, blue high, red green low.

It's a gradient from red to blue or something.
