The proposed implementation plan for the 3D Lunar Lander controller is summarized below:

## Overview
This project implements a real-time 3D Lunar Lander controller in C++20. The system must handle high-frequency state updates, parse large binary terrain files, and output control commands with minimal latency.

## Architecture
- **Main Thread:** Initializes the system, parses initial headers, and spawns worker threads.
- **Terrain Loader:** A dedicated component to parse the binary terrain file into a memory-efficient structure.
- **IO/Control Loop:** A high-priority thread responsible for reading stdin and writing stdout.
- **Controller:** Logic to calculate thrust and orientation commands based on the current state and target.

## Implementation Steps
1.  **Core Data Structures & Terrain Loading:** Define `LanderState` and `Command` structs; implement `Terrain` class.
2.  **IO & Concurrency:** Implement thread-safe IO and background terrain parsing.
3.  **Control Logic:** Implement PID control for position, velocity, and thrust vector.
4.  **Testing & Refinement:** Verify and optimize performance.
