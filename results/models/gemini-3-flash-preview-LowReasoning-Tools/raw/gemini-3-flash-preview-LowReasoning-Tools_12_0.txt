# 3D Bin Packing Solver Plan

## Objective
Create a scalable C# 3D Bin Packing solver that handles polyhedra of varying complexity and container sizes from trivial to extreme (up to 500x500x500). The solver must maximize packing density while staying within a 5-minute execution limit.

## Key Files & Context
- `Program.cs`: Main entry point and solver logic.
- Input: Stdin (V, F, vertices, faces, W, H, D).
- Output: Stdout (count, translations, quaternions).

## Implementation Steps

### 1. Data Structures
- `Vector3`: Simple struct for x, y, z coordinates.
- `Quaternion`: Struct for rotations (w, x, y, z).
- `Polyhedron`: Stores vertices, faces, and pre-calculated AABB.
- `Placement`: Stores translation and rotation for a packed item.

### 2. Geometry & Collision Detection
- **AABB Calculation**: Calculate the bounding box for the polyhedron in its original and rotated states.
- **Rotation Logic**: Implement quaternion-based rotation for vertices.
- **Collision Detection**:
    - **Primary**: AABB-based collision for speed.
    - **Secondary (Optional/Adaptive)**: If scale is small, use more precise checks (e.g., SAT) if necessary, but AABB is likely sufficient for high-density packing of many identical items.

### 3. Adaptive Packing Strategy
- **Scale Assessment**: Determine the scale based on `W * H * D`.
- **Trivial/Medium Scale**:
    - Try multiple orientations (e.g., 6 or 24 orthogonal rotations).
    - Use a First-Fit Decreasing (FFD) or Best-Fit heuristic.
    - Maintain a list of "empty spaces" or use a 3D occupancy grid if memory allows.
- **Large/Extreme Scale**:
    - Use a highly efficient "Shelf" or "Layer" packing algorithm.
    - Minimize collision checks by packing in a structured grid/row format.
    - Prioritize speed to ensure completion within 5 minutes.

### 4. Main Loop & I/O
- Parse input from `Console.In`.
- Execute the chosen strategy.
- Print results to `Console.Out`.

### 5. Performance Optimizations
- Use `System.Numerics.Vector3` and `System.Numerics.Quaternion` for SIMD acceleration.
- Pre-calculate rotated AABBs for common orientations.
- Use efficient data structures for spatial partitioning (e.g., a simple 3D bitset or grid for occupancy).

## Verification & Testing
- **Trivial Case**: Verify it packs a few items correctly in a 5x5x5 box.
- **Large Case**: Ensure it doesn't time out and maintains a reasonable packing density.
- **Collision Check**: Ensure no two polyhedra overlap and all are within the container bounds.
