# Implementation Plan - 3-SAT Solver

The goal is to solve 3-SAT instances with up to 20 million variables and 85 million clauses within 5 minutes using Python. The instances are guaranteed to be satisfiable.

## Proposed Strategy

Given the massive scale, a standard DPLL or CDCL solver in pure Python will be too slow. Instead, I will implement an optimized local search algorithm (a variant of Schöning's algorithm) that uses a "scan-and-flip" approach. This approach is highly efficient in Python because it minimizes object creation and overhead in the inner loop.

### 1. Memory Efficiency
- **Clauses Storage:** 85M clauses * 3 literals = 255M integers. Storing these in a standard Python list would consume ~12GB. I will use `array.array('i')`, which uses 4 bytes per integer, totaling ~1GB.
- **Assignment Storage:** 20M variables stored in a `bytearray`, using ~20MB.
- **Input Parsing:** Reading the entire input at once using `sys.stdin.read().split()` is the fastest but memory-heavy. I will attempt this and ensure memory is freed as soon as the `array.array` is populated.

### 2. Algorithm: Fast Local Search
- **Initialization:** Start with a random assignment (found to be more effective than all-zeros for many instances).
- **Core Loop:** Iterate through all clauses. For each unsatisfied clause, flip one of its three variables at random.
- **Optimizations:**
    - Use short-circuiting in the clause satisfaction check.
    - Use a pre-generated pool of random choices for flipping to avoid calling `random` functions inside the tight inner loop.
    - Use local variable references for global or closure variables to speed up access.
    - Use `map(str, ...)` and chunked output for fast results printing.

### 3. Loop Optimization
The inner loop will look like this:
```python
for i in range(0, total_lits, 3):
    l1 = c[i]
    if l1 > 0:
        if a[l1]: continue
    else:
        if not a[-l1]: continue
    
    # ... check l2 and l3 ...
    
    # If unsatisfied, flip a random literal's variable
    f_var = abs(c[i + r])
    a[f_var] ^= 1
```
This avoids expensive operations like `abs()` or branchy logic in the common case (satisfied clauses).

## Implementation Steps

1.  **Read Input:** Use `sys.stdin.read().split()` for speed, then convert to `array.array('i')`.
2.  **Initialize:** Create a `bytearray` for the assignment and initialize it using `random.getrandbits`.
3.  **Local Search:** Implement the scan-and-flip loop with a time-monitoring exit to ensure completion within 5 minutes.
4.  **Output:** Print "SAT" followed by the space-separated assignment using chunked `sys.stdout.write`.
