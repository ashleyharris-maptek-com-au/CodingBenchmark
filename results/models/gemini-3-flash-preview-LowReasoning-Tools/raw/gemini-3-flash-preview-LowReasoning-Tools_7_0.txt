I have analyzed the requirements for implementing a Constructive Solid Geometry (CSG) Union algorithm in C++. The task requires a solution that is both robust for complex geometries and scalable for meshes with millions of vertices.

My proposed strategy involves:
1.  **Spatial Partitioning**: Using an AABB Tree to efficiently detect intersections between triangles of the two meshes, reducing the complexity from $O(N^2)$ to $O(N \log N)$.
2.  **Triangle Splitting**: Intersecting triangles will be subdivided into smaller fragments along their intersection lines. This ensures that the two meshes conform to each other at the boundary.
3.  **Classification**: Each triangle fragment will be classified as being inside, outside, or on the surface of the other mesh. This will likely be done using a robust point-in-mesh test (e.g., based on winding numbers or ray casting).
4.  **Union Assembly**: The final mesh will be composed of Mesh A fragments outside Mesh B, Mesh B fragments outside Mesh A, and boundary fragments with consistent orientation.
5.  **Robustness**: Using epsilon-based arithmetic and careful handling of degenerate cases to maintain manifold and watertight properties.

### CSG Union Solver Implementation Plan

## Objective
Implement a robust and scalable CSG Union operation in C++ that handles meshes from trivial to millions of vertices.

## Key Components
1.  **Data Structures**:
    *   `Point3`: 3D coordinates.
    *   `Mesh`: Vertices and Faces (triangles).
    *   `AABB Tree`: Spatial partitioning to optimize intersection tests.

2.  **Algorithm: Booleans via Triangle Splitting**
    *   **Intersection**: Identify all intersecting triangle pairs using AABB trees.
    *   **Subdivision**: Fragment triangles of Mesh A against Mesh B and vice versa.
    *   **Classification**: For each fragment, determine if it is inside, outside, or on the boundary of the other mesh.
    *   **Selection**: For Union:
        *   Keep fragments of A outside B.
        *   Keep fragments of B outside A.
        *   Keep one copy of coincident fragments (same orientation).

3.  **Scalability & Performance**:
    *   Use `std::vector` for memory efficiency.
    *   AABB tree for $O(N \log N)$ intersection detection.
    *   Handle large meshes by avoiding redundant geometric calculations.

4.  **Robustness**:
    *   Use a small `EPSILON` for floating-point comparisons.
    *   Ensure the output is manifold and watertight.

## Implementation Steps
1.  Define basic geometry classes (`Point3`, `Triangle`, `AABB`).
2.  Implement AABB tree construction and query.
3.  Implement triangle-triangle intersection and splitting.
4.  Implement point-in-mesh test using ray casting or winding numbers for classification.
5.  Assemble the final mesh and clean up (remove duplicate vertices).
6.  Output results in the specified format.

## Verification
*   Test with simple primitive overlaps (Cubes).
*   Test with high-resolution meshes to ensure performance.
