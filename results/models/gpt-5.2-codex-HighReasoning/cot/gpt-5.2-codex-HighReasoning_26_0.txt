**Designing one-pass streaming k-means**

The code must implement a one-pass streaming k-means that updates centroids online and outputs each point’s cluster assignment immediately, accepting that assignments reflect current centroids and not final ones, since no rereading is possible.**Designing adaptive k-means with streaming and batch modes**

Planning a hybrid approach: for small datasets (≤10M points), store all points, run full k-means with iterations and output final assignments. For larger data, run one-pass online k-means initializing centroids from first K points, then assign each new point to nearest centroid, update it incrementally, and output immediately. The code will include a custom buffered reader to parse mixed-text header and subsequent binary floats efficiently.**Implementing batch streaming output with centroid updates**

The plan is to assign points in batches: find nearest centroid per point, save assignments, update centroids after each batch via weighted averages, then output batch assignments efficiently. This balances output order correctness and centroid stability while scaling to large data sizes.
