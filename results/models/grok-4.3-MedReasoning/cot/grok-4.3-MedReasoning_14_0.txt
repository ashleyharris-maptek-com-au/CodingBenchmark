The task is to write a Minesweeper solver in C# that handles boards from small to massive like 10000x10000. It must find provably safe cells without guessing.
The task is to write a C# program that solves Minesweeper by finding all provably safe cells (unknown cells that must be safe based on the current board state) without guessing.

Input: H W, then H lines of the board.

Output: N, then N lines of x y (col row, 0-based) for safe cells.

Only output if provably safe, i.e., safe in all possible mine configurations consistent with the numbers and flags.

For efficiency on large boards (10k x 10k), need O(H*W) time or close, can't afford graph algorithms per component if many.

Basic local 1-step deduction: for each numbered cell, if remaining mines needed =0, mark all its unknown neighbors as safe.

If = number of unknowns, mark them as mines (but we don't need to output mines).

To get more, we can iterate this process: when we mark some as safe or mine, it affects neighbors' counts, so we need to propagate.

But marking safe means we know it's not mine, so for adjacent numbered cells, their f increases if unknown neighbor marked mine? No.

Let's define:

- Known mines: '*'

- Known safe: we will collect list of proven safe ' '

- Unknown: ' '

 When we deduce a cell is safe, we can treat it as known safe (not mine), so for constraints, it doesn't contribute to mine count.

  If deduce mine, treat as known mine.

To propagate:

Use a queue for propagation.

Initialize:

- Create a list or set of unknowns.

- Create a  2D array for deduced_mine or deduced_safe.

- Set deduced_mine where board has '*'

- Then, for each numbered cell, compute current remaining = num - known_mines_adj

- unknowns_adj = count of ' ' adj that not deduced_mine and not deduced_safe

- If remaining == 0, then all unknowns_adj must be safe, so mark them deduced_safe, and enqueue them if new.

- If remaining == unknowns_adj, mark all as deduced_mine, enqueue.

Then, when we mark new safe or mine, we need to update the numbered cells that are adjacent to them, because their f or rem changes.

So, better: use a queue of "dirty" numbered cells that need recalc.
