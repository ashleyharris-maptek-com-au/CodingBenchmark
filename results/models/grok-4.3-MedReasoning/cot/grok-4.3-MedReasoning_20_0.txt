The task is to write a Rust program for validating drillhole assay data, finding outliers that don't correlate with neighboring points.
The task is to write a Rust program that reads drillhole data from stdin and identifies suspect samples with typos or measurement errors based on not correlating with spatially neighboring data points. It needs to be efficient for large datasets, handle up to 200,000 holes, use only standard Rust library, and output the number of suspects followed by details sorted by confidence descending.

First, parse the input format:

- First line: num_holes num_properties

- Next line: property_name_1 property_name_2 ... (num_properties names)

- Then for each of num_holes:

  - hole_id start_x start_y start_z dir_x dir_y dir_z length num_samples

  - Then num_samples lines: depth prop1 prop2 ...

Depth is along the hole, position = start + normalized(dir) * depth.

But for neighbors, since "spatially neighboring", but with large scale, probably neighbors along the same hole (linear sequence).

I assume that: for each hole, samples are ordered along depth, so consecutive samples are neighbors.

For each property, treat the sequence of values along the hole as 1D sequence, find local outliers.

For efficiency, process per hole per property.

To compute confidence: for each sample i (not ends), compute how anomalous it is compared to neighbors.

Simple method:

- For i in 1..n-1:

  - expected = (v[i-1] + v[i+1]) / 2.0

  - residual = (v[i] - expected).abs()

  - To normalize, use a scale e.g. median absolute deviation (MAD) of residuals in the hole for that prop.

  - confidence = residual / (MAD + eps)

  - if confidence > threshold (say 3.0), flag it.

MAD robust to outliers.

For adaptive: if total samples small, use looser threshold, else 5.0.

Also, need to store the data: need hole_id, sample_idx (0-based), prop_name, conf.

Sample_idx: order in input, 0 to num_samples-1.

Holes may have different num_samples.
