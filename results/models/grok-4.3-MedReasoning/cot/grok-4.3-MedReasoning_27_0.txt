The task is to write a C++ program for graph coloring that reads from stdin and writes to stdout. It needs to handle graphs from 50 nodes to a million nodes, using up to 60 colors.

Input:

First line: num_vertices num_edges num_colors

Then num_edges lines of u v for edges.

 0-indexed.

Output: space-separated colors for each vertex from 0 to n-1.

It must be a valid coloring.

Adaptive logic based on graph complexity.

For large n, use greedy coloring with random order or degree ordering.

Since k up to 60, and chromatic number probably <=k.

For exact 0 to k-1.

I need to implement:

- Read n, m, k

- Read edges, build adj list, undirected I assume.

- Then, based on n:

if n small, say n<50, use backtracking to color with k colors.

else use greedy.

But n from  50 to 1e6, so for 50 perhaps exact.

But 50 is small for backtrack if k=60, but 60^50 huge, no.

Backtrack for coloring is exponential, for n=50 with k=3 maybe ok if dense, but not reliable.

So probably for 50 use greedy too, but adaptive: for small use exact if possible, but better always use good heuristic.

The problem says "sophisticated algorithms for larger instances", "complete within 5 minutes".

For million nodes, must be greedy or similar O(n+m) time.

I will implement greedy with degree ordering, and if fails (needs >k), try random permutations a few times.
