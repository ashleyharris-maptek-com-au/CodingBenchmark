I can’t reliably produce a valid SPIR-V binary hex string by hand. Below is a GLSL fragment shader that implements the heatmap you described using the available inputs and a uniform block. You can compile this with a SPIR-V toolchain (e.g., glslangValidator or glslc) to obtain the .spv binary and then convert it to a hex string if you need the exact SPIR-V hex.

GLSL fragment shader (heatmap based on N·V, with the specified inputs and UBO layout):

#version 450

layout(location = 0) in vec3 worldPos;
layout(location = 1) in vec3 normal;
layout(location = 2) in vec2 uv;
layout(location = 3) in vec3 tangent;
layout(location = 4) in vec3 color; // not used in calculation, but available

layout(set = 0, binding = 0) uniform UBO {
    mat4 model;
    mat4 view;
    mat4 projection;
    vec4 lightPos;
    vec4 cameraPos;
    vec4 params;
} ubo;

layout(location = 0) out vec4 outColor;

void main() {
    vec3 N = normalize(normal);
    vec3 V = normalize(ubo.cameraPos.xyz - worldPos);
    float ndotv = max(dot(N, V), 0.0);

    vec3 col;
    if (ndotv < 0.5) {
        col = vec3(1.0, 2.0 * ndotv, 0.0);
    } else {
        float t = (ndotv - 0.5) * 2.0;
        col = vec3(1.0 - t, 1.0 - t, t);
    }

    outColor = vec4(col, 1.0);
} 

How to obtain the SPIR-V hex string from this shader:
- Save the above as heatmap.frag.
- Compile to SPIR-V:
  glslangValidator -V heatmap.frag -o heatmap.frag.spv
- Convert the binary to a hex string (no spaces/newlines, all lowercase):
  xxd -p heatmap.frag.spv | tr -d '\n'
  or
  hexdump -ve '  %02x' heatmap.frag.spv | tr -d ' \n'
- The resulting long hex string is your SPIR-V binary blob.
