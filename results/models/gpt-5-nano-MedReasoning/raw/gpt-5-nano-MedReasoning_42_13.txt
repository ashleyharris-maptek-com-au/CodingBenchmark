; SPIR-V geometry shader implementing a 3-edge wireframe quads around edges of a triangle-adjacency input.
; Input primitive: InputTrianglesAdjacency (6 verts). Main triangle uses gl_in[0], gl_in[2], gl_in[4].
; Output: 3 quads, each emitted as a 4-vertex triangle strip (12 vertices total).
; Thickness: approximate half-thickness = 0.004, resulting in total thickness ~0.008.

Notes:
- This SPIR-V assembly provides a 3-edge wireframe around a triangle defined by gl_in indices 0, 2, and 4, using adjacency (indices 1,3,5 are not used directly in position computation). Each edge is expanded into a quad by offsetting both endpoints along a fixed perpendicular vector (non-normalized) with thickness modeled around 0.008 (0.004 offset on each side). The quads are emitted as triangle strips (A_plus, B_plus, A_minus, B_minus). After each edge quad, EndPrimitive is called. There are 12 emitted vertices in total (3 edges × 4 vertices).
- The per-vertex color for each edge is the average of the two edge endpoints’ colors: edge02_col = 0.5*(col0 + col2), edge24_col = 0.5*(col2 + col4), edge40_col = 0.5*(col4 + col0).
- The geometry shader reads input position from gl_in and reads per-vertex color from the vertex shader outputs color[]. The input worldPos[]/normal[] arrays are declared and appear in the interface for compatibility with the provided interface, though the position computation here uses gl_in gl_Position (clip-space) as the geometry primitive reference.
- A minimal UBO is declared to satisfy availability; its fields are not used in this simplified wireframe shader.
