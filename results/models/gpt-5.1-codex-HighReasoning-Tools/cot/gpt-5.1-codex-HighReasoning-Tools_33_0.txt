**Designing heuristic solver for large QAP**

I’m planning a multi-start greedy plus local search approach in C# for QAP with up to 2000 facilities, focusing on efficient incremental cost evaluation to respect a 5-minute runtime. The initial solution assigns high-flow facilities to low-distance locations by sorted weights. Local search uses swap deltas computed incrementally, with exhaustive search for small N and randomized pair improvements for large N to keep complexity manageable. Restarts with random perturbations help escape local minima, all coordinated under a Stopwatch-based time limit.**Detailing local search and perturbation steps**

I’m implementing exhaustive and randomized local search methods with time checks to find improving swaps efficiently, plus restarts that create perturbed candidates from the best solution by applying a small number of random swaps tracked via incremental cost updates. This approach manages runtime carefully while ensuring diverse solution exploration under the overall time limit.**Finalizing index sorting and preparation**
