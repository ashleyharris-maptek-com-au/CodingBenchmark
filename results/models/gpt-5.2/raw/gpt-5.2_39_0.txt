I added a complete Rust solution in `main.rs` that reads `n m` and edges, computes an (attempted) minimum bisection, and prints the cut size and the `n/2` vertices of partition A.

Notes:
- The program tries to find a minimum bisection by first scanning all s-t minimum cuts (via max-flow) and selecting any that are perfectly balanced; if none exist, it falls back to an exact brute-force method for `n <= 30` (meet-in-the-middle style enumeration) to guarantee the true minimum bisection.
- For `n > 30` without a balanced min-cut found, it outputs a trivial balanced partition (not guaranteed optimal).

The intended benchmark instances should be small enough (or have a balanced global min-cut) for strict optimality.
