# Design Document: 3D Boid Flocking Simulation Compute Shader

## 1. Objective
Implement a GLSL compute shader to simulate 3D boid flocking behavior. The simulation will update boid positions and velocities based on the three classic rules: separation, alignment, and cohesion.

## 2. Data Structures
The boid data will be stored in a Shader Storage Buffer Object (SSBO).

```glsl
struct Boid {
    vec3 position;
    vec3 velocity;
};

layout(std430, binding = 0) buffer BoidBuffer {
    Boid boids[];
};
```

## 3. Flocking Rules
Each boid's velocity will be updated based on the following rules:
- **Separation:** Steer to avoid crowding local flockmates.
- **Alignment:** Steer towards the average heading of local flockmates.
- **Cohesion:** Steer to move towards the average position of local flockmates.

## 4. Compute Shader Architecture
- **Workgroup Size:** `(256, 1, 1)` (or similar, depending on the number of boids).
- **Logic:**
    1.  Each thread calculates the new velocity for one boid.
    2.  Iterate through all other boids to find neighbors within a specific radius.
    3.  Accumulate forces for separation, alignment, and cohesion.
    4.  Apply forces to the boid's velocity.
    5.  Update position based on the new velocity.
    6.  Handle boundary conditions (e.g., wrap-around or bounce).

## 5. Implementation Steps
1.  Define the SSBO structure in the shader.
2.  Implement the main compute shader logic (`main` function).
3.  Implement helper functions for the three rules.
4.  Add boundary handling logic.
5.  (Optional) Add a simple test/verification script to ensure the shader compiles and runs.

## 6. Verification & Testing
- **Compilation:** Verify the shader compiles without errors.
- **Logic Check:** Ensure boids move and exhibit flocking behavior (visual inspection or debug output).
