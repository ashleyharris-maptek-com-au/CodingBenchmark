The task was to implement a C# solver for the Chinese Postman Problem (Route Inspection Problem) that is efficient enough to handle graphs of "Epic" scale—up to 100,000 nodes and 1,000,000 edges—within a 30-second time limit.

### Strategic Approach

1.  **Parity Correction (Minimum Weight Matching Heuristic):** The core of the Chinese Postman Problem is making a graph Eulerian by adding paths between nodes with odd degrees such that the total added weight is minimized. For large-scale graphs where exact minimum weight matching ($O(V^3)$) is computationally prohibitive, a common and effective heuristic is to use edges from the Minimum Spanning Tree (MST).
    *   I used **Kruskal's algorithm** with a Union-Find (DSU) data structure to find the MST.
    *   I then performed a **bottom-up pass on the spanning forest** to identify which tree edges should be duplicated to ensure every node in the graph has an even degree. This step is $O(M \log M)$ due to sorting and $O(N)$ for the traversal.

2.  **Eulerian Circuit Discovery:** Once the graph is made Eulerian (all nodes have even degrees), an Eulerian circuit must be found that visits every edge in the multigraph exactly once.
    *   I implemented an **iterative version of Hierholzer's algorithm**. The iterative approach is crucial for large graphs to avoid `StackOverflowException` from deep recursion.
    *   The algorithm runs in $O(M')$ where $M'$ is the number of edges in the multigraph ($M' \le M + N - 1$).

3.  **Performance Optimization for "Epic" Scale:**
    *   **Fast I/O:** Reading 1,000,000 lines and printing 1,100,000 integers can be a bottleneck. I implemented a custom `FastReader` using a buffered `Stream` and used a `StreamWriter` with a 1MB buffer for output.
    *   **Memory Efficiency:** All data structures (adjacency lists, DSU, stacks) were implemented using primitive arrays and optimized collections to minimize overhead and garbage collection pressure, fitting well within the 32GB RAM limit.

### Technical Details

*   **Platform:** .NET 10.0 (C# 14).
*   **Time Complexity:** $O(M \log M)$ for sorting edges, $O(M \alpha(N))$ for DSU, and $O(M)$ for circuit finding.
*   **Space Complexity:** $O(N + M)$ to store the graph and auxiliary structures.

The final code is contained in `Program.cs` and is designed to be fully compilable and functional across all specified scales, from trivial to epic.
