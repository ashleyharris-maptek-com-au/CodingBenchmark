To solve this problem, we need to find the minimum vertex cover of a given graph. The vertex cover problem is NP-hard, meaning that for large graphs, exact solutions are computationally expensive. However, we can use a heuristic approach to find a near-optimal solution efficiently. One effective heuristic is the greedy algorithm that selects vertices based on their degree, prioritizing vertices with higher degrees first. This approach provides a 2-approximation for the minimum vertex cover problem.

### Approach
1. **Graph Representation**: We will represent the graph using an adjacency list where each vertex points to its neighbors.
2. **Greedy Algorithm**: The algorithm works by iteratively selecting the vertex with the highest degree, adding it to the vertex cover, and then removing all edges incident to this vertex. This process continues until all edges are covered.
3. **Efficiency**: The algorithm runs in O(E log V) time due to the need to repeatedly find the vertex with the highest degree, which can be optimized using a max-heap (priority queue). However, for simplicity and given the constraints, we can use a frequency map to track the degrees of vertices and update them as edges are removed.

### Solution Code
```rust
use std::collections::{HashMap, HashSet, BinaryHeap};
use std::io::{self, BufRead};

fn main() {
    let stdin = io::stdin();
    let mut lines = stdin.lock().lines();
    
    let first_line = lines.next().unwrap().unwrap();
    let parts: Vec<&str> = first_line.split_whitespace().collect();
    let num_vertices: usize = parts[0].parse().unwrap();
    let num_edges: usize = parts[1].parse().unwrap();
    
    let mut adj: Vec<HashSet<usize>> = vec![HashSet::new(); num_vertices];
    let mut degree = vec![0; num_vertices];
    
    for _ in 0..num_edges {
        let line = lines.next().unwrap().unwrap();
        let parts: Vec<&str> = line.split_whitespace().collect();
        let u: usize = parts[0].parse().unwrap();
        let v: usize = parts[1].parse().unwrap();
        adj[u].insert(v);
        adj[v].insert(u);
        degree[u] += 1;
        degree[v] += 1;
    }
    
    let mut max_heap = BinaryHeap::new();
    for i in 0..num_vertices {
        max_heap.push((degree[i], i));
    }
    
    let mut vertex_cover = HashSet::new();
    let mut covered_edges = HashSet::new();
    
    while let Some((_, u)) = max_heap.pop() {
        if adj[u].is_empty() {
            continue;
        }
        vertex_cover.insert(u);
        for &v in adj[u].iter() {
            if !covered_edges.contains(&(u, v)) && !covered_edges.contains(&(v, u)) {
                covered_edges.insert((u, v));
                // Update degrees of neighbors
                if let Some(pos) = max_heap.iter().position(|&(_, idx)| idx == v) {
                    let (_, idx) = max_heap.iter().nth(pos).unwrap();
                    if idx == &v {
                        // Remove the old entry and reinsert with updated degree
                        let mut temp_heap = BinaryHeap::new();
                        for (d, i) in max_heap.iter() {
                            if i == &v {
                                temp_heap.push((degree[v] - 1, v));
                            } else {
                                temp_heap.push((*d, *i));
                            }
                        }
                        max_heap = temp_heap;
                    }
                }
                adj[v].remove(&u);
                degree[v] -= 1;
            }
        }
        adj[u].clear();
    }
    
    let mut result: Vec<usize> = vertex_cover.into_iter().collect();
    result.sort();
    
    println!("{}", result.len());
    for (i, &v) in result.iter().enumerate() {
        if i > 0 {
            print!(" ");
        }
        print!("{}", v);
    }
    println!();
}
```

### Explanation
1. **Graph Representation**: The graph is stored using an adjacency list where each vertex has a set of its neighbors. This allows efficient edge removal during the algorithm.
2. **Degree Tracking**: The degree of each vertex is tracked to prioritize vertices with higher degrees. A max-heap (priority queue) is used to always select the vertex with the highest degree.
3. **Greedy Selection**: The algorithm starts by selecting the vertex with the highest degree, adding it to the vertex cover, and then removing all edges incident to this vertex. This ensures that the selected vertex covers as many edges as possible, reducing the problem size iteratively.
4. **Edge Removal**: For each selected vertex, all edges connected to it are marked as covered. The degrees of its neighbors are updated, and the heap is adjusted to reflect these changes.
5. **Output**: The vertices in the vertex cover are collected, sorted, and printed in the required format.

This approach efficiently handles large graphs by leveraging a greedy heuristic that provides a near-optimal solution in a reasonable time frame, making it suitable for the given constraints.
