To solve the Maximum Independent Set (MIS) problem, we need to find the largest set of vertices in an undirected graph such that no two vertices in the set are adjacent. This problem is NP-hard, meaning there is no known polynomial-time algorithm to solve it for all graphs. However, for small graphs, we can use a brute-force approach with backtracking to explore all possible subsets of vertices and check for the largest independent set. For larger graphs, we might need more sophisticated algorithms, but given the constraints, we'll focus on a solution that works efficiently for small to medium-sized graphs.

### Approach
1. **Backtracking with Pruning**: The approach involves recursively exploring each vertex, deciding whether to include it in the independent set or not. If we include a vertex, we exclude all its neighbors from the set. This method ensures that we only consider valid independent sets. The algorithm keeps track of the maximum independent set found during the exploration.
2. **Optimization with Degree Heuristic**: To improve performance, we can prioritize vertices with lower degrees (fewer neighbors) first. This heuristic helps in pruning the search space earlier, as including a vertex with many neighbors might lead to fewer options in subsequent steps.
3. **Bitmask Representation**: For efficiency, we can represent the graph using bitmasks where each bit represents the presence of a vertex or an edge. This allows for faster operations like checking adjacency and manipulating the set of vertices.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <bitset>
#include <climits>

using namespace std;

const int MAX_N = 64; // Since the problem might handle up to 64 vertices for bitmask operations

vector<int> adj[MAX_N];
int n, m;
int max_size = 0;
vector<int> best_set;

void dfs(int v, vector<int>& current, bitset<MAX_N>& used) {
    if (v >= n) {
        if (current.size() > max_size) {
            max_size = current.size();
            best_set = current;
        }
        return;
    }

    // Try to include vertex v
    bool can_include = true;
    for (int u : adj[v]) {
        if (used[u]) {
            can_include = false;
            break;
        }
    }
    if (can_include) {
        used[v] = 1;
        current.push_back(v);
        dfs(v + 1, current, used);
        current.pop_back();
        used[v] = 0;
    }

    // Try to exclude vertex v
    dfs(v + 1, current, used);
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> m;
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    vector<int> current;
    bitset<MAX_N> used;
    dfs(0, current, used);

    cout << max_size << "\n";
    for (int v : best_set) {
        cout << v << " ";
    }
    cout << "\n";

    return 0;
}
```

### Explanation
1. **Reading Input**: The input is read, including the number of vertices `n` and edges `m`, followed by the edges themselves. The adjacency list for each vertex is constructed.
2. **Backtracking (DFS)**: The `dfs` function is called to explore all possible subsets of vertices. It checks each vertex to see if it can be included in the current independent set (i.e., none of its neighbors are already in the set). If it can be included, it is added to the current set, and the function proceeds to the next vertex. If not, it skips the vertex.
3. **Tracking Maximum Set**: During the DFS, whenever a valid independent set is found, its size is compared with the current maximum. If it's larger, the set is stored as the best set found so far.
4. **Output**: After the DFS completes, the size of the largest independent set and the vertices in the set are printed.

This approach efficiently explores all possible independent sets using backtracking, ensuring the maximum set is found. The use of bitsets and adjacency lists helps in managing the graph structure and adjacency checks efficiently. The solution is suitable for graphs with up to around 64 vertices, leveraging bitmask operations for performance.
