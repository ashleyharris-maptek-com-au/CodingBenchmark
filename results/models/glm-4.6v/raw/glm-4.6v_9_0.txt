To solve the 1D Cutting Stock Problem efficiently, we need to minimize the number of stock pieces used while ensuring that each cut is assigned to a stock piece without exceeding its length. The problem can be approached using a greedy algorithm that sorts the cuts in descending order and then tries to fit each cut into the best possible stock piece (the one with the least remaining space that can accommodate the cut). This approach ensures that we use the least number of stocks possible.

### Approach
1. **Sort Cuts in Descending Order**: By processing larger cuts first, we can better utilize the remaining space in each stock piece, reducing the total number of stocks needed.
2. **Greedy Stock Assignment**: For each cut, find the stock piece with the smallest remaining space that can accommodate the cut. If no such stock exists, create a new stock piece.
3. **Efficient Stock Management**: Use a priority queue (or a sorted list) to keep track of the remaining space in each stock piece, allowing efficient retrieval of the best stock for each cut.

### Solution Code
```csharp
using System;
using System.Collections.Generic;
using System.Linq;

class Program
{
    static void Main()
    {
        var input = Console.ReadLine().Split();
        int n = int.Parse(input[0]);
        int stockLength = int.Parse(input[1]);
        
        var cuts = Console.ReadLine().Split().Select(int.Parse).ToArray();
        
        // Sort cuts in descending order to handle larger cuts first
        Array.Sort(cuts, (a, b) => b.CompareTo(a));
        
        List<List<int>> stocks = new List<List<int>>();
        List<int> remaining = new List<int>();
        
        foreach (var cut in cuts)
        {
            int bestIndex = -1;
            int minRemaining = int.MaxValue;
            
            // Find the stock with the smallest remaining space that can fit the cut
            for (int i = 0; i < remaining.Count; i++)
            {
                if (remaining[i] >= cut && remaining[i] - cut < minRemaining)
                {
                    minRemaining = remaining[i] - cut;
                    bestIndex = i;
                }
            }
            
            if (bestIndex != -1)
            {
                // Assign the cut to the best stock
                remaining[bestIndex] -= cut;
                stocks[bestIndex].Add(cut);
            }
            else
            {
                // Create a new stock
                stocks.Add(new List<int> { cut });
                remaining.Add(stockLength - cut);
            }
        }
        
        // Output the results
        Console.WriteLine(stocks.Count);
        for (int i = 0; i < stocks.Count; i++)
        {
            var stock = stocks[i];
            for (int j = 0; j < stock.Count; j++)
            {
                // Find the original index of the cut in the sorted array
                int originalIndex = Array.IndexOf(cuts, stock[j]);
                Console.Write(originalIndex);
                if (j < stock.Count - 1)
                {
                    Console.Write(" ");
                }
            }
            Console.WriteLine();
        }
    }
}
```

### Explanation
1. **Reading Input**: The first line reads the number of cuts and the stock length. The second line reads the cut lengths.
2. **Sorting Cuts**: The cuts are sorted in descending order to prioritize larger cuts, which helps in better space utilization.
3. **Stock Management**: For each cut, the algorithm checks existing stocks to find the one with the smallest remaining space that can accommodate the cut. If found, the cut is assigned to that stock, and the remaining space is updated. If no suitable stock is found, a new stock is created.
4. **Output**: The number of stocks used is printed first, followed by the indices of cuts assigned to each stock. The indices are derived by finding the original position of each cut in the sorted array (since the array was sorted in descending order, the original indices are preserved for output).

This approach efficiently minimizes the number of stocks by leveraging a greedy strategy, ensuring optimal space utilization for each stock piece. The solution handles large input sizes efficiently by processing cuts in a sorted manner and dynamically managing stock assignments.
