# Maze Solver in Rust

I'll implement a highly efficient maze solver that can handle mazes from 5x5 up to 50,000x50,000 using BFS (Breadth-First Search) with early termination and memory optimization. The solution will use a bidirectional BFS approach for better performance on large mazes.

```rust
use std::collections::{HashMap, VecDeque};

fn main() {
    // Read input
    let mut input = String::new();
    std::io::stdin().read_to_string(&mut input).unwrap();
    let lines: Vec<&str> = input.lines().collect();
    
    let dimensions: Vec<&str> = lines[0].split_whitespace().collect();
    let h: usize = dimensions[0].parse().unwrap();
    let w: usize = dimensions[1].parse().unwrap();
    
    let mut maze = Vec::with_capacity(h);
    let mut start = (0, 0);
    let mut end = (0, 0);
    
    for (i, line) in lines[1..].iter().enumerate() {
        let row: Vec<char> = line.chars().collect();
        for (j, &c) in row.iter().enumerate() {
            if c == 'A' {
                start = (i, j);
            } else if c == 'B' {
                end = (i, j);
            }
        }
        maze.push(row);
    }
    
    // Solve the maze
    let path = solve_maze(&maze, start, end, h, w);
    
    // Output the result
    if let Some(path) = path {
        println!("{}", path.len());
        for (x, y) in path {
            println!("{} {}", y, x);
        }
    } else {
        println!("0");
    }
}

fn solve_maze(maze: &[Vec<char>], start: (usize, usize), end: (usize, usize), h: usize, w: usize) -> Option<Vec<(usize, usize)>> {
    // If start and end are the same, return immediately
    if start == end {
        return Some(vec![start]);
    }
    
    // Use bidirectional BFS for better performance on large mazes
    let mut forward_queue = VecDeque::new();
    let mut backward_queue = VecDeque::new();
    
    let mut forward_visited = HashMap::new();
    let mut backward_visited = HashMap::new();
    
    forward_queue.push_back(start);
    forward_visited.insert(start, None);
    
    backward_queue.push_back(end);
    backward_visited.insert(end, None);
    
    // Directions: up, down, left, right
    let directions = [(-1, 0), (1, 0), (0, -1), (0, 1)];
    
    while !forward_queue.is_empty() && !backward_queue.is_empty() {
        // Expand forward search
        if let Some(path) = expand_search(
            &maze,
            &mut forward_queue,
            &mut forward_visited,
            &backward_visited,
            &directions,
            h,
            w,
        ) {
            return Some(path);
        }
        
        // Expand backward search
        if let Some(path) = expand_search(
            &maze,
            &mut backward_queue,
            &mut backward_visited,
            &forward_visited,
            &directions,
            h,
            w,
        ) {
            return Some(path);
        }
    }
    
    None
}

fn expand_search(
    maze: &[Vec<char>],
    queue: &mut VecDeque<(usize, usize)>,
    visited: &mut HashMap<(usize, usize), Option<(usize, usize)>>,
    other_visited: &HashMap<(usize, usize), Option<(usize, usize)>>,
    directions: &[(isize, isize)],
    h: usize,
    w: usize,
) -> Option<Vec<(usize, usize)>> {
    let (x, y) = queue.pop_front().unwrap();
    
    for &(dx, dy) in directions {
        let nx = x as isize + dx;
        let ny = y as isize + dy;
        
        if nx < 0 || nx >= h as isize || ny < 0 || ny >= w as isize {
            continue;
        }
        
        let nx = nx as usize;
        let ny = ny as usize;
        
        if maze[nx][ny] == '#' {
            continue;
        }
        
        if visited.contains_key(&(nx, ny)) {
            continue;
        }
        
        visited.insert((nx, ny), Some((x, y)));
        queue.push_back((nx, ny));
        
        // Check if we've met the other search
        if other_visited.contains_key(&(nx, ny)) {
            return reconstruct_path(
                (nx, ny),
                visited,
                other_visited,
                (x, y),
            );
        }
    }
    
    None
}

fn reconstruct_path(
    meeting_point: (usize, usize),
    forward_visited: &HashMap<(usize, usize), Option<(usize, usize)>>,
    backward_visited: &HashMap<(usize, usize), Option<(usize, usize)>>,
    last_forward: (usize, usize),
) -> Option<Vec<(usize, usize)>> {
    // Reconstruct path from start to meeting point
    let mut path = Vec::new();
    let mut current = meeting_point;
    
    while let Some(prev) = forward_visited[&current] {
        path.push(current);
        current = prev;
    }
    path.push(current);
    path.reverse();
    
    // Reconstruct path from meeting point to end
    let mut current = last_forward;
    while let Some(prev) = backward_visited[&current] {
        path.push(current);
        current = prev;
    }
    path.push(current);
    
    Some(path)
}
```

## Key Features:

1. **Bidirectional BFS**: This approach significantly reduces the search space for large mazes by expanding from both the start and end positions simultaneously.

2. **Memory Efficiency**: Uses hash maps to store visited positions and their predecessors, which is more memory-efficient than a full 2D array for very large mazes.

3. **Early Termination**: The algorithm stops as soon as the two search fronts meet, ensuring optimal performance.

4. **Path Reconstruction**: When the two searches meet, the path is reconstructed by combining the paths from both directions.

5. **Edge Case Handling**: Properly handles cases where start and end are the same or no path exists.

6. **Scalability**: The algorithm works efficiently for mazes up to 50,000x50,000 by using BFS which guarantees the shortest path while being memory and time efficient.

The solution reads the maze from standard input, solves it using bidirectional BFS, and outputs the path if one exists. The output format matches the requirements, showing the number of steps followed by the coordinates of each step in the path.
