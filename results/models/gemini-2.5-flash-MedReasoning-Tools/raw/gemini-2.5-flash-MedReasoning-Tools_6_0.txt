The C# solver is implemented in `Program.cs`.

**Key Features:**
- **Hybrid TSP Solver:** For a small number of stations (N <= 10), it uses a brute-force approach to find the optimal permutation. For larger numbers of stations, it employs a greedy Nearest Neighbor heuristic followed by a 2-Opt local search improvement, balancing accuracy and performance.
- **Simplified Orbital Mechanics:** The `CalculateTransferCost` function provides "reasonable delta-V estimates" by approximating transfers based on changes in semi-major axis (energy) and inclination. This simplified model prioritizes computational efficiency required for large N (up to 100 stations) over a full, iterative Lambert solver. Penalties are included for position and velocity mismatch to encourage rendezvous.
- **Robust Input Handling:** The `Program` class handles input parsing and constructs the cost matrix for the TSP solver.
- **Output Format:** The program outputs a single line of N integers representing the visitation order of stations (0 to N-1), as specified.

**Code Structure:**
- `Vector3D` struct for double-precision 3D vector operations.
- `OrbitalState` struct to store position and velocity vectors.
- `OrbitalMechanics` static class containing the `CalculateTransferCost` method.
- `TSPSolver` class with methods for brute-force, Nearest Neighbor, 2-Opt, permutation generation, and path cost calculation.
- `Program` class for main execution logic, input/output, and orchestrating the TSP solution.

The solution successfully compiles without warnings and runs with provided example input, producing the expected output format. The heuristic orbital mechanics and TSP algorithms are designed to meet the performance and scalability requirements for up to 100 stations within the given time limits.
