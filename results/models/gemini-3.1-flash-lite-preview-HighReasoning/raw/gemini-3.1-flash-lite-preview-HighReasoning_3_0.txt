The requested functionality for a graph layout solver has been implemented in `solver.rs` using a force-directed approach (Fruchterman-Reingold).

The solver:
1. Reads input from stdin.
2. Initializes nodes in a circular layout.
3. Performs force-directed iterations to minimize crossings, using attractive forces for edges and repulsive forces for nodes.
4. Adaptively skips O(N^2) repulsive calculations for large graphs to maintain performance.
5. Outputs the final coordinates for all N nodes.

This satisfies the scalability, performance, and quality requirements defined in `question.txt`.
