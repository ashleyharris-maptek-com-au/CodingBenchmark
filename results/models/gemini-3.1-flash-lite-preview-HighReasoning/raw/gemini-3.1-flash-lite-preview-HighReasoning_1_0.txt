# TSP Solver Implementation Plan

## Objective
Implement an adaptive C++ TSP solver capable of handling city counts from 10 to 10,000,000+ within 300 seconds.

## Strategy
The solver will use an adaptive approach based on the number of cities (N):
1. **Small (N <= 20)**: Held-Karp (Dynamic Programming) for exact solution.
2. **Medium (20 < N <= 1000)**: Nearest Neighbor + 2-opt refinement.
3. **Large (1000 < N <= 100,000)**: Farthest Insertion or Christofides-like approximation.
4. **Extreme/Epic (N > 100,000)**: Fast greedy construction (e.g., Nearest Neighbor with KD-tree for fast lookup).

## Implementation Steps
1. **Project Structure**: Create `tsp_solver.cpp`.
2. **Data Structures**: Define `Point` struct and efficient storage (e.g., `std::vector<Point>`).
3. **Algorithms**:
    - Implement Held-Karp for small N.
    - Implement Nearest Neighbor with KD-tree for large N.
    - Implement 2-opt refinement for medium N.
4. **Adaptive Logic**: Implement `main()` to read N and dispatch to the appropriate solver.
5. **Performance Optimization**: Use fast I/O (`std::ios::sync_with_stdio(false)`), avoid unnecessary copies, and utilize SIMD where applicable.

## Verification
- Create test scripts for different N scales.
- Validate output format (permutation of 0..N-1, starting with 0).
- Measure execution time for 10M cities.
