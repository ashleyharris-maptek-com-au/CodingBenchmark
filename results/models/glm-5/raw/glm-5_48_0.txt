**Correction on OpSMod types:**
SPIR-V `OpSMod` works on integers.
`%xm1_int = OpISub %int_t %x_int %int_1` works.
`OpSMod` expects integer types.
Correct sequence for x-1 (uint):
`%x_int = OpConvertUToF %int_t %x` (Error in previous thought: OpConvertUToF converts to float. Correct instruction is `OpConvertUToF`? No. `OpConvertUToF` converts uint to float. We need `OpConvertUToS` or just bitcast/reinterpret if we are careful, but SPIR-V has `OpConvertUToS` for unsigned to signed, or `OpBitcast`).
Actually, since `x` is a coordinate, `OpConvertUToF` is wrong.
Use `OpConvertUToS` or logical handling.
Logical handling without conversion:
If x > 0, x-1.
If x == 0, x-1 wraps to W-1.
This is safer and pure uint.

Revised Streaming Logic for SPIR-V:
k=1 (x+1):
`%xp1 = OpIAdd %uint %x %uint_1`
`%cond_p1 = OpUGreaterThanEqual %bool %xp1 %W`
`%xp1_corr = OpISub %uint %xp1 %W`
`%xp1_final = OpSelect %uint %cond_p1 %xp1_corr %xp1`

k=3 (x-1):
`%cond_z = OpIEqual %bool %x %uint_0`
`%xm1_std = OpISub %uint %x %uint_1`
`%xm1_corr = OpISub %uint %W %uint_1`
`%xm1_final = OpSelect %uint %cond_z %xm1_corr %xm1_std`

k=2 (y+1): Same as x+1.
k=4 (y-1): Same as x-1.

This is cleaner and avoids type conversions.
