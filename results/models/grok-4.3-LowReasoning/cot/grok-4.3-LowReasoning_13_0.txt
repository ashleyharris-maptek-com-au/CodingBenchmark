The task is to write a C++ solver for the Longest Common Substring problem for multiple strings. It needs to handle from small to extremely large inputs, up to 100 strings each 10MB+.
The problem is Longest Common Substring for N strings, find the longest one that appears in all.

Input: first line N, then N strings.

Output: one longest common substring, or empty if none.

Need to handle large cases, up to 100 strings of 10MB each.

Time limit 300s 5min, RAM 128GB.

C++ with MSVC, C++20.

Adaptive: but for simplicity, since scale varies, a general efficient algo should work.

I think binary search on length L, with rolling hash to check existence of common substr of len L.

To find max L, binary search L 0 to min_len.

For each mid, check if exists common substr len mid.

To check: use rolling hash double, prefix hash for each string.

Then for a L, collect hashes from str[0] all windows, put in unordered_set (use pair<uint64,uint64> for double hash).
