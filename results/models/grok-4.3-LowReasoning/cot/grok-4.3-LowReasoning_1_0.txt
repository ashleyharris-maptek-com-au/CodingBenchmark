The task is to write a C++ solver for the Traveling Salesman Problem that handles from 10 to 10 million cities. That's ambitious.
The problem is to solve TSP for up to 10M cities in C++, needing an adaptive algorithm that works for all scales from 10 to 10M cities, finishing in 300s even for largest.

TSP is NP-hard, so for large N, must use heuristics for approximation, but output a valid permutation 0 to N-1 starting with 0.

Input: N, then N lines x y (floats), cities indexed 0 to N-1, 0 is start.

Output: one line with N integers, visit order starting with 0, permutation.

Must be correct permutation, but route reasonable.
