The task is to write a C++ solver for the Job-Shop Scheduling problem. It needs to handle from small to very large instances: up to 1000 jobs, 50 machines, 50k tasks.
The problem is Job-Shop Scheduling, which is NP-hard, so for large instances, I need a heuristic to find a feasible schedule, not necessarily optimal.

I need to write a C++ program that reads from stdin the input format and outputs the schedule in the specified format.

It must handle up to 1000 jobs, 50 machines, 50k tasks, so efficiency is important, O(N log N) max.

I'll use a simple greedy approach: process jobs in order, for each job, schedule its tasks one by one at the earliest possible time given the machine's availability and the job's previous completion time.
