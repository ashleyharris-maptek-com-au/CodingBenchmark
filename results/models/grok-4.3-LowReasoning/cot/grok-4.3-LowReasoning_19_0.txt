The task is to write a C# program for a mining problem in a 3D voxel space. It's complex, involving digging, moving dirt with constraints, and maximizing profit.
The problem is to write a C# program for this mining simulation. But it's extremely complex. The world can be up to 5000x5000x5000, which is impossible to simulate directly due to memory and time constraints.

First, parse the input: read the header with XVoxelCount etc., then 1 byte per voxel with mass and value encoded.

For output, need to list operations: dig and move paths to plant at (0,0) or dump elsewhere.

But for large grids, can't store the entire grid.
