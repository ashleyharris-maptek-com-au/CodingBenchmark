The task is to write a C++ controller for a 3D Lunar Lander game that works across scales from 100m to 100km. It must handle real-time input/output without stalling.
First, the code needs to read the initial header: width depth height gravity max_thrust fuel, then start_x start_y start_z target_x target_y target_z, then the terrain file path on its own line.

Then, parse the terrain file in background if large.

Then, read continuous states from stdin: x y  z  vx  vy  vz  pitch  yaw  roll  pitch_rate  yaw_rate  roll_rate  fuel_remaining

Output thrust pitch_cmd yaw_cmd roll_cmd at >=10Hz.

Use multithreading for reading stdin without blocking.
