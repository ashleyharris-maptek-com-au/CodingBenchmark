I'll implement a greedy heuristic for Minimum Dominating Set.

Algorithm:

- Build adjacency list: vec of vecs for neighbors.

- Use a boolean array for dominated: initially false.

- Use a set or something for candidates.

Simple greedy:

- Initialize dominated = false for all.

-  While not all dominated:

-  For each vertex i not dominated, compute score = 1 + num neighbors not dominated.

-  Pick the one with max score.

-  Mark it and its neighbors as dominated.

But this is O(n^2) per iteration worst n iterations, so O(n^3) time, bad for large n.

If n large, say 1e5, too slow.

But problem doesn't specify n, but with 5min, perhaps n small.

To make efficient, use a priority queue or approximate.

For now, assume n small, use the degree greedy.

I'll go with:

- Build adj list, sort and dedup.

- Compute degrees.

- Sort vertices by degree desc.

- Then iterate in that order, add if not dominated, mark it and neighbors.

This is fast: O(n log n + m)
