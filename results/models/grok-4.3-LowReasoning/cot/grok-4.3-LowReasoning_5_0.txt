The problem is to write a Rust solver for Hamiltonian Path on 2D grids with obstacles. It needs to handle grids up to 2500 cells within 5 minutes, and be correct.

Hamiltonian path is NP-hard, so backtracking would be too slow for large grids. But it says "any grid size from trivial to ludicrous scale" and "adapt based on grid size and obstacle density", and "complete within 5 minutes even for 2500 cell grids".

For 2500 cells, standard backtracking is impossible unless the density or structure allows it to prune quickly.
