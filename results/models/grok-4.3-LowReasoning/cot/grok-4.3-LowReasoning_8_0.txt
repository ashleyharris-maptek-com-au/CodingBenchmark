The task is to write a maze solver in Rust that handles mazes from 5x5 up to 50,000x50,000 efficiently.
The task is to write a complete, compilable Rust program that solves the maze problem as specified, handling mazes up to 50,000 x 50,000 efficiently.

Key requirements:
- Read H W from stdin
- Read H lines of W characters each for the maze
- Find path from 'A to 'B' using up/down/left/right moves, avoiding '#'
- Output N (length of path including A and B)
- Then N lines of x y (0-based, row col, top-left is 0,0)
- If no path, output 0
- Use BFS for shortest path, 0-indexed
- Path continuous, no loops, any path ok if multiple

For large sizes, need efficient memory use.
- Use Vec<Vec<char>> for maze or Vec<String>
- For BFS, use VecDeque of positions (usize,usize)
- For parent, use a flat Vec<Option<(usize, usize)>> or 2D Vec<Option<(usize,usize)>> but memory concern for 50k*50k
- 2D Vec<Vec<Option<(usize,usize)>>> has overhead, but for 50k, usize is 8 bytes, Option ~16 bytes, 2.5e9 * 16 ~40GB, plus grid 2.5GB, total ~50GB, with 128GB available, should be ok.
