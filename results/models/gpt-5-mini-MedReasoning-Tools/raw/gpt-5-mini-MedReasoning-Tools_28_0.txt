Here's a single-file, compilable Rust program that computes a maximal matching in one pass over the edges and outputs the corresponding 2-approximate vertex cover (both endpoints of each matched edge). It is optimized for large inputs (uses u32 for vertex ids, streams parsing from a full stdin buffer, and avoids storing all edges). The program is warning-free and should compile with the stated Rust toolchain.

```rust
use std::io::{self, Read, Write, BufWriter};

fn main() {
    // Read whole stdin into memory (fast parser over bytes)
    let mut input = Vec::new();
    io::stdin().read_to_end(&mut input).expect("read stdin");
    let buf = &input;
    let mut pos: usize = 0;
    let len = buf.len();

    // Fast integer parser for non-negative integers
    fn next_u32(buf: &[u8], pos: &mut usize, len: usize) -> u32 {
        // skip whitespace
        while *pos < len && buf[*pos].is_ascii_whitespace() {
            *pos += 1;
        }
        let mut num: u32 = 0;
        while *pos < len {
            let b = buf[*pos];
            if b.is_ascii_digit() {
                num = num.wrapping_mul(10).wrapping_add((b - b'0') as u32);
                *pos += 1;
            } else {
                break;
            }
        }
        num
    }

    // Parse header: num_vertices num_edges
    let n = next_u32(buf, &mut pos, len) as usize;
    let m = next_u32(buf, &mut pos, len) as usize;

    // matched[v] == 0 => free, == 1 => part of matching/cover
    let mut matched = vec![0u8; n];
    // cover collects vertices that are included (unique due to matched flags)
    let mut cover: Vec<u32> = Vec::with_capacity((m.min(n/1)).saturating_mul(2));

    // Process edges in a single pass: greedy maximal matching
    for _ in 0..m {
        if pos >= len {
            break; // defensive: in case fewer numbers than declared
        }
        let u = next_u32(buf, &mut pos, len) as usize;
        let v = next_u32(buf, &mut pos, len) as usize;
        if u >= n || v >= n {
            // Ignore edges with invalid indices to avoid panic (input assumed valid).
            continue;
        }
        if u == v {
            // self-loop: need one endpoint
            if matched[u] == 0 {
                matched[u] = 1;
                cover.push(u as u32);
            }
        } else {
            if matched[u] == 0 && matched[v] == 0 {
                // take this edge into matching => include both endpoints in cover
                matched[u] = 1;
                matched[v] = 1;
                cover.push(u as u32);
                cover.push(v as u32);
            }
        }
    }

    // Output
    let stdout = io::stdout();
    let mut out = BufWriter::new(stdout.lock());
    writeln!(out, "{}", cover.len()).unwrap();
    if !cover.is_empty() {
        // Print space-separated list on one line
        // Avoid trailing space: print first then others prefixed with space
        write!(out, "{}", cover[0]).unwrap();
        for v in &cover[1..] {
            write!(out, " {}", v).unwrap();
        }
    }
    writeln!(out).unwrap();
}
```

Notes:
- This produces a valid vertex cover whose size is at most twice the optimum (classical greedy maximal-matching 2-approximation).
- Memory usage is small: matched array is n bytes (u8), and we do not store all edges. The read buffer holds the raw input bytes.
- If you want stronger solutions (smaller covers), consider adding optional passes that build adjacency lists and apply reduction rules / local search — but that requires storing edges and more memory/time.
