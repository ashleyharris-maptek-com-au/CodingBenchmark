The user wants me to write a complete HLSL pixel shader for Vulkan. Let me analyze the requirements:

1. It's a fragment shader that visualizes surface normal as color
2. The input includes:
   - worldPos (float3)
   - normal (float3) - need to normalize
   - uv (float2)
   - tangent (float3)
   - color (float3) - varying per vertex with rgb = 0.5 + 0.5*xyz

3. Constant buffer with:
   - model matrix (column-major)
   - view matrix
   - projection matrix
   - lightPos (5.0, 5.0, 5.0, 1.0)
   - cameraPos (0.0, 0.0, 3.0, 1.0)
   - params (0.0, 0.5, 0.0, 0.0)

4. The task: Visualize surface normal as color by mapping normalized normal from [-1, 1] to [0, 1] and output with alpha 1.0

So the shader should:
1. Normalize the input normal
2. Map from [-1, 1] to [0, 1] (normal * 0.5 + 0.5)
3. Output as float4 with alpha 1.0

The constant buffer is provided but not actually used for the normal visualization task. The worldPos, tangent, color inputs are also not used.

Let me write the complete HLSL shader:

```hlsl
[[vk::location(0)]] float3 worldPos : TEXCOORD0;
[[vk::location(1)]] float3 normal   : TEXCOORD1;
[[vk::location(2)]] float2 uv       : TEXCOORD2;
[[vk::location(3)]] float3 tangent  : TEXCOORD3;
[[vk::location(4)]] float3 color    : TEXCOORD4;

[[vk::binding(0, 0)]] cbuffer UBO : register(b0) {
    float4x4 model;
    float4x4 view;
    float4x4 projection;
    float4 lightPos;
    float4 cameraPos;
    float4 params;
};

float4 main(PSInput input) : SV_Target0 {
    // Normalize the interpolated normal
    float3 n = normalize(input.normal);
    
    // Map from [-1, 1] to [0, 1]
    float3 normalColor = n * 0.5 + 0.5;
    
    // Output with alpha = 1.0
    return float4(normalColor, 1.0);
}
```

I'll normalize the incoming normal and then transform it to the color range, ensuring a clean visualization of the surface orientation.
