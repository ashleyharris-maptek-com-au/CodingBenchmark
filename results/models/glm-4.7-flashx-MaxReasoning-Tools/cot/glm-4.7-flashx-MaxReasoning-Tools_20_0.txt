Let me think about the outlier detection approach:

1. For each property, I need to identify values that are anomalous compared to neighbors
2. Confidence score based on how far from expected values

Possible approaches:
1. **Local outlier factor (LOF)** - Good for spatial data but computationally expensive
2. **Z-score / IQR method** - Simple but assumes normal distribution
3. **Median absolute deviation (MAD)** - More robust to outliers
4. **Simple range-based approach** - Compare with min/max of nearby samples

For drillhole data, samples are ordered along a path, so spatial neighbors are essentially consecutive samples in the same hole.

Adaptive strategies:
- **Small datasets**: More detailed analysis with correlation
- **Large datasets**: Statistical methods that are O(n) rather than O(n²)

Let me design the algorithm:

1. Parse input
2. For each hole and each sample, collect the property values from nearby samples (spatial neighbors)
3. Calculate a "local baseline" (e.g., median) for each property
4. For each sample, compute how anomalous its value is compared to the local baseline
5. Output samples with confidence above a threshold

Confidence calculation:
- Could be based on z-score: (value - mean) / std_dev
