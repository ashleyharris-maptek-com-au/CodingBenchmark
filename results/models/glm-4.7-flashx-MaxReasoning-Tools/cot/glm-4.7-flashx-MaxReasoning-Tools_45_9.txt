Actually, I realize I should check the exact format of each instruction again.

OpCapability
- Word count: 2 (opcode + capability)
- Operand: capability (1 = Shader)

OpExtInstImport
- Word count: 3 (opcode + result id + import id)
- Operand: result id, name string (the name string is encoded as literal string with length prefix)

OpMemoryModel
- Word count: 3 (opcode + addressing model + memory model)
- Operand: addressing model (0 = Logical), memory model (1 = GLSL450)

OpEntryPoint
- Word count: 5 + interface_count
- Operand: execution model, entry point id, name, interface variables...
- execution_model values: 0=Vertex, 1=TessellationControl, 2=TessellationEvaluation, 3=Geometry, 4=Fragment, 5=GLCompute, 6=RayGenerationNV, 7=AnyHitNV, 8=ClosestHitNV, 9=MissNV, 10=IntersectionNV, 11=CallableNV

So for Fragment:
execution_model = 4
OpEntryPoint has word count = 5 + interface_count

OpExecutionMode
- Word count: 3 (opcode + entry point id + mode)
- Operand: entry point id, mode (7 = OriginUpperLeft)

OpDecorate
- Word count: 4 + decoration operands
- Operand: target, decoration, ...operand decorations...

For BuiltIn:
decoration = 19 (BuiltIn)
For Location:
decoration = 23
