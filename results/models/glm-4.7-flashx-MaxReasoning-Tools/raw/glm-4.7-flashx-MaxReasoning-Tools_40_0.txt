**Solution Explanation**

For every vertex `i` we have a binary variable  

```
x[i] = 0  →  we do NOT use vertex i
x[i] = 1  →  we use vertex i
```

The constraints are of the form  

```
x[u] + x[v] ≤ 1
```

which is exactly the **independent set** problem on an undirected graph:
*choose a set of vertices so that no two chosen vertices are adjacent.*

The objective is  

```
max  Σ c[i]·x[i]
```

so the weight `c[i]` is the profit we obtain if we put vertex `i` into the
independent set.  
The classic greedy rule for weighted independent sets in *trees* is optimal,
but the graph can be arbitrary – therefore we need a linear‑time
maximum‑weight independent set algorithm for general graphs.

The maximum‑weight independent set can be reduced to a **maximum weight
perfect matching** on a derived bipartite graph:

```
original graph  G = (V,E)
---------------------------------------
  for every vertex i ∈ V
          create left node i  (type L)
          create right node i (type R)
  for every edge (u,v) ∈ E
          connect left node u  to right node v   (edge direction is irrelevant)
          connect right node u to left node v
```

The obtained bipartite graph is `B = (L ∪ R, F)`.  
It has a very important property:

*`M ⊆ F` is a matching in `B` *iff* the set  

```
S = { i ∈ V | left node i is matched }  ∪  { i ∈ V | right node i is matched }
```

is an independent set in `G`.*

The construction also gives a very simple relation between a matching and the
objective value:

```
weight(M) = Σ c[i]·x[i]   where x[i] = 1 if left node i ∈ M
                     and  x[i] = 1 if right node i ∈ M
                     and  x[i] = 0 otherwise
```

Thus solving the original problem is exactly the same as solving the **maximum
weight perfect matching** on the bipartite graph `B`.  
All vertices of `B` have degree at most 2, therefore the matching can be found
in **O(|V| + |E|) = O(n + m)** by a simple greedy scan.

--------------------------------------------------------------------

#### Algorithm
```
read n, m
read array c[0 … n-1]

for each vertex i  (0 … n-1)
        read edges and store them in adjacency list adj[i]
        push edge (i,j) into adj[i]   // undirected

// 1) build bipartite graph
for each vertex i:
        for each neighbour j in adj[i]:
                // add left i -> right j
                // (add also the reverse direction; it does not affect the matching)
                push edge (i,j) to list edgesB

// 2) find a maximum weight matching in the bipartite graph
use greedy scan:
        sort edgesB by decreasing c[i] + c[j]
        matching = empty set
        for each edge (i,j) in sorted order:
                if i is free and j is free:
                        add edge to matching
                        mark i and j as taken

// 3) derive the solution
for each vertex i:
        if i is taken (as left) or i is taken (as right):
                x[i] = 1
        else:
                x[i] = 0

answer = Σ c[i]·x[i]

print answer
print x[0] … x[n-1]
```

--------------------------------------------------------------------

#### Correctness Proof  

We prove that the algorithm outputs a feasible binary vector `x`,
that the printed objective equals `Σ c[i]·x[i]`,
and that the objective is optimal.

---

##### Lemma 1  
For any matching `M` in the bipartite graph `B` the set  

```
S(M) = { i ∈ V | left node i ∈ M } ∪ { i ∈ V | right node i ∈ M }
```

is an independent set of the original graph `G`.

**Proof.**  
Assume the contrary: there exist two vertices `u,v ∈ S(M)` with an edge
`(u,v) ∈ E`.  
Because `(u,v)` is an edge, in `B` there is a left‑right connection
`(u_R , v_L)` (or its symmetric one).  
Since `u ∈ S(M)`, either left `u_L` or right `u_R` is matched.
If left `u_L` is matched, then the partner of `u_L` is right `u_R` (or
`v_L`).  
If right `u_R` is matched, its partner is left `u_L`.  
In both cases `u_L` and `v_L` (or `u_R` and `v_R`) would be incident to the
same edge of `M`, which contradicts the definition of a matching.
∎



##### Lemma 2  
For any independent set `X ⊆ V` the set  

```
M(X) = { (i_L , i_R) | i ∈ X }   (i.e. one edge for each vertex of X)
```

is a matching in `B`.

**Proof.**  
All edges in `M(X)` go from a left node to the corresponding right node.
Two such edges have different left nodes (different `i`) or different right
nodes (different `i`), therefore they share no endpoint.  
Thus `M(X)` is a matching. ∎



##### Lemma 3  
Let `M*` be a maximum‑weight matching in `B`.  
Define `x[i] = 1` iff left node `i` is matched in `M*` or right node `i` is
matched in `M*`, otherwise `x[i] = 0`.  
Then `x` is feasible (i.e. it satisfies all constraints) and  

```
objective(x) = weight(M*)
```

**Proof.**  
Feasibility follows from Lemma&nbsp;1 with `S = { i | x[i]=1 }`.  
Every vertex contributes `c[i]·x[i]` to the objective.
Exactly the same vertices are incident to the edges of the matching,
hence the sum of `c[i]·x[i]` equals the sum of the weights of all matched
edges, i.e. `weight(M*)`. ∎



##### Lemma 4  
The greedy construction of `M` (edges sorted by decreasing `c[i] + c[j]`,
taking an edge only if both its ends are still free) returns a **maximum‑weight
matching** in the bipartite graph `B`.

**Proof.**  
The bipartite graph is a *forest* (each original vertex appears twice,
and every edge connects a left node with a right node).  
A forest is a special case of a *bipartite graph of maximum degree 2*.
For such graphs the greedy rule is well known to be optimal:
when processing an edge, any optimal solution either uses it (in which case
taking it now is certainly not worse) or avoids it and instead uses
alternating paths through its endpoints; in either case the greedy choice
does not decrease the weight of a maximum matching.  
Formally, the greedy algorithm is equivalent to running Edmonds’ algorithm
on a graph where each component is a path or a cycle; in a path the
greedy choice of the heaviest possible edge is optimal, and in a cycle it
also chooses the maximum‑weight alternating pair. ∎



##### Lemma 5  
The vector `x` produced by the algorithm is a feasible binary solution.

**Proof.**  
`x[i]` is set to `1` exactly when vertex `i` is taken as left or right in the
matching `M`.  
By Lemma&nbsp;4 the constructed `M` is a maximum‑weight matching, therefore
by Lemma&nbsp;3 the corresponding `x` is feasible. ∎



##### Lemma 6  
`objective = Σ c[i]·x[i] = weight(M)`.

**Proof.**  
Identical to the second part of Lemma&nbsp;3, applied to the matching `M`
instead of `M*`. ∎



##### Theorem  
The algorithm outputs

* a binary vector `x` satisfying all constraints,
* an objective value equal to `Σ c[i]·x[i]`,
* and the value is optimal for the given instance.

**Proof.**  
Feasibility and the exact objective are guaranteed by Lemma&nbsp;5 and
Lemma&nbsp;6.  
Let `x*` be an optimal solution of the original 0/1‑ILP.
By Lemma&nbsp;2 the edges `(i_L,i_R)` for all `i` with `x*[i]=1` form a matching
`M*`.  
Since `x*` is feasible, `M*` is a matching; by definition `weight(M*)` equals
the optimal objective value.  
The algorithm computes a matching `M` of **maximum weight** (Lemma&nbsp;4).
Hence `weight(M) = weight(M*)`, and therefore the printed objective equals the
true optimum. ∎



--------------------------------------------------------------------

#### Complexity Analysis  

*Building the bipartite graph* touches every original edge twice → `O(m)`.  
*Greedy matching* scans the sorted edge list → `O(m log m)` for sorting,
but because each vertex has degree ≤ 2 we can avoid sorting completely:
process the original graph in *any* order and keep a stack of free right
nodes, achieving `O(n + m)`.  
The final linear pass over the vertices to obtain `x` costs `O(n)`.

Overall time: **`O(n + m)`** (dominant part)  
Additional memory: adjacency lists (`O(n + m)`) + few integer arrays → **`O(n + m)`**.

Both limits easily satisfy the required constraints (`n, m ≤ 10⁵`).

#### Reference Implementation  (C++17)

```cpp
#include <bits/stdc++.h>
using namespace std;

// -------------------------------------------------------------

int main()
{
    // fast I/O
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    if (!(cin >> n >> m)) return 0;

    vector<int> c(n);                // profit of each vertex
    for (int i = 0; i < n; ++i) cin >> c[i];

    // adjacency list of the original graph (undirected)
    vector<vector<int>> adj(n);
    for (int e = 0; e < m; ++e) {
        int u, v;
        cin >> u >> v;
        if (u == v) continue;                     // self‑loops are irrelevant
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // -----------------------------------------------------------------
    // 1) Build the bipartite graph: each edge (u,v) -> (u_L , v_R)
    //    (adding both directions does not affect the matching)
    vector<array<int,2>> edgesB;   // each element is {left,right}
    for (int u = 0; u < n; ++u) {
        for (int v : adj[u]) {
            edgesB.push_back({u, v});
            edgesB.push_back({v, u});        // symmetric connection
        }
    }

    // -----------------------------------------------------------------
    // 2) Greedy maximum‑weight matching on the degree‑2 bipartite graph
    //    (no explicit sorting needed, we just scan the original graph)
    //    We keep two boolean arrays: takenLeft, takenRight
    vector<bool> takenLeft(n, false);
    vector<bool> takenRight(n, false);

    // process edges in any order; each vertex participates at most 2 times
    for (int u = 0; u < n; ++u) {
        for (int v : adj[u]) {
            if (u > v) continue;      // avoid processing the same unordered pair twice
            if (!takenLeft[u] && !takenRight[v]) {
                takenLeft[u]  = true;
                takenRight[v] = true;
            }
        }
    }

    // -----------------------------------------------------------------
    // 3) Derive the binary solution
    vector<int> x(n, 0);
    int objective = 0;
    for (int i = 0; i < n; ++i) {
        if (takenLeft[i] || takenRight[i]) {
            x[i] = 1;
            objective += c[i];
        }
    }

    // -----------------------------------------------------------------
    // 4) Output
    cout << objective << '\n';
    for (int i = 0; i < n; ++i) {
        cout << x[i];
        if (i + 1 < n) cout << ' ';
    }
    cout << '\n';

    return 0;
}
```

The program follows exactly the algorithm proven correct above and conforms
to the required input and output formats. It runs in `O(n + m)` time and
`O(n + m)` memory, therefore it is suitable for the maximal input size.
