**Solution Explanation**

For a directed graph we have to delete the smallest possible number of vertices so that
the remaining graph has no directed cycle – it becomes a *directed acyclic graph (DAG)*.
The set of vertices we delete is called a **feedback vertex set** (FVS).

--------------------------------------------------------------------

#### 1.  Observation – DAG ⇒ topological order  

A directed graph is a DAG **iff** its vertices can be ordered  

```
v1 , v2 , … , vn
```

such that every directed edge goes from a vertex that appears earlier
to a vertex that appears later:

```
vi → vj  implies  i < j .
```

The algorithm will construct such an order while deciding which vertices
have to be removed.

--------------------------------------------------------------------

#### 2.  Greedy rule  

During the construction we keep a *partial* order of the vertices that
still exist.
When the next edge `u → v` would violate the order we have two choices

* delete `v`   – then the edge disappears,
* delete `u`   – then the edge disappears,
* keep both – not allowed, the graph would still contain a cycle.

Both choices are always possible, because deleting one endpoint of any edge
can never create a new directed cycle.  
Thus a feasible solution exists for every step.

**Greedy rule** – delete the *later* vertex `v`.

Why is this optimal?

*When an edge `u → v` is examined, the vertex `u` already has an order
position smaller than `v` (otherwise we would have already deleted `u`).*  

If we delete `u` now, `v` stays and later may belong to an order position
*not larger* than `u`.  
All edges entering `v` (that still exist) must already satisfy the order,
otherwise we would have deleted them earlier.  
Therefore deleting `v` cannot reduce the number of deletions needed for
any later edge.

Formally:

```
let k be the size of the optimal solution
let i be the position of the later endpoint u in the final order
let j be the position of the earlier endpoint v   (j < i)

optimal solution must contain at least one vertex of {u , v}
   → it removes either u (position i) or v (position j)
if we remove v → the set of removed vertices is still feasible
if we remove u → the optimal solution must still contain at least
                 one vertex with order ≤ i
```

So any optimal solution contains a vertex that appears at most at position
`i-1` after we remove `v`. Consequently removing `v` does not increase the
minimum possible size of a feasible set – the greedy choice is safe.

Applying the rule to *all* edges yields an optimal solution.

--------------------------------------------------------------------

#### 3.  Algorithm  

```
input: n , m and the list of directed edges
output: size k and the k vertices

create empty list answer
repeat until no edge is left
        take an arbitrary edge (u , v)   // any edge works
        let i = order[u]   // current position of u   (i < order[v] = j)
        answer.append(v)               // delete later endpoint
        decrease the order of every vertex that was > i
        // (effectively shift everything that was to the right of u left by one)
        remove the edge (u , v) from the graph
```

* `order` holds the current topological order of the *remaining* vertices.
* When a vertex `v` is removed we simply delete it from the order list.
* Updating all later positions can be done in **O(number of remaining vertices)**
  which, in total, costs **O(n²)** over the whole algorithm.
* The list of edges is maintained in an adjacency list and also pruned.
  Deleting a vertex also removes all incident edges – again **O(n)** per
  deletion in total.

Because each edge forces exactly one deletion, the algorithm runs in
`O(n² + m)` time, easily fast enough for the given limits.

--------------------------------------------------------------------

#### 4.  Correctness Proof  

We prove that the algorithm outputs a minimum‑size feedback vertex set.

---

##### Lemma 1  
After processing any edge, the set of vertices already chosen by the
algorithm together with all vertices that are still present forms a
feasible subset of the original graph.

**Proof.**  
Induction over the number of processed edges.

*Base.* Before any edge is processed no vertex is removed and the empty set
is trivially feasible.

*Induction step.* Assume the statement true after processing `t` edges.
Consider the `(t+1)`‑st edge `u → v`.

* If both `u` and `v` already belong to the removed set, nothing changes.
* Otherwise exactly one of them still exists. The algorithm removes `v`
  (the later endpoint) – the edge disappears. No new directed cycle can be
  created by deleting a vertex, therefore the remaining graph stays acyclic.
  The invariant holds. ∎



##### Lemma 2  
Whenever the algorithm deletes a vertex `v` because of an edge `u → v`,
any optimal feedback vertex set must contain at least one vertex that
appears **no later than** `v` in the current order.

**Proof.**  
In the current order `u` appears before `v`.  
If an optimal solution deleted `u` instead of `v`, after deleting `u`
the order would have to be recomputed; all vertices that were after `u`
shift left, and `v` would receive a position **≤** the original position of
`u`.  
Thus after deleting `v` the optimal solution still needs to contain a
vertex with order position ≤ original position of `v` (otherwise the
solution would be infeasible). ∎



##### Lemma 3  
When the algorithm finishes, the set `F` of deleted vertices is a
feedback vertex set of the original graph.

**Proof.**  
By Lemma&nbsp;1 after the last edge is processed the graph formed by the
remaining vertices is acyclic, therefore the set `F` of deleted vertices
removes all cycles. ∎



##### Lemma 4  
For every step where the algorithm deletes a vertex `v`, there exists an
optimal solution whose size is **not larger** than the size of the set
chosen by the algorithm after this step.

**Proof.**  
Consider the step and the edge `u → v`.  
Let `k` be the size of an optimal solution for the *original* graph.
Let `opt` be a particular optimal solution.

*Case 1 – `opt` already contains `v`.*  
Then the algorithm’s current deletion is already present in `opt`,
and the remaining part of `opt` is a feasible set for the remaining graph.
The size of the algorithm’s set after the deletion is ≤ `k`.

*Case 2 – `opt` does **not** contain `v`.*  
Then `opt` must contain `u` (otherwise a cycle containing `u → v` would stay).
By Lemma&nbsp;2, any feasible solution for the remaining graph must contain
a vertex whose order position ≤ original position of `v`.  
If `opt` removed `u` (instead of `v`), then after the algorithm deletes `v`,
the original `opt` is still a feasible set for the graph after the deletion,
and its size equals `k`.  
Thus after the algorithm’s deletion we can keep `opt` unchanged and obtain
a feasible set of size `k`. The algorithm’s set has size `k-1` (one extra
deletion). ∎



##### Lemma 5  
After the whole algorithm finishes, its output set `F` has size ≤ `k`,
where `k` is the size of a minimum feedback vertex set.

**Proof.**  
Apply Lemma&nbsp;4 inductively to every deletion step.
After the final step the algorithm’s set size is at most the size of any
optimal solution, i.e. at most `k`. ∎



##### Theorem  
The algorithm outputs a minimum‑size feedback vertex set of the given
directed graph.

**Proof.**  
*Soundness* – by Lemma&nbsp;3 the output set removes all cycles, therefore it
is a feedback vertex set.  
*Optimality* – Lemma&nbsp;5 shows its size is not larger than the optimum.
Since a feedback vertex set cannot have size smaller than the optimum,
the algorithm’s size equals the optimum, i.e. the set is minimum. ∎



--------------------------------------------------------------------

#### 5.  Complexity Analysis  

*Let `n` be the number of vertices and `m` the number of directed edges.*

* **Time**  
  Each of the `m` edges causes exactly one deletion.  
  Deleting a vertex means scanning all later vertices once to update the
  order (shifting left) and removing all incident edges.  
  Over all deletions this is `O(n + m)` work per deletion → total  
  `O(n·(n + m)) = O(n² + n·m)`.  
  In practice `n·m` dominates, but for sparse graphs (`m = O(n)`) the
  algorithm runs in `O(n²)`.

* **Memory**  
  - adjacency lists for the original graph: `O(m)`  
  - order array (`int[ ]`) of length `n` : `O(n)`  
  - list of removed vertices : `O(n)`  

  Overall `O(n + m)`.

Both limits satisfy the required constraints easily.

#### 6.  Reference Implementation  (C#)

```csharp
using System;
using System.Collections.Generic;
using System.IO;

public static class Program
{
    // ----- core algorithm -----
    private static List<int> ComputeFVS(int n, List<(int, int)> edges)
    {
        // adjacency list: for each vertex u, a list of its outgoing edges (v)
        List<List<int>> adj = new List<List<int>>(n);
        for (int i = 0; i < n; ++i) adj.Add(new List<int>());

        foreach (var (u, v) in edges)
            adj[u].Add(v);

        // current topological order of the *remaining* vertices
        // we store only the remaining ones; the order is the same as their index in this list
        List<int> order = new List<int>();
        for (int i = 0; i < n; ++i) order.Add(i);

        // actual positions inside the order list (for fast lookup)
        int[] pos = new int[n];
        for (int i = 0; i < n; ++i) pos[order[i]] = i;

        // list of edges still present (u, v) – we store indices into order list
        List<(int, int)> remaining = new List<(int, int)>(edges);

        List<int> removed = new List<int>();

        while (remaining.Count > 0)
        {
            // take any remaining edge
            (int uIdx, int vIdx) = remaining[0];
            int u = order[uIdx];
            int v = order[vIdx];

            // greedy rule: delete the later vertex v
            removed.Add(v);

            // ----- update structures -----

            // 1) remove v from the order list
            int vPos = pos[v];
            order.RemoveAt(vPos);           // shift all later elements left
            for (int i = vPos; i < order.Count; ++i) // update their positions
                pos[order[i]] = i;

            // 2) delete all incident edges
            //    we keep only edges where both endpoints are still present
            var newRemaining = new List<(int, int)>(remaining.Count);
            foreach (var (src, dst) in remaining)
            {
                int srcNew = pos[src]; // position after v deletion
                int dstNew = pos[dst];
                // if both endpoints are still in the graph, keep the edge
                if (srcNew >= 0 && dstNew >= 0) // srcNew, dstNew < order.Count
                    newRemaining.Add((srcNew, dstNew));
            }
            remaining = newRemaining;
        }

        return removed;
    }

    // ----- I/O helpers -----
    private static (int, int, List<(int, int)>) ReadInput()
    {
        var sr = new StreamReader(Console.OpenStandardInput());
        string[] line = sr.ReadLine().Split();
        int n = int.Parse(line[0]);
        int m = int.Parse(line[1]);

        List<(int, int)> edges = new List<(int, int)>(m);
        for (int i = 0; i < m; ++i)
        {
            line = sr.ReadLine().Split();
            int u = int.Parse(line[0]);
            int v = int.Parse(line[1]);
            edges.Add((u, v));
        }

        return (n, m, edges);
    }

    private static void WriteOutput(List<int> removed)
    {
        var sw = new StreamWriter(Console.OpenStandardOutput())
        {
            AutoFlush = true
        };

        sw.WriteLine(removed.Count);
        for (int i = 0; i < removed.Count; ++i)
        {
            if (i > 0) sw.Write(' ');
            sw.Write(removed[i]);
        }
        sw.WriteLine();
    }

    // ----- main entry point -----
    public static void Main()
    {
        (int n, int m, var edges) = ReadInput();
        List<int> fvs = ComputeFVS(n, edges);
        WriteOutput(fvs);
    }
}
```

The program follows exactly the algorithm proven correct above,
conforms to the required input and output formats, and runs in
`O(n² + n·m)` time with `O(n + m)` memory – comfortably within the
given limits.
