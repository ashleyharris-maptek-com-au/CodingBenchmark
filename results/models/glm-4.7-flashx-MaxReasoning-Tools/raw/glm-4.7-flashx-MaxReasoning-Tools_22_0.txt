## Key Features

**Piece Support:**
- All 7 Tetris pieces (I, O, T, S, Z, J, L) with 4 rotations each
- Bitmask representation for efficient collision detection

**7 Evaluation Strategies (Adaptive):**
0. **Structure** - Column height balance
1. **Well Heights** - Minimize deep depressions
2. **Aggregate Height** - Minimize total height
3. **Holes** - Count empty spaces below blocks
4. **Bumpiness** - Measure vertical gradients
5. **Height Diversity** - Promote varied heights
6. **Random** - Random exploration

**Adaptive Logic:**
- Strategy selection based on game phase (early/mid/late)
- Board fill percentage analysis
- Recent line clears tracking
- Piece type frequency history

**Key Functions:**
- Collision detection (HasOverlap)
- Line clear simulation (EvaluateLineClears)
- Well depth detection (EvaluateWellDepth)
- Precomputed drop offsets for efficiency

## Usage

**Compilation:**
```bash
csc /optimize tetris_player.cs
```

**Runtime:**
- Input: `stdin.txt` (whitespace-separated integers, piece types)
- Output: `stdout` (one move per piece: "rotation column")
- Move response: < 5 seconds per piece

The solution handles variable board widths (1-20 columns), compiles in < 1 second, uses < 100 MB memory, and supports both short practice sessions and marathon games.
