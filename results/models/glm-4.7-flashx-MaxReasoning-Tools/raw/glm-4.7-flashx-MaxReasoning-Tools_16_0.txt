## Key Features

### 5 Adaptive Scheduling Strategies

| Scale | Strategy | Algorithm |
|-------|----------|-----------|
| **Trivial** (J≤3, M≤3, <50 tasks) | Simple greedy, no machine constraint checking |
| **Small** (J≤20, M≤20, <2000 tasks) | First-fit with machine constraint tracking |
| **Medium** (J≤100, M≤50, 2K-20K tasks) | Batch processing (10 jobs per batch) |
| **Large** (J≤1000, M≤500, 20K-200K tasks) | Time-sliced processing (J/100 jobs per slice) |
| **Extreme** (>1000 jobs, >500 machines, >200K tasks) | Large block scheduling (J/10 jobs per block) |

### Algorithm Details

**First-Fit Scheduling** (used for all but trivial):
```
For each task in order:
  current_time = max(job_previous_time, machine_free_time)
  schedule task at current_time
  update both job completion time and machine free time
```
