**Chinese Postman Problem Solver Summary:**

1. **Read Input**: Parse graph with n nodes and m edges, storing adjacency lists for efficient traversal

2. **Connectivity Check**: DFS to verify graph is connected (required for CPP)

3. **Find Odd Vertices**: Count vertices with odd degree - if none, graph is already Eulerian

4. **Compute Shortest Paths**: Run Dijkstra from each odd-degree vertex to find all-pairs shortest paths

5. **Minimum Weight Perfect Matching**:
   - For ≤20 odd vertices: Exact DP matching O(2^k × k²)
   - For >20 odd vertices: Greedy matching O(k²) per pair

6. **Add Duplicate Edges**: For each matched pair, add edges along their shortest path

7. **Find Euler Circuit**: Hierholzer's algorithm O(E) to find the tour

**Time Complexity:**
- Connectivity: O(n + m)
- Dijkstra: O(k × (m log n)) where k = number of odd vertices
- Matching: O(2^k × k²) exact, or O(k²) greedy
- Euler circuit: O(m + extra edges)

**Space Complexity:** O(n + m)

The solution handles graphs from 6 nodes to 100,000 nodes efficiently by using:
- Array-based adjacency lists
- Built-in `PriorityQueue` for Dijkstra
- Greedy matching for large odd vertex counts
- Efficient Hierholzer's algorithm for Euler circuits
