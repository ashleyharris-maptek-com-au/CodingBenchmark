**Designing a heuristic vertex cover algorithm**

I’m working out a practical plan for a 2-approximation vertex cover using a greedy maximal matching on large graphs, focusing on memory efficiency and fast I/O in Rust. I want to read edges efficiently, track matched vertices with a bool vector to keep things simple, and output the vertex cover list, all while handling very large inputs with minimal memory overhead.
