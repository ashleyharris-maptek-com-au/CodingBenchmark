My solution implements the First-Fit-Decreasing (FFD) algorithm to solve the 1D Cutting Stock Problem. This algorithm is a widely-used heuristic that provides a good approximation for this NP-hard problem, and is generally more effective at minimizing the number of stock pieces ('bins') than a simple First-Fit approach.

The core logic is as follows:
1.  **Sorting:** The cuts are first sorted in descending order of their length. This is the 'Decreasing' part of the algorithm. It prioritizes placing the largest cuts first, which tends to leave more usable space for smaller cuts later.
2.  **Optimized Placement:** For each cut, instead of linearly scanning through all existing stock pieces to find one with enough capacity, I've implemented a more efficient method. I use a `SortedList` data structure to maintain records of the available stock pieces, bucketed by their remaining capacity.
3.  **Efficient Search:** The `SortedList` keeps the capacity values sorted. I can then use a binary search on the keys of this `SortedList` to find the smallest capacity that is greater than or equal to the length of the current cut. This reduces the search time for a suitable stock from O(N) to O(log K), where K is the number of distinct capacity values (K <= N). This optimization is crucial for handling the large number of cuts mentioned in the problem description.
4.  **Updating State:** Once a cut is placed, the stock's remaining capacity is updated, and the stock is moved to the appropriate new capacity bucket in the `SortedList`. If no suitable stock is found, a new one is created.

This approach results in an overall time complexity of approximately O(N log N) (dominated by the initial sort), which is efficient enough to handle millions of cuts within the given time constraints. The solution is contained within a single `Program.cs` file as requested.
