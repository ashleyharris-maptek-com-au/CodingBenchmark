Looking at this problem, I need to handle mazes from tiny to massive scales efficiently. Let me analyze the key challenges:

1. **Memory**: A 50000×50000 maze = 2.5 billion cells. Need flat arrays and efficient storage.
2. **Algorithm**: BFS is optimal for shortest path in unweighted graphs.
3. **I/O**: Fast reading/writing is critical for huge inputs.
4. **Path reconstruction**: Store parent directions (2 bits each) to trace back from goal to start.

My approach:
- Read all input at once for speed
- Store maze and parent arrays as flat `Vec<u8>` for cache efficiency
- Use BFS with early termination when goal is found
- Store parent directions (not coordinates) to save memory
