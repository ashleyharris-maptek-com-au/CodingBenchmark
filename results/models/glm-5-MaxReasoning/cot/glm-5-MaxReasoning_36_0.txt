2. **Branching**: For a vertex v, either include it (and remove v and its neighbors from consideration) or exclude it
3. **Pruning**: If the current best + remaining vertices can't beat the best found, prune
4. **Reduction rules**:

Actually, one key insight: if the graph is disconnected, we can solve each component separately and sum up the results. This can significantly speed things up.

Also, there are several reduction rules that can simplify the problem:
1. **Isolated vertices**: Add to solution
2. **Degree-1 vertices**: Add to solution
3. **Dominated vertices**: If N(v) ⊆ N(u), and u has smaller degree, we can remove v when not including u
4. **Folding for degree-2**: If a vertex has exactly two neighbors that are not adjacent, we can "fold" them

            current.pop_back();
            added--;
