The user wants me to write a GLSL fragment shader that visualizes the surface normal as color. The task is straightforward:

1. Take the world-space normal (which may need normalization due to interpolation)
2. Map it from [-1, 1] range to [0, 1] range
3. Output as RGBA with alpha 1.0

The mapping from [-1, 1] to [0, 1] is done with: `normal * 0.5 + 0.5` or equivalently `(normal + 1.0) * 0.5`

Let me write the complete shader:

```glsl
#version 450

layout(location = 0) in vec3 worldPos;
layout(location = 1) in vec3 normal;
layout(location = 2) in vec2 uv;
layout(location = 3) in vec3 tangent;
layout(location = 4) in vec3 color;

layout(set = 0, binding = 0) uniform UBO {
    mat4 model;
    mat4 view;
    mat4 projection;
    vec4 lightPos;
    vec4 cameraPos;
    vec4 params;
} ubo;

layout(location = 0) out vec4 outColor;

void main() {
    vec3 n = normalize(normal);
    vec3 normalColor = n * 0.5 + 0.5;
    outColor = vec4(normalColor, 1.0);
}
```

This is a simple and correct solution. The normal needs to be normalized because interpolation can change the length of the vector. Then we map from [-1,1] to [0,1] and output with alpha 1.0.
