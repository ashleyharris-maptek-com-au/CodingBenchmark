The user wants Python code to solve the Capacitated Vehicle Routing Problem (CVRP).

**1. Understand the Problem:**
*   **Inputs:**
    *   $N$: Number of customers.
    *   $K$: Maximum number of vehicles.
    *   $CAPACITY$: Vehicle capacity.
    *   Depot coordinates: $(depot\_x, depot\_y)$.
    *   Customer data: $(x_i, y_i, demand_i)$ for $i=1 \dots N$.
*   **Outputs:**
    *   `total_distance`: Sum of distances for all routes.
    *   `num_routes`: Actual number of routes used.
    *   `route_i`: Sequence of 0-based customer indices.
*   **Constraints:**
    *   Each customer visited exactly once.
    *   Total demand on a route $\le CAPACITY$.
    *   Routes start and end at the depot.
    *   Minimize total distance (Euclidean).
*   **Requirements:**
    *   Read stdin, write stdout.
    *   Valid routes.
    *   Heuristics allowed (NP-hard problem).

**2. Algorithm Selection:**
CVRP is NP-hard. For a generic script that handles potentially "large instances" mentioned in the prompt, exact solvers (like Branch and Bound or dynamic programming) will likely time out or run out of memory on anything beyond ~20-30 customers. Heuristics or Meta-heuristics are necessary.

*   *Option A: Clarke-Wright Savings Algorithm.* It's a classic, deterministic, fast, and generally produces decent results. It's easy to implement.
    *   Pros: Fast, simple, scalable.
    *   Cons: Doesn't guarantee optimality, sometimes needs local search improvement.
*   *Option B: Sweep Algorithm.* Clusters customers by angle, then solves TSP within clusters.
    *   Pros: Very fast.
    *   Cons: Can be suboptimal in non-radial distributions.
*   *Option C: Nearest Neighbor + Insertion.* Construct routes greedily.
    *   Pros: Very fast.
    *   Cons: Often poor quality.
*   *Option D: Meta-heuristics (Simulated Annealing, Genetic Algorithms).*
    *   Pros: Can find better solutions.
    *   Cons: Much more code, parameter tuning required, potentially slower.

            d_val = math.sqrt((points[i][0] - points[j][0])**2 + (points[i][1] - points[j][1])**2)
            dist[i][j] = d_val
            dist[j][i] = d_val

    # Clarke-Wright Savings Algorithm
    # 1. Initialize routes
    # routes[i] stores the list of customers for the route starting at index i in `routes` list
    # Since we use a sparse list approach where routes are consumed, 
    # it's easier to use a dictionary for active routes? 
    # No, fixed list with empty markers is fine.
    
    routes = [[] for _ in range(N)] # At most N routes
    route_demands = [0] * N
    route_start = [0] * N
    route_end = [0] * N
    # Map customer index -> route index in `routes`
    node_to_route = [0] * (N + 1) 
    
    active_route_indices = []

    for i in range(1, N + 1):
        idx = i - 1 # route index
        routes[idx] = [i]
        route_demands[idx] = demands[i]
        route_start[idx] = i
        route_end[idx] = i
        node_to_route[i] = idx
        active_route_indices.append(idx)
        
    # 2. Calculate Savings
    # s_ij = d(0,i) + d(0,j) - d(i,j)
    savings = []
    for i in range(1, N + 1):
        for j in range(i + 1, N + 1):
            s = dist[0][i] + dist[0][j] - dist[i][j]
            savings.append((s, i, j))
            
    # Sort savings descending
    savings.sort(key=lambda x: x[0], reverse=True)
    
    # 3. Merge
    for s, i, j in savings:
        r_i_idx = node_to_route[i]
        r_j_idx = node_to_route[j]
        
        if r_i_idx == r_j_idx:
            continue
            
        # Check if i and j are endpoints
        # Endpoints of route r_i_idx
        ri_start = route_start[r_i_idx]
        ri_end = route_end[r_i_idx]
        
        rj_start = route_start[r_j_idx]
        rj_end = route_end[r_j_idx]
        
        # i must be start or end of its route
        # j must be start or end of its route
        i_is_endpoint = (i == ri_start or i == ri_end)
        j_is_endpoint = (j == rj_start or j == rj_end)
        
        if not i_is_endpoint or not j_is_endpoint:
            continue
            
        # Check Capacity
        if route_demands[r_i_idx] + route_demands[r_j_idx] > CAPACITY:
            continue
            
        # We merge r_j into r_i. 
        # We need to figure out the orientation to append/prepend.
        # Case A: i == ri_end, j == rj_start. Append rj to ri.
        # Case B: i == ri_start, j == rj_end. Prepend rj to ri.
        # Case C: i == ri_end, j == rj_end. Reverse rj, append.
        # Case D: i == ri_start, j == rj_start. Reverse rj, prepend.
        
        list_i = routes[r_i_idx]
        list_j = routes[r_j_idx]
        
        new_route = []
        
        if i == ri_end:
            # i is at the end of route i
            if j == rj_start:
                new_route = list_i + list_j
            else: # j == rj_end
                new_route = list_i + list_j[::-1]
        else: # i == ri_start
            # i is at the start of route i
            if j == rj_end:
                new_route = list_j + list_i
            else: # j == rj_start
                new_route = list_j[::-1] + list_i
                
        # Update route i
        routes[r_i_idx] = new_route
        route_demands[r_i_idx] += route_demands[r_j_idx]
        route_start[r_i_idx] = new_route[0]
        route_end[r_i_idx] = new_route[-1]
        
        # Invalidate route j
        routes[r_j_idx] = []
        route_demands[r_j_idx] = 0
        
        # Update node_to_route for nodes in the merged part
        # Actually, we need to update ALL nodes in new_route?
        # Nodes in list_i already point to r_i_idx.
        # Nodes in list_j need to be updated to r_i_idx.
        # Nodes in list_j are those in the `new_route` that were originally in `list_j`.
        # But simpler to just iterate the new combined list?
        # No, performance: only update the nodes that moved.
        # If list_j was reversed, the set of nodes is the same.
        for node in list_j:
            node_to_route[node] = r_i_idx
            
    # Collect final routes
    final_routes = [r for r in routes if r]

    # 4. Check K constraint and force merge if necessary
    # If len(final_routes) > K, we have too many routes.
    # We perform a greedy merge based on cost increase.
    while len(final_routes) > K:
        # Find best merge between any two routes
        best_delta = float('inf')
        best_pair = (-1, -1) # indices in final_routes
        best_config = None
        
        # Optimization: If K is small or this loop runs often, N^2 check is costly.
        # But usually this loop runs 1-2 times max.
        
        for i in range(len(final_routes)):
            for j in range(i + 1, len(final_routes)):
                r1 = final_routes[i]
                r2 = final_routes[j]
                
                # Check capacity
                d1 = sum(demands[n] for n in r1)
                d2 = sum(demands[n] for n in r2)
                if d1 + d2 > CAPACITY: continue
                
                # Try merging end(r1)-start(r2), end(r1)-end(r2), start(r1)-start(r2)...
                # We try all 4 orientations
                u1, v1 = r1[0], r1[-1]
                u2, v2 = r2[0], r2[-1]
                
                # cost = dist(end1, start2) - dist(end1, depot) - dist(depot, start2)
                # This is a heuristic.
                
                c_ee = dist[v1][u2] - dist[v1][0] - dist[0][u2] # join end1-start2
                c_es = dist[v1][v2] - dist[v1][0] - dist[0][v2] # join end1-end2 (reverse r2)
                c_ss = dist[u1][v2] - dist[u1][0] - dist[0][v2] # join start1-end2 (r2 before r1)
                c_se = dist[u1][u2] - dist[u1][0] - dist[0][u2] # join start1-start2 (rev r2, r2 before r1)
                
                # Find min delta
                # Actually we want min cost increase.
                # The base cost of going to depot and back is removed.
                
                min_delta = min(c_ee, c_es, c_ss, c_se)
                
                if min_delta < best_delta:
                    best_delta = min_delta
                    best_pair = (i, j)
                    if min_delta == c_ee: best_config = (r1, r2, False)
                    elif min_delta == c_es: best_config = (r1, r2[::-1], False)
                    elif min_delta == c_ss: best_config = (r2, r1, False)
                    elif min_delta == c_se: best_config = (r2[::-1], r1, False)
        
        if best_pair[0] != -1:
            idx1, idx2 = best_pair
            # We merge route idx2 into idx1 (remove idx2)
            # Configuration tells us how to join.
            # best_config = (part1, part2, dummy_flag)
            p1, p2, _ = best_config
            final_routes[idx1] = p1 + p2
            # Remove idx2
            final_routes.pop(idx2)
        else:
            # Cannot merge further due to capacity
            break

    # 5. Local Search (2-opt) for each route
    def two_opt(route):
        improved = True
        while improved:
            improved = False
            n = len(route)
            for i in range(n - 1):
                for j in range(i + 1, n):
                    # Edges: (i-1, i) and (j, j+1)
                    # Node indices
                    u = route[i-1] if i > 0 else 0
                    v = route[i]
                    w = route[j]
                    x = route[j+1] if j < n-1 else 0
                    
                    d_old = dist[u][v] + dist[w][x]
                    d_new = dist[u][w] + dist[v][x]
                    
                    if d_new < d_old - 1e-9:
                        route[i:j+1] = route[i:j+1][::-1]
                        improved = True
                        break
                if improved:
                    break
                    
    for r in final_routes:
        two_opt(r)
        
    # 6. Output
    total_dist = 0.0
    for r in final_routes:
        if not r: continue
        total_dist += dist[0][r[0]]
        for k in range(len(r) - 1):
            total_dist += dist[r[k]][r[k+1]]
        total_dist += dist[r[-1]][0]
        
    print(f"{total_dist:.2f}")
    print(len(final_routes))
    
    for r in final_routes:
        print(" ".join(str(n - 1) for n in r))

if __name__ == "__main__":
    solve()
