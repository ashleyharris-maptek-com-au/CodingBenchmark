# Drillhole Assay Validator Design

## Objective
Create a robust Rust CLI tool to validate drillhole assay data, ensuring data integrity and consistency.

## Scope
- Input: CSV files containing drillhole assay data.
- Validation Rules:
    - `Depth From` < `Depth To`
    - `Depth From` >= 0
    - `Depth To` > `Depth From`
    - No overlapping intervals for the same drillhole.
    - Data types (numeric).
    - Missing values.
- Output: Validation report (errors/warnings).

## Proposed Solution
- Use `serde` and `csv` crates for data parsing.
- Define a `DrillholeAssay` struct.
- Implement a `Validator` struct/trait to encapsulate validation logic.
- Use `anyhow` for error handling.

## Implementation Plan
1. Initialize Rust project (`cargo init`).
2. Define `DrillholeAssay` struct.
3. Implement validation logic.
4. Add unit tests for validation rules.
5. Implement CLI interface (using `clap`).

## Verification
- Unit tests for each validation rule.
- Integration tests with sample CSV files.
