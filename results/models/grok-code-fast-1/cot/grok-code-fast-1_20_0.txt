First, the task is to write a Rust program that validates drillhole assay data by identifying outliers that don't correlate with spatially neighboring data points.


## Understanding Task Requirements
- The task is to create a Rust program for validating drillhole assay data.  
- It involves identifying outliers by comparing data points with their spatially neighboring points.  

## Processing Input Data
- Input data will be read from stdin, with a specific format to be handled.  
- The program needs to parse and process this input to analyze spatial correlations.

## Processing Input Data
- The input format includes the number of holes, properties, and property names, followed by hole details like ID, start position, direction, length, number of samples, and sample data with depth and properties.
- For each hole, samples are provided with their depths and corresponding property values, which will be used for spatial correlation analysis.

## Defining Output Format
- Output will be sent to stdout, starting with the number of suspect entries, followed by details for each: hole ID, sample index, property name, and confidence, sorted by confidence in descending order.

## Designing Algorithm for Outlier Detection
- Outliers will be detected by comparing each sample’s property values with nearby samples in 3D space, using statistical methods like z-scores based on neighbor values.
- Confidence levels will reflect how anomalous a value is compared to its neighbors, influencing the sorting in the output.

## Handling Scalability and Efficiency
- For small datasets (under 1,000 holes), a brute force neighbor search can be used; for larger datasets, a spatial indexing approach like a grid or octree is needed due to potentially millions of points.
- Given Rust’s standard library lacks spatial structures, implementing a simple grid-based approach or approximation is necessary for efficiency, especially on large scales with AMD64 and AVX capabilities.

## Implementing Neighbor Search  
- To efficiently find neighbors, divide 3D space into cubes and look at a point’s cube and 26 adjacent cubes for neighbors.  
- Choose cube size adaptively based on dataset size, using larger cubes for smaller datasets (e.g., under 1,000 holes) and smaller for larger.  

## Computing Outlier Detection  
- For each sample and property, collect neighbor values, compute median and median absolute deviation (MAD), then calculate z-score as (value - median) / MAD.  
- Flag as suspect if z-score exceeds a threshold, with confidence as the absolute z-score, ensuring robust handling of varying data distributions.  

## Handling Input Processing  
- Read input using BufReader, parsing lines for number of holes, properties, property names, and hole details like ID, start position, direction, length, and samples.  
- Use String for hole IDs and f64 for positions, assuming direction vectors are unit vectors for position calculation along the hole.  

## Optimizing Vector Operations  
- Implement a Vec3 struct for 3D vector math, including operations like addition, subtraction, and scalar multiplication, using standard library for calculations.  
- Compute positions using start point and direction, avoiding square roots where possible by precomputing unit directions for efficiency.  

## Structuring Data Points  
- Store points as (position, hole ID, sample index, property index, value) for easy outlier analysis per property.  
- Build a grid by first finding min and max positions to determine grid resolution, aiming for about 1,000 points per bin for balance.

## Handling Edge Cases  
- Skip outlier detection for points with no neighbors, ensuring the process handles isolated cases.  
- Exclude properties with identical values (MAD = 0) from outlier analysis to avoid division by zero, maintaining robustness.  

## Managing Duplicates  
- Treat points with identical positions as separate entries, preserving all data without averaging or removal.  
- Ensure the outlier detection process accounts for potential duplicates without additional processing overhead.
