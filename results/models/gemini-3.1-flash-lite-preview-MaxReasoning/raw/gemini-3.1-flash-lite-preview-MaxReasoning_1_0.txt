# Adaptive TSP Solver Implementation Plan

## Objective
Implement a high-performance, adaptive Travelling Salesman Problem (TSP) solver in C++20 that handles city counts from 10 to 10,000,000+ within 300 seconds.

## Key Files & Context
- `src/main.cpp`: Entry point, handles I/O and invokes the solver.
- `src/tsp_solver.hpp`: Header for the TSP solver class/functions.
- `src/tsp_solver.cpp`: Implementation of the adaptive TSP strategies.

## Implementation Steps
1.  **Data Structures**: Define a `City` struct with `x` and `y` coordinates.
2.  **Fast I/O**: Implement fast I/O using `std::ios::sync_with_stdio(false); std::cin.tie(nullptr);`.
3.  **Adaptive Strategy**:
    - **Small N (< 100)**: Nearest Neighbor + 2-opt optimization.
    - **Medium N (100 <= N < 10,000)**: Nearest Neighbor + 2-opt optimization.
    - **Large N (N >= 10,000)**: Simple Greedy (Nearest Neighbor) to ensure speed.
4.  **Solver Logic**:
    - Implement `solve(std::vector<City>& cities)`.
    - Ensure the output is a valid permutation starting with 0.
5.  **Testing**:
    - Create a test script to verify output validity and performance.

## Verification & Testing
- **Correctness**: Verify the output is a valid permutation of 0..N-1 starting with 0.
- **Performance**: Measure execution time for N=10, N=1000, N=100,000, N=1,000,000.
- **Constraints**: Ensure no non-standard headers are used.
