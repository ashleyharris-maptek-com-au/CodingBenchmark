# 3D Tetrahedron Packing Solver Plan

## Objective
Design and implement a C++20 solver to maximize the number of regular tetrahedrons packed into a given container polyhedra, ensuring no overlaps and adhering to a 30-second execution time.

## Approach
1.  **Geometric Representation**:
    *   Container: Represent as a convex hull or a mesh (depending on input).
    *   Tetrahedron: Defined by 4 vertices based on edge length `e`.
2.  **Collision Detection**:
    *   Separating Axis Theorem (SAT) for tetrahedron-tetrahedron and tetrahedron-container checks.
3.  **Packing Strategy**:
    *   Spatial partitioning (e.g., Octree or Grid) to limit candidate check space.
    *   Greedy approach with randomized placement/orientation refinement (e.g., Monte Carlo or hill climbing) to maximize density.
4.  **Performance Optimization**:
    *   Parallelize using C++ `std::execution`.
    *   Minimize redundant calculations (pre-compute tetrahedron properties).
    *   Fast collision checks.

## Implementation Steps
1.  **Geometry Utility Library**: Implement basic vector/matrix/geometry math.
2.  **Collision Detection Module**: Implement SAT for convex objects.
3.  **Container/Solver Logic**: Implement the main packing loop.
4.  **Performance Tuning**: Parallelization, optimization.
5.  **Testing**: Create basic test cases (box, sphere-like, etc.) to verify correctness and timing.
