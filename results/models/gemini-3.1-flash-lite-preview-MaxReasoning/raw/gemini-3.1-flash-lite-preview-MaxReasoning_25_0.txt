# Asteroid Interception Trajectory Solver Plan

## Objective
Develop a high-precision C# application to calculate spacecraft trajectories for asteroid kinetic impact deflection, adhering to strict I/O requirements and performance constraints.

## Key Files & Context
- `Program.cs`: Main entry point, I/O handling, and orchestration.
- `TrajectorySimulator.cs`: KDK integrator, gravity propagation, and DE421 model interface.
- `Optimizer.cs`: Mission planning, trajectory optimization (Lambert's problem, differential evolution).
- `Physics.cs`: Constants, vector math, and orbital mechanics utilities.

## Implementation Steps
1. **Data Structures & Physics Engine**: Define `Body`, `State`, and `Vector3` structures. Implement KDK integrator and N-body gravity propagation.
2. **DE421 Interface**: Implement a reader for the provided DE421 ephemeris file to obtain precise planetary positions.
3. **Trajectory Optimization**:
    - Implement a Lambert's problem solver for initial trajectory estimation.
    - Implement a differential evolution or similar optimizer to refine trajectories, handle mid-course corrections, and optimize for delta-v budget.
4. **I/O Handling**: Implement stdin/stdout parsing and formatting according to the specified format.
5. **Integration & Testing**: Combine components, verify against test scenarios, and ensure performance within the 5-minute limit.
