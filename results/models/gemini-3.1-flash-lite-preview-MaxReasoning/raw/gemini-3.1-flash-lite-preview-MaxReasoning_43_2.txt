### HLSL Pixel Shader Template

```hlsl
// Standard HLSL Pixel Shader for Vulkan

// Input structure matching the vertex shader output
struct PSInput {
    float4 position : SV_POSITION; // Clip space position
    float2 uv       : TEXCOORD0;   // Texture coordinates
    float4 color    : COLOR0;      // Vertex color
};

// Resources (bound via descriptor sets in Vulkan)
// t0 maps to descriptor set 0, binding 0
Texture2D    mainTexture : register(t0);
// s0 maps to descriptor set 0, binding 1
SamplerState mainSampler : register(s0);

// Entry point
// SV_Target maps to the color attachment in the Vulkan render pass
float4 main(PSInput input) : SV_Target {
    // Example: Sample texture and multiply by vertex color
    float4 texColor = mainTexture.Sample(mainSampler, input.uv);
    
    return texColor * input.color;
}
```

### Key Considerations for Vulkan
1.  **Entry Point:** The `main` function is the entry point. The return value marked with `: SV_Target` corresponds to the color attachment defined in your Vulkan `VkRenderPass`.
2.  **Resource Binding:** The `register(tX)` and `register(sX)` syntax is used by the DXC compiler to map resources to Vulkan descriptor sets. Ensure your Vulkan pipeline layout matches these register indices.
3.  **Compilation:** Use the DirectX Shader Compiler (DXC) to compile this to SPIR-V for Vulkan:
    `dxc -T ps_6_0 -E main -spirv -fspv-target-env=vulkan1.2 shader.hlsl -Fo shader.spv`
4.  **Coordinate Systems:** HLSL/Vulkan typically uses a Y-up coordinate system for textures, but Vulkan's NDC (Normalized Device Coordinates) has Y-down. You may need to flip the Y-coordinate of your UVs or adjust your projection matrix if your textures appear upside down.
