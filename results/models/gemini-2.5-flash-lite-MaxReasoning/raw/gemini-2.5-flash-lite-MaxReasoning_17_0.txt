The 3D Bin Packing problem has been addressed with a greedy First-Fit Decreasing algorithm using a candidate-corner strategy, which provides a balance between packing quality and computational performance. The solution sorts boxes by their longest dimension and attempts to place them into the container at the first available corner position that satisfies bounds and non-overlap constraints.

### Implementation

The solution is contained in two files:

1.  **`Solver.cs`**: Contains the complete C# logic, including the `Box` class, the packing heuristic, and the `Main` method for reading input and printing the output.
2.  **`BinPacking3D.csproj`**: A minimal .NET project file to allow for easy compilation and execution using `dotnet build` and `dotnet run`.

### Usage

To compile and run the solver:

```bash
# Ensure you are in the directory containing the files
dotnet build
# Run the application (input format as specified)
dotnet run < input.txt
```

This implementation adheres to the specified constraints and should be efficient enough to handle the 3D bin packing problem within the required performance bounds.
