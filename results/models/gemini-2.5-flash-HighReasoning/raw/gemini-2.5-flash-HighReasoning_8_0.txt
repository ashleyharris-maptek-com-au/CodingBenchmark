The Rust code for the maze solver has been provided in `src/main.rs`.
The `Cargo.toml` file has also been created.

The maze solver uses a Breadth-First Search (BFS) algorithm to find the shortest path from 'A' (start) to 'B' (end) in a given maze.

Key features:
- Reads maze dimensions (H, W) and maze grid from standard input.
- Identifies 'A' and 'B' coordinates.
- Uses `std::collections::VecDeque` for efficient BFS traversal.
- Employs a `visited` matrix to store parent coordinates, enabling path reconstruction.
- Handles movement in four cardinal directions (up, down, left, right).
- Correctly identifies and avoids walls ('#').
- Reconstructs the path from 'B' to 'A' and reverses it to present the path from 'A' to 'B'.
- Outputs the number of steps and the coordinates of each step in the path.
- Outputs '0' if no path is found.
- The solution is designed to be efficient for large mazes by using BFS, which is optimal for unweighted graphs, and by avoiding recursion that could lead to stack overflow issues with very large mazes.
