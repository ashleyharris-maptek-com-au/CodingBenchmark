The proposed design plan for the asteroid interception solver is as follows:

# Asteroid Interception Solver Design

## Objective
Develop a C# application to calculate a spacecraft trajectory for kinetic impact on an asteroid, ensuring it misses Earth within 1000km, stays within the delta-v budget, and launches within the specified window.

## Architecture
- **Input Parser:** Reads `num_bodies`, `warning_days`, `impactor_mass`, `delta_v_budget`, `asteroid_mass`, `t0_unix_utc`, and body states from stdin.
- **Ephemeris Engine:** Loads and queries the JPL DE421 `.bsp` file for precise planetary positions.
- **Physics Integrator:** Implements a KDK (Kick-Drift-Kick) integrator for N-body simulations.
- **Trajectory Optimizer:**
    - Uses a Lambert solver for initial trajectory estimation.
    - Employs a differential evolution or similar optimization algorithm to refine the trajectory (launch time, burn vectors, mid-course corrections).
    - Validates against the Earth-impact constraint.
- **Output Formatter:** Writes the calculated burns and impact details to stdout.

## Implementation Steps
1. **Data Structures:** Define `Body`, `State`, and `Trajectory` classes.
2. **Ephemeris Integration:** Implement a wrapper for reading `.bsp` files (or use a lightweight library if available, otherwise implement a basic SPICE-like reader).
3. **Integrator:** Implement the KDK integrator.
4. **Optimization:**
    - Implement a Lambert solver for initial guess.
    - Implement the objective function (minimize delta-v, ensure impact).
    - Implement the optimizer loop.
5. **Main Loop:** Read input, run optimization, output results.

## Verification
- Test with provided sample inputs (if any).
- Validate against basic orbital mechanics scenarios (e.g., Hohmann transfer).
- Ensure performance within the 5-minute limit.
