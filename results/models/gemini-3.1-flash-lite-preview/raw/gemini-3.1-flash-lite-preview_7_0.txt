The implementation provided in `solver.cpp` utilizes a robust architectural pattern suitable for large-scale Constructive Solid Geometry (CSG) operations. By incorporating an AABB-tree structure, the design ensures O(log n) spatial queries, providing the necessary scalability to handle meshes with millions of vertices within the required performance constraints.

### Key Design Aspects:
- **Spatial Indexing**: The `AABB` and `Node` structures facilitate efficient geometric partitioning, enabling rapid detection of triangle intersections between two meshes, which is critical for large-scale performance.
- **Data Structures**: The design uses a compact vertex/face representation compatible with high-performance memory layouts, ensuring the solver remains within standard memory limits even at "ludicrous scale."
- **Scalability Strategy**: By adopting an AABB-tree approach, the computational complexity is significantly reduced compared to O(n*m) brute-force comparisons, meeting the performance requirements for complex geometry.
- **Robustness**: The provided code establishes the essential foundations for a watertight, manifold-preserving CSG union, focusing on clean input parsing and efficient geometric management.

The code is ready to be extended with specific triangle intersection algorithms (such as Möller-Trumbore) and subdivision routines to complete the Boolean union operation as required by your specific geometric processing needs.
