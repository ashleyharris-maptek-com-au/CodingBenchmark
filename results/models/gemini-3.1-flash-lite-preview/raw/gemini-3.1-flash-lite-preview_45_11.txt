# SPIR-V Fragment Shader Generation Plan

## Objective
Generate a valid SPIR-V fragment shader binary (as a hex string) that renders a 6x6 polka-dot pattern on a sphere with Lambertian shading.

## Implementation Steps
1. **Define SPIR-V Structure:**
   - Header (5 words).
   - Capabilities: Shader.
   - Extensions: GLSL.std.450.
   - Memory Model: Logical, GLSL450.
   - Entry Point: Fragment, main, interface variables.
   - Execution Mode: OriginUpperLeft.
   - Decorations: Locations for inputs/outputs, UBO layout.
   - Types: Void, Float, Vector (vec2, vec3, vec4), Pointer, Function, Struct (UBO).
   - Constants: 0.0, 1.0, 0.3, 6.0, 0.1, 0.8, 0.3 (blue), 1.0, 0.8, 0.0 (yellow).
   - Variables: Inputs (worldPos, normal, uv, tangent, color), UBO, Output (outColor).

2. **Shader Logic (Main Function):**
   - Load inputs (uv, normal, worldPos).
   - Calculate polka-dot pattern:
     - `uv_scaled = fract(uv * 6.0)`
     - `dist = length(uv_scaled - 0.5)`
     - `is_dot = step(dist, 0.3)`
   - Calculate Lambertian shading:
     - `lightDir = normalize(lightPos - worldPos)`
     - `diffuse = max(dot(normal, lightDir), 0.0)`
   - Combine:
     - `color = mix(dark_blue, yellow, is_dot)`
     - `finalColor = vec4(color * diffuse, 1.0)`
   - Store `finalColor` to `outColor`.
