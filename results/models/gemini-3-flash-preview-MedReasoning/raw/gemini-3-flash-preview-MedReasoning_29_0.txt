I have implemented a high-performance Maximum Clique solver in C# that is optimized for a wide range of graph complexities, from small dense graphs to large sparse instances with up to 1,000,000 vertices.

### Technical Highlights:
- **Degeneracy Ordering:** The graph is pre-processed using an $O(N+M)$ algorithm to establish a degeneracy ordering. This decomposition strategy significantly reduces the search space, especially for sparse graphs.
- **Parallel Branch and Bound:** Top-level search branches are processed in parallel using `Parallel.For`, leveraging multi-core architectures. A global `max_so_far` bound is maintained across all threads for efficient pruning.
- **Bit-Parallel MCQ Algorithm:** Sub-problems are solved using the bitset-based MCQ algorithm with greedy coloring bounds. This allows for extremely fast edge checks and intersection operations using hardware-accelerated bit manipulation (`BitOperations.TrailingZeroCount`).
- **Memory Efficiency:** The solver uses compact adjacency lists (`int[][]`) and avoids large bitset matrices for very large neighborhoods to stay within the 32GB memory limit.
- **Fast I/O:** A custom buffered `FastReader` is used to handle large edge lists (e.g., $10^8$ edges) from `stdin` efficiently.
- **Time Management:** An internal `Stopwatch` monitors the execution time, ensuring a valid solution is returned even if the 5-minute limit is approached.

The complete implementation is provided in `Program.cs`.
