The SPIR-V assembly for the passthrough geometry shader is complete. This shader takes a triangle as input and re-emits it unchanged as a triangle strip with 3 vertices, producing an identical result to rendering without a geometry shader.

The shader includes all required components:
- Geometry capability and GLSL450 memory model
- Proper entry point with all interface variables listed
- InputTriangles execution mode (3 vertices per invocation)
- OutputTriangleStrip execution mode
- OutputVertices 3 (maximum vertices emitted)
- Invocations 1 (single invocation per primitive)

The shader processes each of the 3 input vertices individually:
1. Loads the input gl_Position from the gl_PerVertex block
2. Loads the corresponding input color
3. Stores both to the output gl_PerVertex block and output color
4. Emits the vertex with OpEmitVertex
5. Ends the primitive with OpEndPrimitive

This implementation will work with a sphere mesh (triangle list) and produce the same rendering result as if no geometry shader were present.
