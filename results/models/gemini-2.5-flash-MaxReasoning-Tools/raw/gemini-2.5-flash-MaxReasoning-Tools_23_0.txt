The Lunar Lander controller is implemented in Rust, leveraging asynchronous I/O with the `tokio` runtime to handle real-time input from stdin and output to stdout.

**Key Implementation Details:**
- **Asynchronous I/O:** `tokio::spawn` is used to create separate asynchronous tasks for reading input and writing output, ensuring that the program can respond in real-time and avoid blocking.
- **Shared State:** The game state (`GameState`) is shared between the input and output tasks using `Arc<Mutex<GameState>>`. This allows safe, concurrent access and modification of the lander's state and initial map information.
- **Input Parsing:** The `input_handle` task reads initial map parameters and continuous lander state updates from stdin, parsing them into `InitialMap` and `LanderState` structs. Robust error handling for parsing is included (e.g., `unwrap_or_default`).
- **Output Generation:** The `output_handle` task periodically calls the `get_control_commands` function to determine thrust and turn values, then writes these to stdout at a frequency of 20Hz.
- **Basic Control Logic (`get_control_commands`):**
    - **Vertical Control:** A simple proportional control mechanism adjusts thrust to achieve a target vertical velocity, aiming for a soft landing. It considers gravity and maximum thrust.
    - **Horizontal Control:** A PID-like control (proportional and derivative) attempts to guide the lander towards the target x-coordinate.
    - **Angular Control:** A similar PID-like control is used to correct the lander's angle, aiming to keep it facing upwards (angle 0).
    - All control outputs are clamped to their valid ranges (thrust 0.0-1.0, turn -1.0-1.0).

This implementation provides a foundation for real-time control, with the asynchronous structure mitigating the risk of I/O blocking and the control logic offering a basic strategy for safe landing.
