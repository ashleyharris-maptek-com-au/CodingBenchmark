The Rust solver for the Hamiltonian Path problem on 2D grids with obstacles has been implemented.

**Algorithm:**
The solver uses a backtracking algorithm enhanced with two key optimizations to improve performance and scalability:
1.  **Connectivity Check**: Before making a move, a Breadth-First Search (BFS) is performed on the remaining unvisited, non-obstacle cells. If these cells do not form a single connected component, the current path is pruned, as it's impossible to visit all remaining cells. This prevents exploring dead-end branches early.
2.  **Warnsdorff's Rule Heuristic**: When choosing the next cell to visit from the current cell's neighbors, the solver prioritizes neighbors that have the fewest unvisited, non-obstacle neighbors themselves. This heuristic aims to avoid prematurely isolating parts of the grid, thereby reducing the search space and leading to solutions more quickly for many types of grids.

**Data Structures:**
-   `grid: Vec<Vec<bool>>`: Represents the grid, where `true` indicates an obstacle.
-   `visited: Vec<Vec<bool>>`: Tracks visited cells in the current path.
-   `path: Vec<(usize, usize)>`: Stores the coordinates of the current Hamiltonian path.
-   `total_clear_cells: usize`: Pre-calculated count of non-obstacle cells, used to determine if a full path has been found.

**Key Features Implemented:**
-   **Input Parsing**: Reads grid dimensions, obstacle count, and cycle requirement from stdin. Robust error handling is included for parsing errors.
-   **Starting Point**: Path always starts at (0,0).
-   **Complete Traversal**: Ensures every non-obstacle cell is visited exactly once.
-   **Adjacent Movement**: Movement is restricted to immediately adjacent cells (up/down/left/right).
-   **Cycle Requirement**: If `require_cycle` is true, the algorithm verifies that the last cell in the path is adjacent to (0,0).
-   **Output Format**: Outputs path length followed by coordinates, or `0` if no path exists.

**Performance Considerations:**
The problem emphasizes performance for grids up to 2500 cells within 5 minutes. While Hamiltonian Path is NP-complete, the implemented optimizations are standard and often effective for grid-based instances. The connectivity check, though potentially costly (involving BFS), is critical for correctness and pruning in complex scenarios. Warnsdorff's rule helps guide the search efficiently. The overall performance will depend on the density and distribution of obstacles.
