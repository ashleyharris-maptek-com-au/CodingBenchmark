Here's a complete, compilable Rust solution that outputs a 2-approximation to the Minimum Vertex Cover (for general graphs). It constructs a maximal matching greedily and returns the endpoints of the matching as the vertex cover. This is correct (a vertex cover) and guarantees a cover size at most twice the optimum, with linear-time streaming over the edges.

Notes:
- Time: O(n + m)
- Memory: O(n)
- The approach is a safe, fast heuristic suitable for graphs with up to 2M vertices and 15M edges.
- The output is two lines:
  - First line: the number of chosen vertices
  - Second line: the vertex indices (space-separated)

Code (Rust 1.92 compatible, no nightly features):

```rust
use std::io::{self, Read, Write};

fn read_usize(input: &[u8], idx: &mut usize) -> usize {
    let n = input.len();
    // Skip non-digit characters
    while *idx < n {
        let b = input[*idx];
        if b >= b'0' && b <= b'9' {
            break;
        }
        *idx += 1;
    }
    // Parse digits
    let mut val: usize = 0;
    while *idx < n {
        let b = input[*idx];
        if b < b'0' || b > b'9' {
            break;
        }
        val = val * 10 + (b - b'0') as usize;
        *idx += 1;
    }
    val
}

fn main() {
    // Read entire stdin into a byte buffer
    let mut input = Vec::new();
    io::stdin().read_to_end(&mut input).unwrap();
    let mut pos = 0usize;

    // Read number of vertices and edges
    let n = read_usize(&input, &mut pos);
    let m = read_usize(&input, &mut pos);

    // Track which vertices are already in the matching (i.e., used)
    let mut used = vec![0u8; n];
    // Store endpoints of the maximal matching (these vertices form a vertex cover)
    let mut endpoints: Vec<usize> = Vec::new();

    // Process edges in a streaming fashion
    for _ in 0..m {
        let u = read_usize(&input, &mut pos);
        let v = read_usize(&input, &mut pos);

        // If neither endpoint is already matched, take this edge into the matching
        if u < n && v < n && used[u] == 0 && used[v] == 0 {
            used[u] = 1;
            used[v] = 1;
            endpoints.push(u);
            endpoints.push(v);
        }
    }

    // Output the result:
    // line 1: number of selected vertices
    // line 2: the vertices (space-separated)
    let mut out = io::BufWriter::new(io::stdout());
    writeln!(out, "{}", endpoints.len()).unwrap();

    if !endpoints.is_empty() {
        write!(out, "{}", endpoints[0]).unwrap();
        for &x in endpoints.iter().skip(1) {
            write!(out, " {}", x).unwrap();
        }
    }
    writeln!(out).unwrap();
}
```

Usage notes:
- Input format is exactly as you described: first line "num_vertices num_edges", followed by m lines of "u v" (0-indexed).
- The output is two lines: the first is the count of vertices in the cover, the second lists the vertex indices in the cover.
- This is a 2-approximation algorithm for general graphs; it guarantees coverage of all edges but may not be the absolute minimum for large or dense graphs.
