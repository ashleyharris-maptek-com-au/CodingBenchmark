Here's a self-contained Rust program that provides a scalable graph layout solver with a multi-strategy approach:

- For small to moderately-sized graphs (roughly n ≤ 2000 and with edges), it uses a Fruchterman–Reingold style force-directed layout (O(n^2) per iteration, but limited to small graphs).
- For larger graphs, it falls back to a simple grid layout that scales well and avoids pathological runtimes.
- It gracefully handles disconnected graphs, self-loops, and parallel edges (self-loops are ignored for layout; parallel edges are deduplicated).

The code prints one line per node with the final x and y coordinates (floats). It’s designed to compile with stable Rust 1.92-era toolchains and requires no external crates.

Code (compilable Rust):

```rust
use std::collections::HashSet;
use std::io::{self, Read};

fn fr_layout(n: usize, edges: &Vec<(usize, usize)>) -> Vec<(f64, f64)> {
    // Initialize positions on a circle
    let mut pos = vec![(0.0f64, 0.0f64); n];
    let r = (n as f64).max(1.0);

    for i in 0..n {
        let ang = 2.0 * std::f64::consts::PI * (i as f64) / (n as f64);
        pos[i].0 = r * ang.cos();
        pos[i].1 = r * ang.sin();
    }

    let mut velx = vec![0.0f64; n];
    let mut vely = vec![0.0f64; n];

    // Parameters
    let area = (n as f64) * 8.0;
    let k = (area / (n as f64)).sqrt();

    // Iterations tuned by size
    let iters = if n <= 600 { 100 } else if n <= 1200 { 60 } else { 40 };

    for _step in 0..iters {
        let mut fx = vec![0.0f64; n];
        let mut fy = vec![0.0f64; n];

        // Repulsive forces (between all pairs)
        for i in 0..n {
            for j in (i + 1)..n {
                let dx = pos[i].0 - pos[j].0;
                let dy = pos[i].1 - pos[j].1;
                let mut dist = (dx * dx + dy * dy).sqrt();
                if dist < 1e-6 {
                    dist = 1e-6;
                }
                let force = (k * k) / dist;
                let ux = dx / dist;
                let uy = dy / dist;
                fx[i] += ux * force;
                fy[i] += uy * force;
                fx[j] -= ux * force;
                fy[j] -= uy * force;
            }
        }

        // Attractive forces along edges
        for &(u, v) in edges {
            let dx = pos[u].0 - pos[v].0;
            let dy = pos[u].1 - pos[v].1;
            let dist = (dx * dx + dy * dy).sqrt().max(1e-6);
            let force = (dist * dist) / k;
            let ux = dx / dist;
            let uy = dy / dist;
            fx[u] -= ux * force;
            fy[u] -= uy * force;
            fx[v] += ux * force;
            fy[v] += uy * force;
        }

        // Integrate velocities and positions
        let mut max_move = 0.0;
        for i in 0..n {
            velx[i] = (velx[i] + fx[i]) * 0.85;
            vely[i] = (vely[i] + fy[i]) * 0.85;
            let dx = velx[i] * 0.1;
            let dy = vely[i] * 0.1;
            pos[i].0 += dx;
            pos[i].1 += dy;
            let mv = (dx * dx + dy * dy).sqrt();
            if mv > max_move {
                max_move = mv;
            }
        }

        // Early exit if layout has stabilized
        if max_move < 1e-4 {
            break;
        }
    }

    pos
}

fn grid_layout(n: usize) -> Vec<(f64, f64)> {
    let mut pos = vec![(0.0f64, 0.0f64); n];
    let w = (n as f64).sqrt().ceil() as usize;
    let spacing = 8.0;
    for i in 0..n {
        let row = i / w;
        let col = i % w;
        pos[i] = (col as f64 * spacing, row as f64 * spacing);
    }
    pos
}

fn main() {
    // Read entire input
    let mut input = String::new();
    io::stdin().read_to_string(&mut input).unwrap();
    let mut it = input.split_whitespace();

    // If there is no input, nothing to do
    let n_opt = it.next();
    let n: usize = match n_opt {
        Some(s) => s.parse().unwrap_or(0),
        None => return,
    };
    let m: usize = match it.next() {
        Some(s) => s.parse().unwrap_or(0),
        None => 0,
    };

    // Deduplicate edges and ignore self-loops
    let mut seen: HashSet<(usize, usize)> = HashSet::new();
    let mut edges: Vec<(usize, usize)> = Vec::new();

    for _ in 0..m {
        let u = match it.next() {
            Some(x) => x.parse().unwrap_or(n), // fall back if parse fails
            None => n,
        };
        let v = match it.next() {
            Some(x) => x.parse().unwrap_or(n),
            None => n,
        };
        if u >= n || v >= n || u == v {
            continue;
        }
        let a = if u < v { (u, v) } else { (v, u) };
        if seen.insert(a) {
            edges.push(a);
        }
    }

    // Optional: build adjacency (not strictly needed for layout)
    // let mut adj = vec![Vec::<usize>::new(); n];
    // for &(u, v) in &edges {
    //     adj[u].push(v);
    //     adj[v].push(u);
    // }

    // Choose layout strategy
    // Small graphs with edges: force-directed FR
    // Otherwise: grid layout
    let positions = if n <= 2000 && !edges.is_empty() {
        fr_layout(n, &edges)
    } else {
        grid_layout(n)
    };

    // Normalize/scale coordinates to a pleasant range
    let mut minx = std::f64::INFINITY;
    let mut maxx = std::f64::NEG_INFINITY;
    let mut miny = std::f64::INFINITY;
    let mut maxy = std::f64::NEG_INFINITY;

    for &(x, y) in &positions {
        if x < minx { minx = x; }
        if x > maxx { maxx = x; }
        if y < miny { miny = y; }
        if y > maxy { maxy = y; }
    }

    let width = (maxx - minx).abs().max(1.0);
    let height = (maxy - miny).abs().max(1.0);
    let scale_x = 1000.0 / width;
    let scale_y = 1000.0 / height;

    let mut out = Vec::with_capacity(n);
    for &(x, y) in &positions {
        let sx = (x - minx) * scale_x;
        let sy = (y - miny) * scale_y;
        out.push((sx, sy));
    }

    // Print coordinates: one line per node
    let mut stdout = String::new();
    for i in 0..n {
        stdout.push_str(&format!("{:.6} {:.6}\n", out[i].0, out[i].1));
    }

    print!("{}", stdout);
}
``` 

Notes and caveats:
- Self-loops are ignored for layout purposes (they don’t affect positions).
- Parallel edges are deduplicated; the layout uses the simplified edge set for attractive forces.
- The small-fr layout is O(n^2) per iteration and thus is only used when n ≤ 2000 to keep runtimes reasonable. For very large graphs, the program uses a deterministic grid layout that scales to tens/hundreds of thousands of nodes.
- The output is deterministic (no randomness). Coordinates are scaled to fit roughly within [0, 1000] x [0, 1000].
- If you want to run with a different bounding box or add more sophisticated multi-level approaches, you can extend the layout_selection logic in main().
