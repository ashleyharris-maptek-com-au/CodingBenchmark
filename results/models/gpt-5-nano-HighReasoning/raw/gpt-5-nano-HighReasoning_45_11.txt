Here’s a complete, ready-to-compile fragment shader in GLSL that implements your 6x6 UV polka-dot pattern with circular dots (radius 0.3 in cell space), yellow dots on a dark blue background, Lambertian shading, and alpha 1.0. It uses a UBO for a light position to drive Lambert shading. You can compile this to SPIR-V and then convert the binary to a hex string for your .spv.

GLSL fragment shader (Fragment shader, GLSL 450)
- Input interface (as per your available inputs):
  - location 0: vec3 worldPos
  - location 1: vec3 normal
  - location 2: vec2 uv
- Output:
  - location 0: vec4 outColor
- Uniform:
  - set 0, binding 0: a UBO containing lightPos (in world space)

Shader:
#version 450
layout(location = 0) in vec3 worldPos;
layout(location = 1) in vec3 normal;
layout(location = 2) in vec2 uv;

layout(location = 0) out vec4 outColor;

layout(set = 0, binding = 0) uniform UBO {
    mat4 model;
    mat4 view;
    mat4 proj;
    vec4 lightPos; // world-space position
    vec4 camPos;
    vec4 params;
} ubo;

void main() {
    // 6x6 grid in UV space, polka-dot pattern in cell space
    vec2 uv6 = uv * 6.0;
    vec2 local = fract(uv6) - 0.5;      // range [-0.5, 0.5)
    float dist = length(local);          // distance from cell center

    vec3 dotColor = vec3(1.0, 0.8, 0.0); // yellow
    vec3 bgColor  = vec3(0.1, 0.1, 0.3); // dark blue

    vec3 baseColor = (dist < 0.3) ? dotColor : bgColor;

    // Lambertian shading
    vec3 n = normalize(normal);
    vec3 lightDir = normalize(ubo.lightPos.xyz - worldPos);
    float lam = max(dot(n, lightDir), 0.0);

    // Basic ambient to avoid complete darkening when facing away
    vec3 ambient = vec3(0.05);

    vec3 color = baseColor * lam + ambient;
    outColor = vec4(color, 1.0);
}

How to produce the SPIR-V hex
- Save the GLSL to a file, e.g., dot6_frag.glsl
- Compile to SPIR-V:
  - glslangValidator -V dot6_frag.glsl -S frag -o dot6_frag.spv
- Convert the SPIR-V binary to a hex string (raw bytes, little-endian representation). Example (Unix-like shell):
  - xxd -p -c 4 dot6_frag.spv | tr -d '\n'
  - This prints the raw SPIR-V bytes as hex, with 8 characters per 4-byte word (since SPIR-V is 32-bit words in little-endian).
- Use the resulting hex string as the SPIR-V binary input for your pipeline.
