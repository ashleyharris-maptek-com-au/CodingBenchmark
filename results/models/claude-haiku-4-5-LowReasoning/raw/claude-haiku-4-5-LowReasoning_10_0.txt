I've written a C++ bin packing solver for the 2D Cutting Stock Problem. Here's the approach:

**Algorithm:**
1. **Sort by area** (largest first) - Places larger, harder-to-fit rectangles first
2. **Greedy placement** - For each rectangle, tries to place it in existing boards
3. **Smart position candidates** - Only tries positions at corners of existing rectangles rather than all possible positions (major efficiency gain)
4. **Rotation support** - Tries both orientations and picks whichever fits
5. **Collision detection** - Verifies no overlaps and stays within board bounds

**Key optimizations for scalability:**
- Uses position hints rather than exhaustive grid search - reduces complexity significantly
- Fast I/O with `sync_with_stdio(false)`
- Efficient duplicate removal for candidate positions
- Works well for both small (trivial) and large (ludicrous) problem sizes

The solution outputs the number of boards needed, then for each rectangle (in input order): `board_index x y rotated`.
