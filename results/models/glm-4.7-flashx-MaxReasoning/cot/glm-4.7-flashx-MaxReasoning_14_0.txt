1.  **Understand the Goal:** The user wants a C# Minesweeper solver. Key requirements:
    *   Input: Board size (H, W), followed by H rows of strings.
    *   Output: List of provably safe (empty, unrevealed) cells with their 0-indexed coordinates.
    *   Constraints: Handle trivial to massive boards (10000x10000). No guessing (only safe moves). Efficient.
    *   Environment: Windows 11, 64-bit, lots of memory, supports SIMD (though C# standard `System.Numerics` might be tricky without external libs, but standard `System` is fine).

2.  **Algorithm Strategy:**
    *   *Core Logic:* This is a Constraint Satisfaction Problem (CSP). I need to find cells that must be empty based on the current revealed numbers and flagged mines.
    *   *Data Structures:*
        *   Board representation: A 2D array or 1D array. For 10000x10000, a 2D array `char[,]` or `byte[,]` is okay (100M chars ~ 100MB if using `char`, but `byte` is better). Actually, a 1D `int[]` or `bool[]` is more cache-friendly. Let's use a `byte` array where `0`=unknown, `1`=flagged, `2`=revealed number.
        *   Neighbor lists: To efficiently iterate neighbors, pre-calculating neighbor offsets is crucial. Each cell has 8 neighbors (except edges).
    *   *Algorithmic Steps:*
        1.  **Initialize:** Parse input.
        2.  **Pattern Matching (The "Easy" Logic):**
            *   0-cell logic: All unrevealed neighbors of a '0' are safe. This is the fastest way to clear a massive board.
            *   1-n logic (1-1 pattern): Two revealed cells that share exactly one unrevealed neighbor. If one is 1, that neighbor is a mine. If one is a number and that number equals the count of unrevealed neighbors, all neighbors are safe.
            *   1-n logic (1-1 pattern variant): Two revealed cells with a gap? No, simpler is usually better for a robust solver. The "count == unrevealed neighbors" logic is robust.
        3.  **Constraint Propagation:**
            *   While we find new safe cells, update neighbors.
            *   If a neighbor of a revealed cell is flagged, reduce that cell's count.
            *   If a revealed cell's count reaches 0, use the 0-cell logic again.
            *   If a revealed cell's count equals the number of remaining unrevealed neighbors, those neighbors are mines (and thus safe).
        4.  **Advanced Logic (The "Hard" Logic - Chaining):**
            *   A simple propagation loop is usually sufficient for most "trivial to moderate" cases.
            *   For "ludicrous" scale (10000x10000), we need a smart iterative approach, not recursion to avoid stack overflow. BFS/Queue is best.

3.  **Optimization for Large Boards (10k x 10k):**
    *   *Memory:* 10000x10000 = 100,000,000 cells. Using `byte` array (byte 0-255) is sufficient.
        *   `0`: Unknown
        *   `1`: Flagged Mine
        *   `2`: Safe Revealed (0-8)
    *   *Access:* 1D indexing `y * width + x` is significantly faster than `arr[y, x]` due to boundary checks.
    *   *Neighbors:* `int[8]` offsets.
    *   *Queue:* Use `System.Collections.Concurrent.ConcurrentQueue`? Or just standard `Queue`? For a single-threaded solver on Windows with 64 cores, standard `Queue` is usually fast enough for this specific I/O-bound/CP-heavy logic. Let's stick to `System.Collections.Generic.Queue`.
    *   *Pre-allocations:* Avoid creating lists/queues inside the inner loop. Reuse.

4.  **Refining the Solver Logic (Step-by-Step):**

    *   **Input Parsing:** Read all lines into a list.
    *   **Board Construction:** Convert to 1D `byte` array. Note: Input might have spaces ' ' or digits '0'-'8' or flags '*'.
    *   **The Solver Loop:**
        *   We need a queue of "Work to do".
        *   *Initial Work:* Add all revealed cells ('0'-'8') to the queue. Actually, we can run logic immediately on them.
        *   *Pass 1: Propagate 0s.*
            *   Find '0'. Look at 8 neighbors. If neighbor is ' ', mark as safe. Add neighbor to queue.
            *   *Wait, when we mark a neighbor safe, do we check the neighbors of that neighbor?*
            *   Usually, in Minesweeper, if we mark a cell as safe, we *reveal* it. Revealing a cell means checking its neighbors.
            *   *Correction:* We shouldn't reveal cells in the solver output unless we are adding them to a list of "safe moves". However, to solve the board, we treat marked safe cells as "revealed" (i.e., they are no longer ' ') so we can propagate constraints through them.
        *   *Pass 2: Prove Mines/Rest.*
            *   Iterate through all revealed cells (count == known_value).
            *   Count unrevealed neighbors (' ').
            *   Count flagged neighbors ('*').
            *   Unrevealed left = Count - Flags.
            *   If Unrevealed left == 0: All are safe (unlikely to hit here, but possible).
            *   If Unrevealed left == Cell_Value: The unrevealed neighbors MUST be mines.
                *   If a neighbor is '*', it was already a mine (invalid state, but possible input).
                *   If a neighbor is ' ', mark as '*'.
                *   If we mark a mine, does it change anything? Yes, it affects the count of neighbors for *other* revealed cells.
            *   If Unrevealed left == Cell_Value: The unrevealed neighbors MUST be SAFE. (Wait, that's the same logic. If I have a '2' with 2 unknowns, they could be mines OR safe. But if I have a '1' with 1 unknown, it's a mine. If I have a '1' with 0 unknowns, the '1' is a red herring/error).
            *   *Correction:* The standard logic is:
                *   `UnrevealedNeighbors = Count`
                *   `FlaggedNeighbors = Count of '*'`
                *   `Remaining = UnrevealedNeighbors - FlaggedNeighbors`
                *   If `Remaining == 0`, safe.
                *   If `Remaining == CellValue`, all unrevealed neighbors are Mines.
                *   If `CellValue == UnrevealedNeighbors`, all unrevealed neighbors are Safe.
    *   **Special Case - 1-1 Pattern:**
        *   This is often the missing piece in simple solvers.
        *   Look for two revealed cells $A$ and $B$ that share a common neighbor $C$.
        *   If $A$ and $B$ are distinct, and $A + B = 1$ (sum of their values), then $C$ is a mine.
        *   Also, if they share multiple neighbors, it gets complex. For a "simple" solver that handles massive boards, basic constraint propagation is usually preferred over complex pairwise pattern matching unless necessary. Given the "10000x10000" requirement, pattern matching O(N^2) is impossible. We must rely on single-cell constraint propagation. The prompt asks for "adaptive logic", so I will focus on the efficient propagation first.

5.  **Handling the "10000x10000" Constraint:**
    *   If we iterate through *every* cell to check neighbors, it's $O(H \times W \times 8)$. For 100M cells, that's 800M operations. This is fine in C# if optimized (stackalloc, 1D arrays).
    *   However, we don't need to iterate *all* cells constantly. We only care about "Revealed" cells.
    *   *Data Structure:* Keep a list of indices for "Revealed" cells.
    *   Whenever we mark a cell safe or mine, we don't add it to the revealed list. We only add it to the "Safe" or "Mine" pool.
    *   *Wait,* if a cell becomes "Revealed" (Safe), it affects its neighbors. If we just have a list of revealed cells, we don't have the updated state of newly revealed cells to propagate.
    *   *Solution:* We must iterate through all *currently revealed* cells at each "pass".
    *   Optimization: Every time we find a new safe cell or a new mine, we can update the counts of its neighbors.
    *   Algorithm:
        1.  Create a queue of "Revealed Cell Indices".
        2.  Initial Queue: All cells that are '0'-'8'.
        3.  While Queue is not empty:
            *   Pop index `idx`.
            *   Get neighbors. Count unrevealed (' '), flagged ('*').
            *   Update `Unrevealed` count.
            *   Update `Flagged` count.
            *   Apply rules:
                *   Rule 1 (Empty): If `CellValue == 0`, mark all neighbors 'Safe'.
                *   Rule 2 (All Mines): If `Unrevealed == CellValue`, mark all neighbors 'Mine'.
                *   Rule 3 (All Safe): If `Unrevealed == CellValue` (and strictly, `Flagged == 0`? No, that means 0 mines left to place, so they must be safe). Wait. If I have a '1' and 1 unrevealed, it's a mine. If I have a '1' and 1 flagged, the flagged is the mine. If I have a '1' and 0 unrevealed, the board state is inconsistent or the '1' was just an info marker.
                *   Actually, the most useful logic for a solver is:
                    *   `CountFlags = 0`
                    *   `CountUnknown = 0`
                    *   Iterate neighbors.
                    *   If neighbor is ' ', `CountUnknown++`.
                    *   If neighbor is '*', `CountFlags++`.
                    *   If `CountUnknown == CellValue + CountFlags`:
                        *   All ' ' neighbors are Safe. (They hold the remaining 0 mines).
                    *   If `CountUnknown == CellValue`:
                        *   All ' ' neighbors are Mines.
    *   *Efficiency Check:*
        *   Iterating the queue of revealed cells every step.
        *   New revealed cells are added to the queue.
        *   We need a list of all "active" revealed cells (indices) to iterate over.

    using System.Collections.Generic;
    using System.IO;

    public class MinesweeperSolver
    {
        static void Main() {
            // Setup
            // Input
            // Logic
            // Output
        }

        // Constants for cell types
        const byte UNKNOWN = 0;
        const byte REVEALED_SAFE = 1; // holds value 0-8
        const byte FLAGGED_MINE = 2;

        void Solve(string[] lines) {
            int H = int.Parse(lines[0].Split()[0]);
            int W = int.Parse(lines[0].Split()[1]);
            byte[] board = new byte[H * W];

            // Parse input
            // ...

            Queue<int> workQueue = new Queue<int>();
            List<int> safeMoves = new List<int>();
            List<int> flaggedMoves = new List<int>(); // To track what we marked

            // Initialize work queue with all revealed cells (0-8)
            for (int y = 0; y < H; y++) {
                for (int x = 0; x < W; x++) {
                    int idx = y * W + x;
                    char c = lines[y + 1][x];
                    if (c >= '0' && c <= '8') {
                        board[idx] = (byte)(c - '0' + REVEALED_SAFE);
                        workQueue.Enqueue(idx);
                    } else {
                        // Keep as UNKNOWN or parse FLAG
                        if (c == '*') board[idx] = FLAGGED_MINE;
                        else board[idx] = UNKNOWN;
                    }
                }
            }

            // Solver Loop
            while (workQueue.Count > 0) {
                int idx = workQueue.Dequeue();
                int y = idx / W;
                int x = idx % W;
                int value = board[idx] - REVEALED_SAFE; // Actual mine count

                int countUnknown = 0;
                int countFlagged = 0;

                // Iterate Neighbors
                // Offsets: NW, N, NE, W, E, SW, S, SE
                for (int dy = -1; dy <= 1; dy++) {
                    for (int dx = -1; dx <= 1; dx++) {
                        if (dy == 0 && dx == 0) continue;

                        int ny = y + dy;
                        int nx = x + dx;

                        if (ny >= 0 && ny < H && nx >= 0 && nx < W) {
                            int nIdx = ny * W + nx;
                            byte nType = board[nIdx];
                            if (nType == UNKNOWN) countUnknown++;
                            else if (nType == FLAGGED_MINE) countFlagged++;
                        }
                    }
                }

                // Logic: If unrevealed neighbors == current value, they are mines
                if (countUnknown == value) {
                    for (int dy = -1; dy <= 1; dy++) {
                        for (int dx = -1; dx <= 1; dx++) {
                            if (dy == 0 && dx == 0) continue;
                            int ny = y + dy;
                            int nx = x + dx;
                            if (ny >= 0 && ny < H && nx >= 0 && nx < W) {
                                int nIdx = ny * W + nx;
                                if (board[nIdx] == UNKNOWN) {
                                    // Flag this mine
                                    board[nIdx] = FLAGGED_MINE;
                                    // Do we add neighbors of this mine to queue? No, mines are flags.
                                    // Do we add the mine to flagged list for output? Yes.
                                    // Optimization: We need to process neighbors of this mine.
                                    // Wait, flagged mines don't propagate info unless they are counted as mines.
                                    // Our loop checks neighbors. We just need to ensure the flagged mines
                                    // are updated in the board so they count correctly.
                                    // Actually, adding flagged mine to a queue to process its neighbors is wasteful
                                    // because it just tells us "I am a mine". But it doesn't help solve *other* cells
                                    // unless we re-check the original revealed cell.
                                    // We ARE re-checking original revealed cells (the workQueue).
                                    // So we don't need to flag specific updates for propagation, just for output.
                                }
                            }
                        }
                    }
                }

                // Logic: If unrevealed neighbors + flagged neighbors == value, they are safe
                // (Essentially remaining unknown must be 0)
                if (countUnknown + countFlagged == value) {
                    for (int dy = -1; dy <= 1; dy++) {
                        for (int dx = -1; dx <= 1; dx++) {
                            if (dy == 0 && dx == 0) continue;
                            int ny = y + dy;
                            int nx = x + dx;
                            if (ny >= 0 && ny < H && nx >= 0 && nx < W) {
                                int nIdx = ny * W + nx;
                                if (board[nIdx] == UNKNOWN) {
                                    // Mark Safe
                                    board[nIdx] = REVEALED_SAFE; // Temporarily revealed
                                    // Add to work queue so we process its neighbors
                                    workQueue.Enqueue(nIdx);
                                    // Add to output list
                                    safeMoves.Add(nIdx);
                                }
                            }
                        }
                    }
                }
            }
        }
    }
