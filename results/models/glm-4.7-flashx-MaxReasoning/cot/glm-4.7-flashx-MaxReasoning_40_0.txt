Sort remaining by degree descending.
Assign color 1 to first.
Iterate others: assign smallest color not used by neighbors.
The largest color class size gives the max sum of weights in that class.
This is a valid lower bound for the remaining subgraph.
This significantly speeds up the search for harder instances.
I'll try to implement this.

**Greedy Coloring Implementation:**
Input: `vector<int> rem`
1.  Sort `rem` by degree descending.
2.  Create `color` array for `rem`.
3.  `colors_assigned` count.
4.  For each $v$ in `rem`:
    *   Find colors of neighbors in `rem` that are already colored.
    *   Greedy pick smallest available color.
5.  Find max color index.
6.  Sum costs of vertices in that max color class.

This bound is $O(N^2)$ in worst case (checking all neighbors).
For $N=100$, it's fine. For $N=500$, it's 250k ops, negligible.
I will include this.

                }

            return comp_adj[a].size() > comp_adj[b].size();
