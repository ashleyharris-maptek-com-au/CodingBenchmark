The user wants a SPIR-V binary fragment shader encoded as a hex string.
The shader needs to implement Blinn-Phong shading on a unit sphere.
It must accept specific inputs and read from a UBO.

**Requirements Analysis:**
1.  **Language/Format:** Raw SPIR-V binary, hex encoded string (no spaces/newlines).
2.  **Header:** Magic number (0x07230203), Version (1.0), Generator (0), Bound (need to calculate ID count), Reserved.
3.  **Capabilities:** Shader.
4.  **Extensions:** GLSL.std.450 (needed for normalize, dot, etc.).
5.  **Memory Model:** Logical, GLSL450.
6.  **Entry Point:** Fragment, "main", input location 0 (worldPos) and 2 (uv).
7.  **Execution Mode:** OriginUpperLeft.
8.  **Decorations:**
    *   Inputs need `Location`.
    *   UBO struct needs `Block`, `Offset`, `DescriptorSet`, `Binding`.
    *   Output needs `Location 0`.
9.  **Types & Variables:**
    *   Input types (float, vec2, vec3, vec4).
    *   Output type (vec4).
    *   UBO type (struct).
    *   Global variable for UBO.
    *   Local variables (N, L, V, H, diffuse, specular, etc.).
10. **Logic:**
    *   Calculate N (normal). Since it's a unit sphere, worldPos and normal are the same vector (but user inputs normal explicitly, so just load it). Or normalize worldPos if worldPos is the radius. The prompt says "rendered on a unit sphere". The input `worldPos` is at the surface, so `worldPos` length is 1. So `N = worldPos`.
    *   Calculate V (view vector). Camera is at `cameraPos` in UBO. `V = normalize(cameraPos - worldPos)`.
    *   Calculate L (light direction). Light is at `lightPos` in UBO. Since it's a unit sphere, L is just the normalized light position. Or is the light "in world space"? Yes, usually.
    *   Calculate H = normalize(L + V).
    *   Calculate diffuse = max(dot(N, L), 0).
    *   Calculate specular = pow(max(dot(N, H), 0), 64).
    *   Final color = `vec4(0.1 + 0.6*diffuse + 0.8*specular, 1.0)`.
11. **SPIR-V Opcode decoding:** Need to map decimal opcodes to words.
